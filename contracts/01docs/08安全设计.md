# 协议定义层（Contracts）安全设计

> contracts/ 的安全关注点：协议层面的输入验证、敏感字段标注、数据最小化

---

## 一、安全设计原则

| 原则 | 说明 |
|------|------|
| **输入验证左移** | Proto 定义层即标注字段约束，减少运行时校验 |
| **敏感字段标注** | PII字段在proto中明确标注 |
| **最小暴露** | 不同调用方看到不同的字段子集（通过BFF裁剪） |
| **向后兼容安全** | 协议变更不引入安全漏洞 |

---

## 二、敏感字段标注规范

```protobuf
message UserAccount {
  string user_id = 1;
  // @sensitive: PII - 需脱敏
  string phone_number = 2;
  // @sensitive: PII - 不可输出到日志
  string email = 3;
  // @internal: 仅服务端可见
  string password_hash = 4;
}
```

---

## 三、字段约束

| 约束类型 | Proto实现 | 示例 |
|---------|-----------|------|
| 长度限制 | 注释标注 | `// max: 256 chars` |
| 值域限制 | 枚举 | `enum UserStatus { ... }` |
| 正则约束 | 注释标注 | `// pattern: ^[a-z0-9_]{3,32}$` |
| 必填校验 | proto3 optional | 区分未设置和默认值 |

---

## 四、协议安全审查清单

| 检查项 | 说明 |
|--------|------|
| 无明文密码字段 | 密码哈希不通过Proto传输 |
| Token有过期时间 | 所有Token消息含expires_at |
| 分页有上限 | page_size有max限制注释 |
| 无内部实现泄露 | Proto不含DB表结构/内部状态 |
| 错误信息不含堆栈 | Error消息不含内部错误详情 |
