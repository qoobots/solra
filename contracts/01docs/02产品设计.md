# 协议定义层（Contracts）产品设计

> contracts/ 的产品形态：一个自描述的、可版本化的、多语言代码生成就绪的协议仓库

---

## 一、产品形态定义

contracts/ 不是一个传统意义上的"产品"，而是索拉 Monorepo 的**协议产品**——它的用户是开发者，它的界面是 `.proto` 文件和生成的代码。

### 1.1 核心产品价值

```
开发者A（avt-service）  ──引用──→  contracts/proto/avt/avt.proto
                                    ↓ buf generate
开发者B（iOS客户端）    ←──引用──  avt_pb.swift（自动生成）
```

> 一次定义，多处使用；一处变更，全仓感知。

### 1.2 协议产品的三个层次

| 层次 | 内容 | 消费者 |
|------|------|--------|
| **定义层** (.proto/.avsc/.yaml) | 数据模型 + RPC定义 + 事件Schema | 所有开发者 |
| **代码生成层** (自动生成) | Java/TS/Python/C++/Swift/Kotlin 代码 | 各子项目编译系统 |
| **文档层** (proto-doc) | 协议参考文档 + 变更日志 | 架构评审 + 新人Onboarding |

## 二、目录结构设计

```
contracts/
├── proto/                    # Protocol Buffers 定义
│   ├── common/               # 公共模型（P0）
│   │   ├── common.proto      # 基础类型：UserId/SpaceId/Timestamp
│   │   ├── pagination.proto  # 分页/排序/过滤
│   │   └── error.proto       # 统一错误码
│   ├── avt/                  # 虚拟人交互（P0）
│   ├── spc/                  # 空间消费（P0）
│   ├── auth/                 # 身份认证（P0）
│   ├── saf/                  # 内容安全（P0）
│   ├── crt/                  # 空间创作（P1）
│   ├── soc/                  # 社交在场（P1）
│   ├── grw/                  # 用户成长（P1）
│   ├── not/                  # 通知消息（P1）
│   └── mon/                  # 商业化（P2）
├── events/                   # 领域事件定义
│   └── *.avsc                # Avro schema
├── openapi/                  # OpenAPI 3.0 规范
│   ├── services/             # 服务间 API
│   └── client/               # 客户端 BFF API
├── buf.yaml                  # Buf 配置
├── buf.gen.yaml              # 代码生成配置
└── buf.lock                  # 依赖锁定
```

## 三、协议设计原则

| 原则 | 说明 | 实践 |
|------|------|------|
| **公共优先** | 可复用的类型提取到 common/ | UserId不定义在avt里 |
| **版本化字段编号** | 字段编号不可变，预留扩展区间 | reserved 1-10, 100-200 |
| **命名上下文无关** | Proto包名不含服务名 | `solra.avt.v1` 而非 `solra.avt_service.v1` |
| **枚举优于布尔** | 状态用枚举而非bool | `Status status = 1;` 而非 `bool is_active = 1;` |
| **字段必选/可选明确** | 使用 optional 标注可选字段 | proto3 optional |
| **注释即文档** | 每个字段必须有中文注释 | `// 用户唯一标识，32位UUID` |

## 四、版本管理策略

### 4.1 Proto 版本命名

```
solra.{domain}.{version}
例：solra.avt.v1 / solra.avt.v2alpha1
```

### 4.2 兼容性约定

| 变更类型 | 兼容性 | 处理 |
|---------|--------|------|
| 新增字段 | ✅ 向后兼容 | 自由添加 |
| 删除字段 | ❌ 破坏性 | 标记 reserved |
| 重命名字段 | ❌ 破坏性 | 新增+废弃旧字段 |
| 新增 RPC | ✅ 向后兼容 | 自由添加 |
| 删除 RPC | ❌ 破坏性 | 标记 deprecated |

## 五、产品演进路线

| 阶段 | 目标 | 交付 |
|------|------|------|
| M1: 公共模型 (P0) | 5个核心Proto域定义完成 | common/avt/spc/auth/saf |
| M2: 支撑域 (P1) | 4个支撑Proto域 | crt/soc/grw/not |
| M3: 事件体系 (P1) | 核心领域事件定义 | events/*.avsc |
| M4: OpenAPI (P2) | 服务间+客户端API规范 | openapi/*.yaml |
| M5: 商业化 (P2) | mon Proto域 | proto/mon/ |
| M6: 版本管理 (P3) | 事件版本兼容工具 | 事件迁移工具 |
