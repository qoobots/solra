# 协议定义层（Contracts）部署设计

> contracts/ 本身不需要部署——其部署"产品"是代码生成产物在各子项目中的引用

---

## 一、部署模型

contracts/ 作为"源代码级依赖"，以以下方式部署到各子项目：

```
contracts/ (Git仓库) 
    ↓ CI: buf generate
生成产物 → 发布到各子项目的 generated/ 目录
    ├── services/common/src/generated/java/
    ├── ai-services/shared/generated/
    ├── core/generated/
    ├── clients/ios/generated/
    ├── clients/android/generated/
    └── clients/web/src/generated/
```

---

## 二、版本发布策略

| 策略 | 说明 |
|------|------|
| **Git Submodule** | 各子项目通过 git submodule 引用 contracts/ |
| **Monorepo 直接引用** | 同仓库内直接路径引用（当前方案） |
| **Buf Schema Registry** | 通过 BSR 发布Proto包（未来方案） |

---

## 三、CI/CD 流程

```
Git Push (proto变更)
  → buf lint检查
  → buf breaking检查
  → buf generate (全语言)
  → 将生成产物提交到各子项目 generated/
  → 触发依赖子项目的构建流水线
```

---

## 四、环境管理

| 环境 | Proto版本策略 |
|------|--------------|
| dev | 直接从 main 分支引用 |
| staging | 从 release 分支引用 |
| prod | 从 tag 版本引用 |
