# 协议定义层（Contracts）数据设计

> contracts/ 的数据模型设计——Proto Message 字段定义规范与跨子项目数据流

---

## 一、核心数据模型

### 1.1 公共基础类型 (common)

```protobuf
// 基础标识
message UserId { string value = 1; }
message SpaceId { string value = 1; }
message AssetId { string value = 1; }
message SessionId { string value = 1; }

// 时间戳
message SolraTimestamp {
  int64 epoch_millis = 1; // Unix毫秒时间戳
}

// 地理位置
message GeoPoint {
  double latitude = 1;
  double longitude = 2;
}

// 分页
message PageRequest {
  int32 page = 1;
  int32 size = 2;
  string cursor = 3; // 游标分页，优先使用
}

message PageResponse {
  int32 total = 1;
  bool has_more = 2;
  string next_cursor = 3;
}

// 统一错误
message SolraError {
  ErrorCode code = 1;
  string message = 2;
  map<string, string> details = 3;
}

enum ErrorCode {
  ERROR_UNSPECIFIED = 0;
  ERROR_UNAUTHORIZED = 1;
  ERROR_NOT_FOUND = 2;
  ERROR_RATE_LIMITED = 3;
  ERROR_INTERNAL = 99;
}
```

### 1.2 数据模型间关联

```
UserId ←── 所有业务消息引用
SpaceId ←── spc/crt/soc 域消息引用
AssetId ←── crt/spc 域消息引用
SolraTimestamp ←── 所有带时间戳的消息
PageRequest/Response ←── 所有列表查询
SolraError ←── 所有RPC响应
```

## 二、数据流转

### 2.1 同步RPC数据流

```
[客户端] → gRPC Request(proto) → [服务端] → gRPC Response(proto) → [客户端]
```

### 2.2 异步事件数据流

```
[服务A] → 发布 Event(Avro schema) → [Kafka] → [服务B] 消费 → 处理
                             ↘ [服务C] 消费 → 处理
```

---

## 三、字段设计规范

| 规则 | 说明 |
|------|------|
| 字段编号从1开始，不可变 | 删除字段用reserved保留编号 |
| 预留扩展区间 | `reserved 100 to 200;` |
| 必选/可选明确 | proto3使用optional |
| 列表字段复数命名 | `repeated UserId participant_ids = 10;` |
| Map谨慎使用 | Map的key类型受限，复杂场景用repeated Message |
| 不用Any/oneof过度 | 优先用明确类型而非Any |
