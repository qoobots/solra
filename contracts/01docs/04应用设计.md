# 协议定义层（Contracts）应用架构设计

> contracts/ 在索拉整体架构中的位置——作为"语言层"贯穿所有子项目的接口契约

---

## 一、架构定位

contracts/ 是 Monorepo 的**接口契约层**，位于应用架构的"横向基础设施"位置：

```
┌─────────────────────────────────────────────────────────┐
│ clients/ (iOS / Android / Web)                          │
├─────────────────────────────────────────────────────────┤
│ core/ (C++17 Core SDK)                                  │
├─────────────────────────────────────────────────────────┤
│                  ↑ 引用 contracts/ 生成的代码              │
│  ┌───────────────────────────────────────────────┐     │
│  │ contracts/ ← 单一事实来源 (Single Source of Truth) │  │
│  │  ├── proto/   → 数据模型 + gRPC定义              │  │
│  │  ├── events/  → 领域事件 Avro schema            │  │
│  │  └── openapi/ → REST API 规范                   │  │
│  └───────────────────────────────────────────────┘     │
│                  ↓ 引用 contracts/ 生成的代码              │
├─────────────────────────────────────────────────────────┤
│ services/ (Java Spring Boot ×10)                        │
├─────────────────────────────────────────────────────────┤
│ ai-services/ (Python FastAPI ×5)                        │
├─────────────────────────────────────────────────────────┤
│ infra/ (K8s / Docker / Terraform)                       │
└─────────────────────────────────────────────────────────┘
```

---

## 二、通信模式设计

### 2.1 同步通信：gRPC (Proto定义)

```
服务A ──gRPC(protobuf)──→ 服务B
客户端 ──gRPC(protobuf)──→ API Gateway ──→ 后端服务
```

### 2.2 异步通信：事件 (Avro定义)

```
服务A ──发布事件(Kafka)──→ 服务B、服务C
proto/ 定义数据模型
events/ 定义事件Schema
```

### 2.3 REST通信：OpenAPI

```
Web客户端 ──HTTP/JSON──→ BFF服务 ──gRPC──→ 后端服务
openapi/ 定义 REST 接口规范
```

## 三、代码生成流水线

```
┌─────────┐    ┌──────────┐    ┌─────────────────────────┐
│ .proto  │───→│ buf lint │───→│ buf generate             │
│ 文件    │    │ 格式检查  │    │ 多语言代码生成             │
└─────────┘    └──────────┘    │ ├── Java → services/    │
                               │ ├── TS   → clients/web  │
                               │ ├── Python → ai-services│
                               │ ├── C++   → core/       │
                               │ ├── Swift → clients/ios │
                               │ └── Kotlin→ clients/android│
                               └─────────────────────────┘
```

---

## 四、与其他子项目的依赖关系

| 子项目 | 引用方式 | 引用的contracts |
|--------|---------|----------------|
| services/* | protobuf-java 编译产物 | proto/common + 自身领域proto |
| ai-services/* | protobuf-python 编译产物 | proto/avt + proto/saf |
| core/ | protobuf-cpp 编译产物 | proto/common + proto/spc |
| clients/ios | swift-protobuf 编译产物 | proto/spc + openapi/client |
| clients/android | protobuf-kotlin 编译产物 | proto/spc + openapi/client |
| clients/web | protobuf-ts 编译产物 | proto/spc + openapi/client |

## 五、版本管理架构

```
Proto 版本化策略:
  solra.{domain}.v1  → 稳定版本
  solra.{domain}.v2alpha1 → 实验版本
  solra.{domain}.v2 → 新稳定版本 (保证 v1→v2 有迁移路径)
```
