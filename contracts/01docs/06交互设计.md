# 协议定义层（Contracts）交互设计

> contracts/ 的"交互"面向开发者——Proto编写体验、审查流程、代码生成反馈

---

## 一、开发者交互流程

### 1.1 新增 Proto 域流程

```
开发者 → 创建 proto/{domain}/ 目录
       → 编写 .proto 文件
       → 运行 buf lint (本地)
       → 运行 buf generate (验证代码生成)
       → 提交 PR
       → CI 自动检查 lint + breaking
       → 架构师 Review
       → 合并 → 代码生成产物自动更新
```

### 1.2 修改已有 Proto 流程

```
开发者 → 修改 .proto 文件
       → 运行 buf lint (本地)
       → 运行 buf breaking (本地检测不兼容变更)
       → 提交 PR（附CHANGELOG说明）
       → CI 检查通过
       → Review + 合并
```

---

## 二、审查清单

| 检查项 | 要求 |
|--------|------|
| 字段编号不冲突 | reserved 声明已废弃编号 |
| 命名规范 | 符合 Proto 命名规范 |
| 注释完整 | 每个Message/RPC/Field有中文注释 |
| 公共类型提取 | 可复用类型不在领域proto中重复定义 |
| 枚举值有0值 | 所有enum第一个值为UNSPECIFIED=0 |
| 无破坏性变更 | buf breaking check通过 |

## 三、开发工具

| 工具 | 用途 | 安装 |

## 四、反馈循环

```
保存.proto → [30s] buf lint 本地反馈 → [2m] CI 全量检查 → [5m] 代码生成产物更新
```
