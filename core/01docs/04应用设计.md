# 核心引擎层（Core）应用架构设计

> Solra Core SDK 在索拉整体应用架构中的位置——作为客户端与云端之间的"双端共享中间件"

---

## 一、架构定位

```
┌──────────────────────────────────────────┐
│  clients/ios (SwiftUI)                    │
│  clients/android (Compose)                │
│  clients/web (React)                      │
├──────────────────────────────────────────┤
│          ↑ FFI 桥接层 ↑                    │
├──────────────────────────────────────────┤
│  core/ ← C++17 核心引擎 (双端共享)         │
│  ├── render/      3D渲染 → GPU             │
│  ├── inference/   端侧推理 → NPU            │
│  ├── streaming/   流式加载 → CDN            │
│  ├── webrtc/      P2P通信 → 信令服务器       │
│  ├── animation/   动画系统                 │
│  └── platform/    平台抽象 (iOS/Android)    │
├──────────────────────────────────────────┤
│          ↓ gRPC / REST ↓                  │
├──────────────────────────────────────────┤
│  services/ + ai-services/ (云端)          │
└──────────────────────────────────────────┘
```

---

## 二、模块间依赖

```
platform/
    ↑ 所有模块依赖
    ├── render/      ← 独立模块
    ├── inference/   ← 独立模块
    ├── streaming/   → render/ (加载完成后渲染)
    ├── webrtc/      → streaming/ (远程资产传输)
    └── animation/   → render/ (骨骼→渲染)
```

---

## 三、线程模型

```
主线程 (UI Thread)
    ↑ 回调
工作线程池 (4-8线程)
    ├── 渲染线程 (1条，GPU独占)
    ├── 推理线程 (1-2条，NPU独占)
    ├── 加载线程 (2-4条，IO密集)
    └── 网络线程 (1条，WebRTC)
```

---

## 四、生命周期

```
solra_core_init()     → 初始化CMake/GPU/推理引擎
  ├── space_enter()   → 加载空间+渲染
  ├── chat_start()    → 启动推理+WebRTC
  ├── chat_end()      → 停止推理
  └── space_leave()   → 卸载空间
solra_core_destroy()  → 释放所有资源
```
