# 核心引擎层（Core）产品设计

> Solra Core SDK——跨平台C++17核心引擎的产品形态、API设计与集成方案

---

## 一、产品形态

### 1.1 SDK定位

Solra Core SDK 是一个**嵌入式C++17静态库**，作为客户端与云端服务之间的"中间件"：

```
┌──────────────────────────────────────────┐
│  客户端 UI 层 (SwiftUI / Compose / React) │
├──────────────────────────────────────────┤
│  Solra Core SDK (C++17)                  │
│  ├── render/    3D渲染引擎               │
│  ├── inference/ 端侧推理                 │
│  ├── streaming/ 流式加载                 │
│  ├── webrtc/    P2P通信                  │
│  ├── animation/ 动画系统                 │
│  └── platform/  平台抽象层               │
├──────────────────────────────────────────┤
│  操作系统 (iOS / Android / Windows)      │
└──────────────────────────────────────────┘
```

### 1.2 交付形态

| 平台 | 产物 | 构建系统 |
|------|------|---------|
| iOS | `SolraCore.framework` (arm64) | CMake → Xcode |
| Android | `libsolracore.so` (arm64-v8a) | CMake → Gradle |
| Windows | `SolraCore.lib` (x64) | CMake → MSVC |
| macOS | `libSolraCore.a` (arm64/x64) | CMake → Xcode |

## 二、API设计原则

### 2.1 C API 边界

```
C++17 内部实现 → 暴露 C ABI 接口 → Swift/Kotlin FFI 调用
```

| 原则 | 说明 |
|------|------|
| **C ABI稳定** | 对外暴露纯C接口，确保跨语言FFI兼容 |
| **不透明指针** | Handle模式封装内部状态 |
| **错误码返回** | 不使用异常跨越FFI边界 |
| **回调线程安全** | 所有回调保证在主线程调用 |

### 2.2 核心API分类

| 模块 | 头文件 | 函数数量估算 | 说明 |
|------|--------|-------------|------|
| 初始化 | `solra_core.h` | ~5 | SDK生命周期管理 |
| 渲染 | `solra_render.h` | ~15 | 场景创建/材质/光照/相机 |
| 推理 | `solra_inference.h` | ~10 | 模型加载/推理/流式输出 |
| 加载 | `solra_streaming.h` | ~12 | 资产请求/优先级/缓存 |
| 通信 | `solra_webrtc.h` | ~8 | 连接管理/数据通道 |
| 动画 | `solra_animation.h` | ~10 | 表情/口型/骨骼 |
