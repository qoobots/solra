# 核心引擎层（Core）交互设计

> Core SDK 作为嵌入式SDK，其"交互"体现在 Native↔SDK 的FFI调用协议与回调时序

---

## 一、FFI 调用模式

```
Swift/Kotlin → C ABI → C++ 内部实现 → 异步回调 → Swift/Kotlin
```

### 1.1 同步调用

```c
// 同步返回结果，不阻塞主线程>1ms
SolraResult solra_render_get_fps(SolraHandle handle, float* fps);
```

### 1.2 异步调用（回调模式）

```c
typedef void (*solra_callback_t)(void* user_data, const SolraEvent* event);

// 注册回调 → 异步操作 → 回调触发
void solra_streaming_load_space(
    SolraHandle handle, 
    const char* space_id,
    solra_callback_t callback,
    void* user_data
);
```

---

## 二、核心交互流程

### 2.1 空间进入流程

```
用户点击空间卡片
  → [SwiftUI/Compose] 调用 solra_streaming_load_space()
  → [Core SDK] 流式加载 + 回调进度 (0-100%)
  → [Core SDK] 加载完成 → 回调 solra_render_scene_ready()
  → [SwiftUI/Compose] 切换全屏3D视图
  → [Core SDK] 开始渲染循环 30/60fps
```

### 2.2 虚拟人对话流程

```
用户输入文本
  → [Native] 调用 solra_inference_send_message()
  → [Core SDK] 推理引擎生成Token
  → [Core SDK] 逐Token回调 solra_inference_on_token()
  → [Core SDK] 合TTs → 口型同步 → 动画
  → [Native] 显示文本+播放音频+渲染表情
```

---

## 三、错误处理

```
所有C API返回 SolraResult 枚举:
  SOLRA_OK = 0
  SOLRA_ERROR_NOT_INITIALIZED
  SOLRA_ERROR_INVALID_ARGUMENT
  SOLRA_ERROR_GPU_UNAVAILABLE
  SOLRA_ERROR_NPU_UNAVAILABLE
  SOLRA_ERROR_NETWORK_ERROR
  SOLRA_ERROR_OUT_OF_MEMORY
```
