# 核心引擎层（Core）质量设计

> Core SDK 质量保障策略——测试金字塔、性能基准、崩溃监控

---

## 一、测试金字塔

```
         ┌─────┐
         │ E2E │  ~20 场景 (真机测试)
         ├─────┤
         │集成  │  ~50 用例 (模块交互)
         ├─────┤
         │ 单元  │  ~300 用例 (每个类/函数)
         └─────┘
```

### 1.1 单元测试 (Google Test)

| 模块 | 预计用例数 | 重点 |
|------|----------|------|
| render | ~60 | GPU抽象层正确性 |
| inference | ~40 | Token生成精度 |
| streaming | ~50 | 分块/缓存/断点续传 |
| webrtc | ~30 | 连接建立/断开 |
| animation | ~40 | 动画混合/过渡 |
| platform | ~50 | 平台函数正确性 |

### 1.2 集成测试

- 加载空间→渲染→截图对比
- 推理请求→Token流→回调验证
- 流式加载→断网→恢复→完整性校验

### 1.3 端到端测试

- 真机: "进入空间→对话→退出" 完整流程
- 性能: FPS≥30, 内存<256MB, 崩溃率<0.1%

---

## 二、性能基准

| 指标 | 目标 | 测试工具 |
|------|------|---------|
| 渲染帧率 | ≥30fps(中端) / ≥60fps(旗舰) | Google Benchmark |
| 推理首Token | <500ms (1.5B模型) | 自研计时 |
| 空间进入 | <2秒 (200MB空间) | Instrument |
| 内存峰值 | <256MB | Instruments/MAT |
| 包体积增量 | <15MB(iOS) / <12MB(Android) | 构建产物分析 |
