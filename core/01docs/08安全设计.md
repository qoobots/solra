# 核心引擎层（Core）安全设计

> Core SDK 端侧安全——模型保护、数据隔离、输入验证

---

## 一、端侧安全威胁模型

| 威胁 | 风险等级 | 缓解措施 |
|------|---------|---------|
| 模型文件泄露 | 高 | 模型加密存储 + 运行时解密 |
| 对话数据泄露 | 中 | 本地数据加密 |
| API Key泄露 | 高 | KeyChain/KeyStore存储 |
| 逆向工程 | 中 | 关键模块混淆+反调试 |
| 内存注入 | 低 | 地址空间随机化(ASLR默认) |

---

## 二、模型保护

```
模型下发流程:
  [云端] → 加密(GGUF → AES-256-GCM → .solram 格式)
  [CDN分发] → .solram 加密模型
  [Core SDK] → 运行时解密 → llama.cpp加载
  [内存] → 用完即擦除 (memset 0)
```

---

## 三、数据隔离

| 数据 | 存储保护 | 访问控制 |
|------|---------|---------|
| 用户Token | iOS KeyChain / Android KeyStore | 仅Core SDK可读 |
| 对话历史 | AES加密文件 | 沙盒内 |
| API密钥 | 硬件安全模块 | 运行时获取 |
| 敏感日志 | 禁止记录 | linter规则阻断 |

---

## 四、输入验证

```cpp
// 所有外部输入必须校验
- 空间ID: UUID v4 格式校验
- 用户输入: 长度<4096, UTF-8 合法性
- 资产URL: HTTPS only, 白名单域名
- 模型文件: 签名校验 (SHA-256)
```
