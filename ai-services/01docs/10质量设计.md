# AI服务层（AI-Services）质量设计

> ai-services/ 质量保障——模型精度、推理性能、服务稳定性

---

## 一、测试策略

| 层级 | 工具 | 说明 |
|------|------|------|
| 单元测试 | pytest | 模型加载/预处理/后处理 |
| 集成测试 | pytest-asyncio | API端点+模型集成 |
| 精度测试 | 自研 | 标准数据集推理精度 |
| 性能测试 | locust | 并发推理吞吐/延迟 |

---

## 二、模型质量基准

| 指标 | 目标 |
|------|------|
| 文本安全分类准确率 | >95% |
| TTS自然度 MOS | >4.0 |
| 嵌入检索 Recall@10 | >90% |
| 推荐 CTR 提升 | >30% vs 基线 |

---

## 三、性能基准

| 指标 | 目标 |
|------|------|
| LLM推理 TTFT (7B) | <500ms |
| LLM吞吐 (7B) | >50 tokens/s |
| 安全分类延迟 | <200ms |
| TTS合成实时率 | >1.0 (实时) |
| GPU利用率 | >70% |

---

## 四、监控告警

| 告警 | 条件 |
|------|------|
| P0-推理延迟过高 | P99 > 3s 持续5分钟 |
| P0-错误率飙升 | 错误率 > 5% |
| P1-GPU显存不足 | 可用显存 < 10% |
| P1-队列堆积 | 队列深度 > 100 |
