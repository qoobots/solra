# 嵌入模型与语音合成模型选型方案

> 关联：ai-services/ 子项目 P1 模型选型
> 版本：v1.0 | 日期：2026-06-29

## 一、嵌入模型选型 (Embedding)

### 1.1 需求分析

| 维度 | 要求 |
|------|------|
| 语言 | 中英双语（可扩展多语言） |
| 维度 | 768-1024d |
| 推理延迟 | <100ms (GPU) / <500ms (CPU) |
| 精度 | STS-B ≥ 0.78 |
| 最大输入 | ≥512 tokens |
| 推理框架 | sentence-transformers / ONNX Runtime |

### 1.2 候选模型对比

| 模型 | 参数量 | 维度 | STS-B(中文) | 速度(token/s) | 内存 | 许可证 |
|------|--------|------|-------------|--------------|------|--------|
| **text2vec-large-chinese** ⭐ | 326M | 1024 | **0.843** | 120 | 1.3GB | Apache 2.0 |
| BGE-M3 (BAAI) | 568M | 1024 | 0.828 | 80 | 2.2GB | MIT |
| multilingual-e5-large-instruct | 560M | 1024 | 0.812 | 75 | 2.1GB | MIT |
| GTE-Qwen2-7B-instruct | 7B | 3584 | 0.861 | 15 | 14GB | Apache 2.0 |
| jina-embeddings-v3 | 572M | 1024 | 0.825 | 90 | 2.1GB | Custom |

### 1.3 选型结论

**首选：text2vec-large-chinese**
- 中文 STS-B 最高 (0.843)，兼顾英文
- Apache 2.0 商业友好
- CoSENT 训练目标，语义区分能力强
- 1.3GB 显存友好，支持 CPU 推理

**备选：BGE-M3**（如需多语言/多粒度检索）

### 1.4 多模态嵌入增强

| 模型 | 用途 | 说明 |
|------|------|------|
| **nomic-embed-vision-v1.5** | 图片嵌入 | 与 nomic-embed-text 对齐向量空间 |
| **Chinese-CLIP (ViT-B/16)** | 中文图文对齐 | 中文场景更好 |

多模态策略：
```
文本 → text2vec-large-chinese (1024d)
图片 → nomic-embed-vision-v1.5 (768d) → 投影到1024d
→ 统一向量空间 → Milvus 混合索引
```

## 二、语音合成模型选型 (TTS)

### 2.1 需求分析

| 维度 | 要求 |
|------|------|
| 语言 | 中文为主，英文为辅 |
| 音色 | 自然度 ≥ 4.0 MOS |
| 实时率 (RTF) | ≤ 0.3 (GPU) |
| 流式输出 | 支持分句/分块流式 |
| 发音人 | ≥3 种预设音色，支持少量样本复刻 |
| 推理框架 | PyTorch / ONNX |

### 2.2 候选模型对比

| 模型 | 参数量 | MOS | RTF(GPU) | 流式 | 音色定制 | 许可证 |
|------|--------|-----|----------|------|---------|--------|
| **CosyVoice 2** ⭐ | ~300M | **4.3** | 0.15 | ✅ 流式chunk | ✅ 3s复刻 | Apache 2.0 |
| ChatTTS | ~330M | 4.1 | 0.20 | ❌ | ❌ | CC BY-NC 4.0 |
| XTTS-v2 (Coqui) | 450M | 4.0 | 0.25 | ✅ 流式 | ✅ 6s复刻 | CPML |
| GPT-SoVITS | ~400M | 4.2 | 0.18 | ❌ | ✅ 5s复刻 | MIT |
| Fish Speech 1.5 | ~350M | 4.1 | 0.16 | ✅ 流式 | ✅ 10s复刻 | Apache 2.0 |

### 2.3 选型结论

**首选：CosyVoice 2**
- 中文 MOS 最高 (4.3)
- Apache 2.0 商业友好
- 原生流式 chunk 输出，端到端延迟低
- 3 秒样本即可复刻音色（zero-shot）
- 阿里达摩院持续维护，社区活跃

**备选：Fish Speech 1.5**（如需风格多样性）

### 2.4 预设音色规划

| 音色ID | 名称 | 描述 | 场景 |
|--------|------|------|------|
| voice-001 | 知心 | 温柔女声 25-30岁 | 虚拟人默认音色 |
| voice-002 | 明朗 | 活力男声 25-30岁 | 探索/引导场景 |
| voice-003 | 沉稳 | 磁性男声 35-40岁 | 深度解说/冥想 |
| voice-004 | 灵动 | 少女音 18-22岁 | 互动/社交场景 |

## 三、模型注册与版本管理 (MLflow)

### 3.1 MLflow 组件

```
infra/mlflow/
├── docker-compose.yml        # MLflow Server + MinIO + PostgreSQL
├── config/
│   └── mlflow.env           # 环境变量
└── README.md
```

### 3.2 模型生命周期

```
注册 → 暂存(Staging) → 生产(Production) → 归档(Archived)
         ↓ 评测           ↓ A/B测试          ↓ 保留30天
```

### 3.3 版本命名规范

```
<model-name>/<version>  e.g.  cosyvoice-2/v1.0.0
版本号：<major>.<minor>.<patch>
  major: 模型架构变更
  minor: 训练数据/微调变更
  patch: 量化/推理优化
```
