# AI服务层（AI-Services）需求分析

> 索拉 Python FastAPI AI推理服务群——LLM路由、内容安全、推荐、嵌入、TTS五大AI能力

---

## 一、需求背景

索拉的核心差异化依赖AI能力——虚拟人对话、内容安全审核、个性化推荐、语音合成。ai-services/ 是 Monorepo 的AI推理层。

### 1.1 AI服务全景

| 服务 | 优先级 | 职责 |
|------|--------|------|
| llm-router | P0 | LLM推理路由与调度 |
| safety-model-service | P0 | 内容安全模型推理 |
| embedding-service | P1 | 文本/多模态嵌入 |
| recommendation-pipeline | P1 | 推荐算法流水线 |
| tts-service | P1 | 语音合成 |

---

## 二、功能需求

### 2.1 llm-router [P0]

| 编号 | 功能 | 说明 |
|------|------|------|
| AI-LLM-01 | 多模型路由 | 按场景选择端侧SLM/云端LLM |
| AI-LLM-02 | 推理请求排队 | 优先级调度+并发控制 |
| AI-LLM-03 | 流式输出 | SSE 方式推送 Token |
| AI-LLM-04 | Fallback策略 | LLM不可用时降级方案 |
| AI-LLM-05 | 推理指标 | 延迟/吞吐/Token成本 |

### 2.2 safety-model-service [P0]

| 编号 | 功能 | 说明 |
|------|------|------|
| AI-SAF-01 | 文本安全分类 | 色情/暴力/政治/广告 |
| AI-SAF-02 | 图片安全检测 | 多模态安全模型 |
| AI-SAF-03 | 批量审核 | 高吞吐异步审核 |

### 2.3 embedding-service [P1]

| 编号 | 功能 | 说明 |
|------|------|------|
| AI-EMB-01 | 文本嵌入 | 内容向量化 |
| AI-EMB-02 | 多模态嵌入 | 文本+图片联合嵌入 |

### 2.4 recommendation-pipeline [P1]

| 编号 | 功能 | 说明 |
|------|------|------|
| AI-REC-01 | 协同过滤 | 基于用户行为的推荐 |
| AI-REC-02 | 内容推荐 | 基于嵌入相似度 |
| AI-REC-03 | 冷启动策略 | 新用户/新内容推荐 |

### 2.5 tts-service [P1]

| 编号 | 功能 | 说明 |
|------|------|------|
| AI-TTS-01 | 中文语音合成 | 自然度MOS>4.0 |
| AI-TTS-02 | 情感语音 | 情感标签控制语气 |
| AI-TTS-03 | 流式合成 | 边生成边播放 |

---

## 三、非功能需求

| 维度 | 要求 |
|------|------|
| **推理延迟** | LLM P99<2000ms, Safety P99<200ms |
| **并发** | 单模型 50+ 并发推理 |
| **GPU利用率** | >70% |
| **可用性** | 核心服务 99.9% |
| **模型热更新** | 不中断服务切换模型版本 |
