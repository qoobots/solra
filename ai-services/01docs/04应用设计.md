# AI服务层（AI-Services）应用架构设计

> ai-services/ 在索拉架构中的位置——AI推理能力层

---

## 一、架构定位

```
services/ (Java)
    ↓ gRPC/HTTP调用
ai-services/ (Python FastAPI)
    ├── llm-router ← avt-service (虚拟人对话编排需要)
    ├── safety-model ← saf-service (内容审核需要)
    ├── embedding ← spc-service + recommendation-pipeline
    ├── recommendation ← spc-service (推荐分发需要)
    └── tts ← avt-service (语音合成需要)
    ↓ 模型推理
GPU Cluster (CUDA)
```

---

## 二、推理流水线

```
avt-service 请求
  → llm-router 路由决策
    ├── 简单对话 → 端侧SLM (Core SDK本地推理)
    └── 复杂对话 → 云端LLM (vLLM推理)
      → 流式Token → TTS → 音频流 → 客户端
```

---

## 三、服务间通信

| 调用方 | 被调用方 | 协议 |
|--------|---------|------|
| avt-service (Java) | llm-router / tts | gRPC/HTTP |
| saf-service (Java) | safety-model-service | gRPC/HTTP |
| spc-service (Java) | embedding / recommendation | gRPC/HTTP |

---

## 四、模型生命周期

```
模型选型 → MLflow 注册
  → 性能基准测试
  → GPU 集群部署
  → 灰度发布 (5%→50%→100%)
  → 监控 (延迟/错误率/Token成本)
  → 退役 (旧版本7天无流量自动下线)
```
