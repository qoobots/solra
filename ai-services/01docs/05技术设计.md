# AI服务层（AI-Services）技术设计

> ai-services/ 技术选型与工程规范——Python 3.11+ / FastAPI / Poetry

---

## 一、技术选型

| 维度 | 选择 |
|------|------|
| 语言 | Python 3.11+ |
| 框架 | FastAPI + Uvicorn |
| 模型推理 | vLLM (云端) / llama.cpp-python (端侧) |
| 深度学习 | PyTorch 2.x / ONNX Runtime |
| 验证 | Pydantic v2 |
| 异步 | asyncio + httpx(aio) |
| 包管理 | Poetry |
| 测试 | pytest + pytest-asyncio |
| 观测 | OpenTelemetry + Prometheus |
| GPU | CUDA 12.x |

---

## 二、工程结构

```
ai-services/
├── pyproject.toml              # Poetry 根配置
├── shared/                     # 共享基础设施
│   ├── middleware/             # 公共中间件
│   ├── scheduler/              # GPU调度器
│   ├── registry/               # 模型注册(MLflow)
│   └── metrics/                # Prometheus指标
├── llm-router/
│   ├── pyproject.toml
│   ├── api/                    # FastAPI 路由
│   ├── router/                 # 路由逻辑
│   └── models/                 # 模型适配器
├── safety-model-service/ ...
├── embedding-service/ ...
├── recommendation-pipeline/ ...
├── tts-service/ ...
└── models/                     # 模型文件(存储)
```
