# AI服务层（AI-Services）产品设计

> ai-services/ 的产品形态——5个 Python FastAPI 推理微服务 + 共享基础设施

---

## 一、产品形态

每个AI服务遵循统一模板：

```
{service}/
├── api/              # FastAPI 路由
├── models/           # 模型加载与推理
├── schemas/          # Pydantic 数据模型
├── config/           # 配置管理
└── tests/            # 测试
```

---

## 二、统一技术栈

| 维度 | 选择 |
|------|------|
| 语言 | Python 3.11+ |
| 框架 | FastAPI |
| 模型服务 | vLLM / llama.cpp-python / ONNX Runtime |
| 验证 | Pydantic v2 |
| 异步 | asyncio + httpx |
| GPU | CUDA 12.x / PyTorch 2.x |
| 观测 | OpenTelemetry + Prometheus |
| 构建 | Poetry |

---

## 三、共享基础设施 shared/

| 模块 | 说明 |
|------|------|
| 公共中间件 | 日志/限流/鉴权 |
| GPU资源调度器 | CUDA显存分配与管理 |
| 模型注册管理 | MLflow集成/版本管理 |
| 推理指标采集 | Prometheus metrics |

---

## 四、模型管理 models/

| 模型 | 优先级 | 说明 |
|------|--------|------|
| 端侧SLM (1-3B) | P0 | Qwen2.5-1.5B / Phi-3-mini |
| 云端LLM (7-13B) | P0 | Qwen2.5-7B / DeepSeek-V2-Lite |
| 多模态安全 | P0 | 自研+开源安全分类器 |
| 嵌入模型 | P1 | text2vec / multilingual-e5 |
| 语音合成 | P1 | VITS / ChatTTS / CosyVoice |
