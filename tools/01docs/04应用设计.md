# 开发工具链（Tools）应用架构设计

> tools/ 在索拉架构中的位置——开发者效率层，DevOps支撑

---

## 一、架构定位

```
tools/ ← 独立于业务服务，面向开发流程
  ├── dev-env/    → 开发环境管理
  ├── codegen/    → 协议→代码转换
  ├── load-test/  → 性能验证
  └── release/    → 版本发布管理
```

---

## 二、与其他组件的关联

```
contracts/proto/
    ↓ tools/codegen/ →
services/generated/
ai-services/generated/
core/generated/
clients/*/generated/

services/* / ai-services/*
    ↓ tools/load-test/ →
性能报告

services/* / ai-services/*
    ↓ tools/release/ →
Docker Registry / Helm Repo
```

---

## 三、工具集成

```
Makefile (顶层入口)
  ├── make dev-setup  → tools/dev-env/
  ├── make proto-gen  → tools/codegen/
  ├── make load-test  → tools/load-test/
  └── make release    → tools/release/
```
