# 开发工具链（Tools）部署设计

> tools/ 的"部署"——工具在开发者本机运行，或集成到CI/CD流水线

---

## 一、本地运行模式

```
# 克隆仓库后，首次运行
make dev-setup
  → 安装所有依赖
  → 初始化数据库
  → 配置环境变量

# 日常开发
make proto-gen    # Proto变更后
make load-test    # 上线前验证
make release      # 发布时
```

---

## 二、CI/CD 集成

```yaml
# .github/workflows/release.yml
jobs:
  release:
    steps:
      - uses: actions/checkout@v4
      - name: Create Release
        run: make release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

---

## 三、工具分发

```
开发者本地: git clone + make dev-setup
CI Runner: GitHub Actions 预装依赖
新成员: README.md → 3步启动
```
