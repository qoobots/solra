# 微服务层（Services）技术设计

> services/ 技术选型与工程规范——Java 17 + Spring Boot 3 + Gradle 多模块构建

---

## 一、技术选型

| 维度 | 选择 | 版本 |
|------|------|------|
| 语言 | Java | 17 LTS |
| 框架 | Spring Boot | 3.3.x |
| RPC | gRPC (grpc-java) | 1.60+ |
| ORM | Spring Data JPA + Hibernate | 6.x |
| 数据库 | PostgreSQL | 16 |
| 缓存 | Redis (Lettuce) | 7.x |
| 消息 | Apache Kafka | 3.6+ |
| 搜索 | Elasticsearch | 8.x |
| 构建 | Gradle (Kotlin DSL) | 8.x |
| 测试 | JUnit 5 + Mockito + Testcontainers | |
| 观测 | Micrometer + OpenTelemetry | |

## 三、标准依赖

```kotlin
// 每个服务的 build.gradle.kts 标准依赖
dependencies {
    implementation(project(":services:common"))
    implementation("org.springframework.boot:spring-boot-starter")
    implementation("net.devh:grpc-server-spring-boot-starter")
    implementation("org.springframework.boot:spring-boot-starter-data-jpa")
    implementation("org.springframework.kafka:spring-kafka")
    runtimeOnly("org.postgresql:postgresql")
    testImplementation("org.springframework.boot:spring-boot-starter-test")
    testImplementation("org.testcontainers:postgresql")
}
```

## 四、编码规范

| 规则 | 说明 |
|------|------|
| 包命名 | `solra.{service}.{layer}` |
| 类命名 | 后缀: Service/Controller/Repository/Entity |
| DTO | Record 类型 (Java 17+) |
| 异常 | 统一 `SolraBusinessException` + 错误码枚举 |
| 日志 | SLF4J + 结构化日志 (JSON格式) |
