# 微服务层（Services）产品设计

> services/ 的产品形态——10个高内聚低耦合的 Spring Boot 微服务 + 1个共享库

---

## 一、产品形态

每个微服务遵循统一的分层架构模板：

```
{service}/
├── api/          # 接口层 (gRPC/REST Controller)
├── domain/       # 领域层 (Entity/ValueObject/Repository)
├── infrastructure/  # 基础设施层 (DB/Message/External)
└── application/  # 应用层 (Service/DTO/Assembler)
```

---

## 二、统一技术栈

| 维度 | 选择 | 版本 |
|------|------|------|
| 语言 | Java | 17 LTS |
| 框架 | Spring Boot | 3.3+ |
| RPC | gRPC (Spring gRPC) | 1.60+ |
| REST | Spring Web MVC | 6.x |
| ORM | Spring Data JPA / MyBatis | 按需 |
| 消息 | Spring Kafka | 3.x |
| 缓存 | Spring Cache + Redis | 7.x |
| 数据库 | PostgreSQL (主) / MongoDB (文档类) | 16+ |
| 构建 | Gradle Kotlin DSL | 8.x |

## 四、公共库 common

```
services/common/
├── api/              # 公共 DTO/Response 包装
├── exception/        # 统一异常体系
├── util/             # 工具类
├── security/         # Spring Security 公共配置
└── observability/    # 链路追踪/指标公共配置
```
