# 微服务层（Services）工程目录结构

> services/ 完整目录树与各服务职责

---

## 一、目录树

```
services/
├── 01docs/                          # 本文档目录
│   ├── 01需求分析.md
│   ├── 02产品设计.md
│   ├── 03功能清单.md
│   ├── 04应用设计.md
│   ├── 05技术设计.md
│   ├── 06交互设计.md
│   ├── 07数据设计.md
│   ├── 08安全设计.md
│   ├── 09部署设计.md
│   ├── 10质量设计.md
│   ├── 11API设计.md
│   ├── 12工程目录.md                # 本文
│   └── 13开发进度.md
├── build.gradle.kts                 # 根构建脚本
├── settings.gradle.kts              # 模块注册
├── common/                          # 共享库 [P0]
│   ├── build.gradle.kts
│   └── src/
│       ├── main/java/solra/common/
│       │   ├── api/                 # 公共DTO
│       │   ├── exception/           # 统一异常
│       │   ├── security/            # 安全配置
│       │   ├── util/                # 工具类
│       │   └── observability/       # 可观测性
│       └── test/
├── auth-service/                    # 身份认证 [P0]
│   ├── build.gradle.kts
│   └── src/
│       ├── main/java/solra/auth/
│       │   ├── api/grpc/            # gRPC实现
│       │   ├── api/rest/            # REST接口
│       │   ├── application/         # 应用服务
│       │   ├── domain/              # 领域模型
│       │   └── infrastructure/      # 基础设施
│       └── test/
├── saf-service/                     # 内容安全 [P0]
├── avt-service/                     # 虚拟人交互 [P0]
├── spc-service/                     # 空间消费 [P0]
├── soc-service/                     # 社交在场 [P1]
├── grw-service/                     # 用户成长 [P1]
├── not-service/                     # 通知消息 [P1]
├── crt-service/                     # 空间创作 [P1]
└── mon-service/                     # 商业化 [P2]
```

---

## 二、各服务职责

| 服务 | 优先级 | 域类型 | 核心职责 |
|------|--------|--------|---------|
| common | P0 | 共享 | DTO/异常/安全/工具 |
| auth-service | P0 | 通用 | 注册/登录/OAuth/Token/RBAC |
| saf-service | P0 | 通用 | 文本审核/图片审核/举报/申诉 |
| avt-service | P0 | 核心 | 对话/记忆/情感/个性 |
| spc-service | P0 | 核心 | 空间元数据/推荐/搜索/CDN |
| soc-service | P1 | 支撑 | 会话/在场/手势同步 |
| grw-service | P1 | 支撑 | 档案/成就/信仰等级 |
| not-service | P1 | 支撑 | 站内通知/推送/收件箱 |
| crt-service | P1 | 支撑 | 项目管理/资产库/AI搭建 |
| mon-service | P2 | 支撑 | 订阅/虚拟物品/订单/支付 |
