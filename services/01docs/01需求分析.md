# 微服务层（Services）需求分析

> 索拉 Java Spring Boot 微服务群——10个微服务承载核心域+支撑域+通用域的完整业务能力

---

## 一、需求背景

索拉采用领域驱动设计（DDD）限界上下文拆分微服务。services/ 是 Monorepo 的业务逻辑层，承载10个独立微服务。

### 1.1 微服务全景

| 服务 | 域类型 | 优先级 | 职责 |
|------|--------|--------|------|
| auth-service | 通用域 | P0 | 用户认证/授权/Token管理 |
| saf-service | 通用域 | P0 | 内容安全审核/风控 |
| avt-service | 核心域 | P0 | 虚拟人对话/记忆/情感 |
| spc-service | 核心域 | P0 | 空间内容分发/推荐 |
| common | 共享库 | P0 | DTO/工具类/异常定义 |
| soc-service | 支撑域 | P1 | 社交在场/多人同步 |
| grw-service | 支撑域 | P1 | 用户成长体系/成就 |
| not-service | 支撑域 | P1 | 通知/推送/收件箱 |
| crt-service | 支撑域 | P1 | 空间创作/AI辅助搭建 |
| mon-service | 支撑域 | P2 | 商业化/订阅/支付 |

---

## 二、功能需求概述

### 2.1 核心域服务

**avt-service（虚拟人交互）**:
- 虚拟人实时对话编排
- 对话历史管理
- 虚拟人记忆系统
- 情感状态管理
- 个性化训练数据管理

**spc-service（空间消费）**:
- 空间元数据管理
- 推荐算法编排
- 空间搜索与分类
- 预览卡片生成
- CDN分发编排

### 2.2 支撑域服务

**soc-service / grw-service / not-service / crt-service / mon-service** （详情见各服务文档）

### 2.3 通用域服务

**auth-service**: 注册/登录/OAuth/Token刷新/权限校验
**saf-service**: 文本审核/图片审核/用户举报/申诉
**common**: 共享DTO/异常/工具类

---

## 三、非功能需求

| 维度 | 要求 |
|------|------|
| **可用性** | 核心服务 99.9%，支撑服务 99.5% |
| **响应时间** | P99 < 200ms (核心域), < 500ms (支撑域) |
| **并发** | 单服务 1000 QPS 起步 |
| **数据一致性** | 最终一致性 (事件驱动) |
| **可观测性** | 全链路追踪 + 指标 + 日志 |
