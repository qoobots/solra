# 微服务层（Services）质量设计

> services/ 质量保障——测试策略、代码质量、性能基准、故障演练

---

## 一、测试策略

```
         ┌─────┐
         │ E2E │  ~30 业务场景 (全链路)
         ├─────┤
         │集成  │  ~100 用例 (服务间+DB+Kafka)
         ├─────┤
         │ 单元  │  ~500+ 用例 (覆盖率>80%)
         └─────┘
```

### 1.1 单元测试

- JUnit 5 + Mockito
- 核心业务逻辑覆盖率 > 80%
- PR 时自动运行

### 1.2 集成测试

- Testcontainers (PostgreSQL/Redis/Kafka)
- 每个 gRPC/REST 端点至少1个集成测试
- 事件发布/订阅验证

### 1.3 契约测试

- Proto 协议兼容性验证
- gRPC 接口契约测试

## 三、性能基准

| 服务类型 | QPS目标 | P99延迟 | 并发连接 |
|---------|---------|---------|---------|
| 核心域服务 | 2000+ | <200ms | 500 |
| 支撑域服务 | 1000+ | <500ms | 200 |
| 通用域服务 | 5000+ | <100ms | 1000 |
