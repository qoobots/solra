# 微服务层（Services）部署设计

> services/ 部署方案——Docker + Kubernetes + Helm + ArgoCD GitOps

---

## 一、容器化

```dockerfile
# 标准 Dockerfile
FROM eclipse-temurin:17-jre-alpine
COPY build/libs/*.jar app.jar
EXPOSE 8100
ENTRYPOINT ["java", "-jar", "/app.jar"]
```

## 二、Kubernetes 部署

### 2.1 资源配置

| 服务 | CPU Request | CPU Limit | Memory Request | Memory Limit | 副本数(min) |
|------|------------|-----------|---------------|-------------|------------|
| auth-service | 250m | 500m | 256Mi | 512Mi | 2 |
| saf-service | 250m | 500m | 256Mi | 512Mi | 2 |
| avt-service | 500m | 1000m | 512Mi | 1Gi | 3 |
| spc-service | 500m | 1000m | 512Mi | 1Gi | 3 |
| 支撑域服务 | 250m | 500m | 256Mi | 512Mi | 2 |

### 2.2 部署策略

```yaml
strategy:
  type: RollingUpdate
  rollingUpdate:
    maxUnavailable: 0    # 零停机
    maxSurge: 1
```

## 三、GitOps 流程

```
Git Push (main)
  → CI: 构建+测试+镜像推送
  → ArgoCD: 检测image tag变更
  → ArgoCD: 自动同步到 K8s 集群
  → 健康检查: Readiness/Liveness Probe
  → 自动回滚: 健康检查失败 → 恢复上一版本
```
