# 微服务层（Services）应用架构设计

> services/ 在索拉架构中的位置——业务逻辑核心层，DDD+事件驱动+CQRS

---

## 一、架构定位

```
clients/ ─→ API Gateway (APISIX) ─→ services/
                                       ├── auth-service (认证)
                                       ├── saf-service  (安全)
                                       ├── avt-service  (虚拟人) ← core域
                                       ├── spc-service  (空间)   ← core域
                                       ├── soc-service  (社交)
                                       ├── grw-service  (成长)
                                       ├── not-service  (通知)
                                       ├── crt-service  (创作)
                                       └── mon-service  (商业化)
                                            ↓
                                    ai-services/ (AI推理)
```

---

## 二、服务内部分层

```
┌─────────────────────────────────┐
│  api/        接口层              │  ← gRPC实现 / REST Controller
│  ├── grpc/   gRPC ServiceImpl   │
│  └── rest/   REST Controller    │
├─────────────────────────────────┤
│  application/  应用层            │  ← Service / DTO / Assembler
├─────────────────────────────────┤
│  domain/       领域层            │  ← Entity / ValueObject / Repository接口
├─────────────────────────────────┤
│  infrastructure/ 基础设施层       │  ← JPA/MyBatis/Redis/Kafka/外部服务
└─────────────────────────────────┘
```

---

## 三、通信模式

### 同步通信 (gRPC)

```
auth-service ←──gRPC── 所有服务 (鉴权)
avt-service  ←──gRPC── spc-service (虚拟人关联)
```

### 异步通信 (Kafka + Avro)

```
avt-service ──发布──→ AvatarSpokeEvent
  → soc-service 消费 (更新在场状态)
  → grw-service 消费 (更新互动计数)
  → not-service 消费 (生成通知)
```

---

## 四、数据一致性策略

| 场景 | 策略 |
|------|------|
| 跨服务数据关联 | 最终一致性 + 补偿 |
| 核心业务事务 | Saga 模式 |
| 读模型同步 | CQRS + 事件驱动 |
| 缓存一致性 | Cache-Aside + TTL |
