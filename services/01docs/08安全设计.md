# 微服务层（Services）安全设计

> services/ 安全架构——认证授权、API安全、数据保护、内容安全

---

## 一、认证授权架构

```
请求 → APISIX (API网关)
        ├── 认证插件: JWT验证
        ├── 授权插件: RBAC检查
        └── 限流插件: Rate Limiting
              ↓
        services/* (后端服务)
        └── @PreAuthorize 方法级权限控制
```

---

## 二、安全层级

| 层级 | 机制 | 说明 |
|------|------|------|
| 传输层 | TLS 1.3 | 所有外部通信加密 |
| 网关层 | JWT + RBAC + Rate Limit | API网关统一管控 |
| 服务层 | Spring Security + Method Security | 服务级权限 |
| 数据层 | 敏感字段加密 + 审计日志 | 数据保护 |
| 内容层 | saf-service 统一审核 | 内容安全 |

---

## 三、认证流程

```
1. 用户登录 → auth-service 颁发 JWT (access_token + refresh_token)
2. 请求携带 access_token → APISIX 验证
3. Token过期 → 客户端使用 refresh_token 刷新
4. 服务间调用 → gRPC 双向TLS + Token传播
```

---

## 四、安全审查清单

| 检查项 | 要求 |
|--------|------|
| 无硬编码密钥 | 全部通过环境变量/Secret管理 |
| SQL注入防护 | JPA参数绑定 |
| XSS防护 | 输出编码 |
| CSRF防护 | REST API无状态+Token |
| 敏感日志 | 不记录密码/Token/身份证 |
| 依赖安全 | Dependabot + Snyk 扫描 |
