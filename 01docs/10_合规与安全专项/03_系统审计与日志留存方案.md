# 03_系统审计与日志留存方案

> **文档编号**：10-03 | **版本号**：v1.0 | **编制人**：安全组 | **状态**：已发布 | **更新日期**：2026-06-30

---

## 1. 审计范围

| 审计类型 | 内容 | 保留期 |
|----------|------|--------|
| 用户操作审计 | 登录/登出/关键操作 | 180天 |
| 管理员操作审计 | 所有后台操作 | 1年 |
| API调用审计 | 所有API请求日志 | 90天 |
| 数据变更审计 | 关键表变更记录 | 1年 |
| 安全事件审计 | 异常登录/攻击/告警 | 1年 |

## 2. 日志采集架构

```
应用日志 → Fluent Bit → Kafka → Loki (热存储90天) → MinIO (冷存储1年)
```

## 3. 审计日志格式

```json
{
  "timestamp": "2026-06-30T12:00:00Z",
  "userId": "uuid",
  "action": "USER_LOGIN",
  "resource": "auth",
  "result": "SUCCESS",
  "ip": "192.168.1.1",
  "userAgent": "...",
  "requestId": "uuid"
}
```

## 4. 日志安全

- 日志传输TLS加密
- 日志存储加密
- 日志防篡改（只追加+完整性校验）
- 访问权限控制（最小权限）
