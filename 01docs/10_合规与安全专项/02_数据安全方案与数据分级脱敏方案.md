# 02_数据安全方案与数据分级脱敏方案

> **文档编号**：10-02 | **版本号**：v1.0 | **编制人**：安全组 | **状态**：已发布 | **更新日期**：2026-06-30

---

## 1. 数据分级

| 级别 | 定义 | 数据示例 | 保护措施 |
|------|------|----------|----------|
| L3 绝密 | 泄露造成严重损害 | 支付密钥/根证书/JWT密钥 | Vault+双人控制+审计 |
| L2 机密 | 泄露造成较大损害 | 用户手机号/邮箱/密码/身份证/聊天记录 | AES-256加密+访问控制+脱敏展示 |
| L1 内部 | 泄露造成一般损害 | 业务运营数据/用户昵称/空间信息 | 访问控制+审计日志 |
| L0 公开 | 可公开 | 空间公开信息/用户公开资料 | 无特殊保护 |

## 2. 数据脱敏规则

| 数据类型 | 脱敏规则 | 示例 |
|----------|----------|------|
| 手机号 | 保留前3后4 | 138****1234 |
| 邮箱 | 保留首字母+域名 | a***@example.com |
| 身份证 | 保留前6后4 | 320102********1234 |
| 真实姓名 | 保留姓氏 | 张** |
| 银行卡 | 保留后4位 | ****1234 |
| IP地址 | 保留前2段 | 192.168.*.* |

## 3. 数据生命周期

```
采集 → 存储(加密) → 使用(脱敏) → 归档(压缩加密) → 销毁(安全擦除)
```

## 4. 合规要求

- 《个人信息保护法》：最小必要原则，用户可删除/导出
- 《数据安全法》：分类分级，风险评估
- GDPR (如需国际用户)：数据主体权利，跨境传输合规
