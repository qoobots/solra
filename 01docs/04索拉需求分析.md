# 索拉（Solra）需求分析文档

> 基于"存在主义社交"下一代3D空间社交产品的完整需求规格，涵盖功能需求、非功能需求、用户场景与三阶段分步交付策略

---

## 一、产品愿景与需求范围

### 1.1 产品一句话定义

> **索拉是集人工智能、3D实时交互、算法分发能力于一身的"下一代抖音"。**

### 1.2 需求范围的边界定义

| 维度 | 包含 | 不包含（MVP阶段） | 不包含（长期） |
|------|------|------------------|---------------|
| **内容形态** | 3D可交互空间 + AI虚拟人 | VR/AR硬件适配 | 全脑机接口 |
| **社交模式** | 异步（进出空间）+ 同步（多人同空间） | 大型多人实时活动（>50人同空间） | 全域持久化元宇宙 |
| **AI能力** | 虚拟人对话、情感计算、行为生成 | 虚拟人自主创作空间 | 通用AGI |
| **平台** | iOS + Android 移动端为主 | Web端、PC端 | 车载/IoT |
| **创作者工具** | AI辅助空间搭建 + 虚拟人训练 | 专业级3D建模工具 | 全自动生成 |
| **商业化** | 虚拟物品内购 + 订阅制（产品验证期启动） | 品牌空间 + API平台（规模验证期） | 金融/支付牌照业务 |

## 二、用户角色与核心场景

### 2.1 用户角色定义

| 角色 | 代号 | 描述 | 核心诉求 | 对应信仰漏斗层级 |
|------|------|------|---------|----------------|
| **游客** | Visitor | 首次进入索拉，未注册或注册未深度使用 | "这到底是什么东西？" | 游客 |
| **刷空间者** | Swiper | 以消费空间内容为主，类似抖音的"刷"行为 | "下一个空间是什么？" | 使用者 |
| **空间主** | SpaceOwner | 搭建并经营自己的空间，训练虚拟人 | "我的空间有人来吗？" | 常客 |
| **社交活跃者** | Socializer | 在空间中与他人/虚拟人深度互动 | "TA怎么回应我？" | 常客→信徒 |
| **创作者** | Creator | 设计空间模板/虚拟人模板并交易 | "我的作品能变现吗？" | 信徒 |
| **布道者** | Evangelist | 自发传播索拉信仰，招募新用户 | "你还没用索拉？" | 布道者 |
| **品牌方** | Brand | 在索拉构建品牌空间 | "用户在我的空间里感受到什么？" | B端客户 |
| **系统管理员** | Admin | 平台运营、内容审核、社区治理 | "平台安全可控吗？" | — |

### 2.2 核心用户场景（User Journey）

#### 场景一：游客的"首次被看见"之旅（激活路径）

```
1. 朋友分享一条索拉3D空间卡片 → 点击进入
2. 空间在2秒内流式加载完成
3. 进入空间后，一个虚拟人转向你，说："嗨，新朋友，你是第一次来索拉吧？"
4. 你随口回复了一句话，虚拟人做出肢体反应（点头/微笑）
5. 你逛了一圈空间，触发了一个隐藏彩蛋（虚拟人讲了个冷笑话）
6. 退出时，系统展示"下一个空间预览卡片"——你看到一个海滩空间，似乎有虚拟人在跳舞
7. [决定性时刻] 虚拟人在你退出时说："明天还来吗？我记住你了。"
8. 第二天，你收到推送："你的虚拟人朋友说想你了" → 你再次进入
```

> **需求关键点**：2秒加载、虚拟人首次互动个性化、退出推下一个空间、次日召回推送。此场景必须在技术验证期（原型1-3）内跑通。

#### 场景二：空间主的"创造存在"之旅（深度参与路径）

```
1. 用户连续使用7天后，收到引导："想拥有一个自己的空间吗？"
2. 选择"空白空间"或"模板空间"开始
3. 通过AI辅助生成（文字描述→3D场景）搭建第一个空间
4. 训练第一个专属虚拟人（选择性格模板 + 上传对话样本）
5. 发布空间，获取分享卡片
6. 第二天查看"空间访问数据"：3人访问，平均停留4分钟
7. [决定性时刻] 收到第一条"访客留言"："你的空间好酷，那个虚拟人讲的笑话我截图了"
8. 开始持续装修空间、训练虚拟人、吸引更多访客
9. 解锁"空间主"身份标识，进入推荐流量池
```

> **需求关键点**：AI辅助搭建（降低创作门槛）、空间数据分析、访客反馈闭环、创作者成长体系。

#### 场景三：社交活跃者的"与人同在"之旅（社交深度参与）

```
1. 进入一个"多人活跃空间"（同时有5-8人在场）
2. 看到其他用户由虚拟化身代表，正在进行活动（游戏/聊天/表演）
3. 一个虚拟人主持人主动邀请你参与："新来的朋友，要不要加入我们？"
4. 你通过语音/文字参与互动，虚拟化身实时同步你的动作
5. [爽点] 你的发言引发其他虚拟人和用户的积极回应
6. 添加了2个好友，约定明天同一时间再来
7. 退出时收到好友邀请确认，空间被加入"我的常去"
```

> **需求关键点**：多人状态同步（延迟<200ms）、虚拟人主持人机制、好友系统、常去空间收藏。

### 2.3 场景优先级矩阵

| 场景 | 覆盖角色 | 技术验证期 | 产品验证期 | 规模验证期 | 对应的信仰支柱 |
|------|---------|-----------|-----------|-----------|--------------|
| 场景一：首次被看见 | 游客→使用者 | **核心** | 优化 | 规模化 | 在场即意义 |
| 场景二：创造存在 | 使用者→常客→信徒 | 基础版 | **核心** | 规模化 | 创造即存在 |
| 场景三：与人同在 | 社交活跃者 | — | **核心** | 规模化 | AI看见人 |
| 场景四：创作者交易 | 创作者 | — | 测试版 | **核心** | 创造即存在 |
| 场景五：品牌空间 | 品牌方 | — | — | **核心** | 空间即归属 |

## 三、功能需求（Functional Requirements）

### 3.1 空间消费系统（对标抖音"刷视频"）

#### FR-SPACE-001：空间流式加载

| 属性 | 值 |
|------|-----|
| **优先级** | P0 — 无此功能产品不成立 |
| **描述** | 用户从点击空间到可交互的时间不超过2秒 |
| **验收标准** | ① 200MB基准空间在5G网络下进入时间<2秒；② 900MB高保真空间进入时间<5秒；③ 加载期间展示加载进度动画；④ 支持后台预加载下一个推荐空间 |
| **对应原型** | 第1个原型 |
| **技术依赖** | 流式渲染管线、预测性缓存算法、CDN分发 |

#### FR-SPACE-002：空间内容推荐分发

| 属性 | 值 |
|------|-----|
| **优先级** | P0 |
| **描述** | 基于用户行为的空间推荐算法，实现个性化空间流 |
| **验收标准** | ① 推荐算法综合用户停留时长、互动频率、虚拟人反馈、显式评分（喜欢/跳过）四个信号；② 冷启动空间（新发布）获得保底曝光量（≥100次/天）；③ 推荐多样性指数不低于0.6（避免同类空间过度集中） |
| **对应原型** | 第3-4个原型 |
| **技术依赖** | 推荐系统（协同过滤+内容识别）、空间特征向量提取 |

#### FR-SPACE-003：空间预览卡片

| 属性 | 值 |
|------|-----|
| **优先级** | P0 |
| **描述** | 退出当前空间时，自动展示下一个推荐空间的3D预览 |
| **验收标准** | ① 预览卡片为3D动态渲染（非静态截图）；② 展示空间标题、虚拟人数量、当前活跃人数；③ 用户可上划跳过/点击进入；④ 预览加载不增加退出操作的延迟感知（<300ms） |
| **对应原型** | 第2-3个原型 |
| **技术依赖** | 轻量3D渲染快照、退出动画衔接 |

#### FR-SPACE-004：空间搜索与分类

| 属性 | 值 |
|------|-----|
| **优先级** | P1 — 产品验证期必需 |
| **描述** | 用户可按类别、标签、关键词搜索空间 |
| **验收标准** | ① 支持按空间类型（聊天/游戏/表演/展览/治愈）、风格标签、虚拟人类型筛选；② 搜索响应时间<1秒；③ 展示热门空间排行榜（日/周/月） |
| **对应原型** | 第5-6个原型 |

### 3.2 虚拟人交互系统（索拉核心差异化能力）

#### FR-AVATAR-001：虚拟人实时对话

| 属性 | 值 |
|------|-----|
| **优先级** | P0 — 索拉的灵魂 |
| **描述** | 空间中的虚拟人可与用户进行自然语言实时对话 |
| **验收标准** | ① 语音/文字双模态输入支持；② 响应延迟<1秒（端侧推理）/<3秒（云端推理）；③ 对话支持上下文记忆（当前会话内）；④ 虚拟人对话时同步肢体动画（嘴型、表情、头部运动） |
| **对应原型** | 第1个原型 |
| **技术依赖** | 端侧SLM（小语言模型）+ 云端LLM混合推理、TTS、口型同步 |

#### FR-AVATAR-002：虚拟人长期记忆

| 属性 | 值 |
|------|-----|
| **优先级** | P1 — Plus及以上订阅功能 |
| **描述** | 虚拟人记住用户的历史互动，实现跨会话的个性化回应 |
| **验收标准** | ① 记住用户名字、偏好、上次聊天话题（≥3个记忆点）；② 记忆持久化≥30天；③ 用户可查看和管理虚拟人记忆内容；④ 记忆隐私保护：用户可删除特定记忆 |
| **对应原型** | 第3-4个原型 |
| **技术依赖** | 用户画像持久化、记忆检索RAG、隐私合规 |

#### FR-AVATAR-003：虚拟人情感模型

| 属性 | 值 |
|------|-----|
| **优先级** | P1 — Pro及以上订阅功能 |
| **描述** | 虚拟人具备多维度情感状态，可随互动动态变化 |
| **验收标准** | ① 支持≥5种情感维度（开心、好奇、冷淡、吃醋、失落）；② 情感变化遵循可解释的状态机（非随机）；③ 情感状态影响对话风格和行为（如"吃醋"时语速加快、提及用户与其他虚拟人的互动）；④ Pro用户可查看虚拟人情感日志 |
| **对应原型** | 第4-6个原型 |
| **技术依赖** | 情感计算模型、行为决策树+LLM混合 |

#### FR-AVATAR-004：虚拟人社交主动性

| 属性 | 值 |
|------|-----|
| **优先级** | P0 |
| **描述** | 虚拟人在空间中主动感知用户并发起互动，而非被动等待 |
| **验收标准** | ① 用户进入空间时虚拟人主动注视并打招呼（≤3秒内）；② 检测到用户长时间沉默（>30秒）时主动发起轻互动；③ 在多人空间中，确保每个在场用户都被至少1个虚拟人注意到；④ 主动互动频次可调节（用户设置/系统学习） |
| **对应原型** | 第2个原型 |
| **技术依赖** | 空间感知（用户位置/朝向检测）、主动互动触发条件引擎 |

#### FR-AVATAR-005：虚拟人个性化训练

| 属性 | 值 |
|------|-----|
| **优先级** | P2 — 产品验证后期 |
| **描述** | 空间主可训练自己空间的虚拟人性格和行为模式 |
| **验收标准** | ① 提供≥5种性格模板（温柔/毒舌/佛系/热情/高冷）；② 支持上传对话样本微调（≥20条样本可见效果）；③ 训练后的虚拟人保持一致性≥80%（不出现性格突变）；④ Ultra用户可导入个人聊天数据训练专属虚拟人 |
| **对应原型** | 第5-7个原型 |
| **技术依赖** | 性格嵌入向量、少样本微调、一致性约束 |

### 3.3 空间创作与管理系统

#### FR-CREATE-001：AI辅助空间搭建

| 属性 | 值 |
|------|-----|
| **优先级** | P1 — 产品验证期核心 |
| **描述** | 通过文字描述或参考图片，AI自动生成3D空间 |
| **验收标准** | ① 输入"一个日式禅意庭院，有流水和竹筒"→ 生成可用的3D空间骨架（>60%完成度）；② 生成时间<30秒；③ 支持迭代修改（"把竹筒换成石灯笼"）；④ 生成结果可进一步手动编辑 |
| **对应原型** | 第3-4个原型 |
| **技术依赖** | 3D资产生成模型、场景组合优化、空间语义理解 |

#### FR-CREATE-002：空间编辑器

| 属性 | 值 |
|------|-----|
| **优先级** | P1 |
| **描述** | 提供可视化的空间编辑功能，摆放家具/装饰/灯光/虚拟人 |
| **验收标准** | ① 拖拽式操作，支持移动端触摸交互；② 提供≥100个免费基础装饰物品；③ 支持撤销/重做（≥20步）；④ 实时预览编辑效果；⑤ 编辑器的打开/操作/保存全流程流畅度≥30fps |
| **对应原型** | 第3-5个原型 |

#### FR-CREATE-003：空间模板市场

| 属性 | 值 |
|------|-----|
| **优先级** | P2 — 产品验证后期 |
| **描述** | 创作者可售卖空间模板，用户可购买一键应用 |
| **验收标准** | ① 模板应用成功率≥95%；② 支持模板预览（3D漫游）；③ 购买流程≤3步；④ 平台抽成30%，创作者实时查看收益 |
| **对应原型** | 第6-8个原型 |
| **技术依赖** | 交易系统、模板兼容性校验 |

#### FR-CREATE-004：空间数据分析

| 属性 | 值 |
|------|-----|
| **优先级** | P2 |
| **描述** | 空间主可查看自己空间的访问和互动数据 |
| **验收标准** | ① 展示访客数、停留时长分布、互动热力图（空间中最常被互动的区域）；② 展示虚拟人被对话次数、情感反馈分布；③ 数据T+1更新（次日可见）；④ Pro用户可查看T+0实时数据 |
| **对应原型** | 第5-7个原型 |

### 3.4 社交与多人系统

#### FR-SOCIAL-001：多人空间同步

| 属性 | 值 |
|------|-----|
| **优先级** | P1 — 产品验证期必需 |
| **描述** | 多人同时在同一空间时，状态实时同步 |

| **对应原型** | 第4-5个原型 |
| **技术依赖** | WebRTC数据通道、状态同步协议、延迟补偿 |

#### FR-SOCIAL-002：空间内社交互动

### 3.5 用户成长与信仰转化系统

#### FR-GROWTH-001：用户等级与成就

| 属性 | 值 |
|------|-----|
| **优先级** | P2 |

#### FR-GROWTH-002：布道者体系

| 属性 | 值 |

## 九、风险与依赖项

### 9.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| 端侧推理延迟不达标（>1秒） | 虚拟人对话体验崩盘 | 中 | 先用纯文本模式（无TTS），再逐步叠加语音；或先接受3秒延迟，待端侧模型成熟后优化 |
| 3D流式加载在移动端不稳定 | 空间进入时间>5秒，用户流失 | 中高 | 先用200MB小空间验证，确认管线后再支撑900MB；提供"轻量模式"降级选项 |
| 端侧NPU兼容性差 | 大量低端机型无法使用端侧推理 | 高 | 必须设计纯云端推理降级方案（延迟3-5秒）；第1年不追求覆盖面，先在中高端机型验证 |
| 多人同步在弱网下体验差 | 社交场景不可用 | 中 | 先做2人同步验证，逐步扩展到多人；弱网自动降级为异步社交模式 |

### 9.2 市场与竞争风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| 字节/ Meta在索拉验证前推出同类产品 | 先发优势丧失 | 中 | 加速技术验证期（缩短至12-18个月）；专注"信仰"定位差异化，不与巨头在功能层面竞争 |
| 用户不存在"进入3D空间社交"的需求 | 产品不成立 | 低中 | 前3个原型即验证核心爽点假设；若验证失败，转向轻量2.5D方案 |
| 中国5G渗透速度不及预期 | 市场窗口推迟 | 低 | 索拉产品不依赖5G全覆盖，可通过WIFI+预加载覆盖早期用户 |

### 9.3 合规风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| AI虚拟人对话内容监管政策变化 | 合规成本飙升 | 中 | 在设计阶段内置内容安全过滤；所有虚拟人对话实时接入审核管道 |
| 虚拟物品交易涉及赌博认定 | 抽卡模式需调整 | 中 | 抽卡采用"无重复/好感度补偿"设计，规避赌博特征；法律合规团队早期介入 |
| 数据出境/隐私合规 | 跨境运营受限 | 中高 | 中国用户数据100%境内存储和处理；APP分国内版/国际版独立运营 |

## 十、附录

### 10.1 术语表

| 术语 | 定义 |
|------|------|
| **空间** | 索拉中的3D可交互虚拟场景，包含3D环境+虚拟人+事件系统 |
| **虚拟人** | 由大模型驱动的、在空间中具备自然对话和情感响应的AI角色 |
| **刷空间** | 用户通过连续进入推荐空间来消费内容的行为，对标抖音的"刷视频" |
| **空间预览卡片** | 退出当前空间时展示的下一个推荐空间的3D动态预览 |
| **决定性时刻** | 用户成长漏斗中每层转化的不可逆体验锚点 |
| **存在值** | 索拉用户成长体系的积分单位，衡量用户"在索拉中的存在深度" |
| **布道者** | 索拉信仰体系中的最高用户层级，自发传播索拉并享有官方认证 |
| **品牌在场权** | 品牌在索拉中以空间/虚拟人角色"存在"而非"投放广告"的权利 |
| **PMF** | Product-Market Fit，产品市场匹配 |
