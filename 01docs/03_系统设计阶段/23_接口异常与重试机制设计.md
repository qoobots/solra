## 2. 异常分类

### 2.1 异常类型体系

```
SolraException (抽象基类)
├── BusinessException          # 业务异常 (2xxxx)
│   ├── ValidationException    # 参数校验异常
│   ├── NotFoundException      # 资源不存在
│   ├── ConflictException      # 资源冲突
│   ├── ForbiddenException     # 权限不足
│   ├── RateLimitException     # 限流
│   └── AuthException          # 认证异常
├── SystemException            # 系统异常 (1xxxx)
│   ├── ServiceUnavailableException
│   ├── DatabaseException
│   ├── CacheException
│   └── MessageQueueException
└── ThirdPartyException        # 第三方异常 (3xxxx)
    ├── WechatException
    ├── PaymentException
    ├── SmsException
    └── PushException
```

### 2.2 异常处理代码示例

```java
// GlobalExceptionHandler.java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<ApiResponse> handleBusiness(BusinessException e) {
        return ResponseEntity
            .status(e.getHttpStatus())
            .body(ApiResponse.error(e.getCode(), e.getMessage()));
    }

    @ExceptionHandler(SystemException.class)
    public ResponseEntity<ApiResponse> handleSystem(SystemException e) {
        log.error("System error", e);
        return ResponseEntity
            .status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(ApiResponse.error(e.getCode(), "服务器内部错误"));
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ApiResponse> handleUnknown(Exception e) {
        log.error("Unknown error", e);
        return ResponseEntity
            .status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(ApiResponse.error(10500, "未知错误"));
    }
}
```

## 3. 重试机制

### 3.1 重试原则

| 场景 | 是否重试 | 原因 |
|------|----------|------|
| 网络超时 | ✅ | 瞬时故障 |
| 服务暂不可用(503) | ✅ | 可能恢复 |
| 数据库死锁 | ✅ | 重试大概率成功 |
| 参数错误(400) | ❌ | 重试不会改变结果 |
| 认证失败(401) | ❌ | 需刷新Token后重试 |
| 权限不足(403) | ❌ | 重试不会改变结果 |
| 资源不存在(404) | ❌ | 幂等，重试无意义 |
| 业务规则冲突 | ❌ | 需修改请求 |

### 3.2 客户端重试 (REST)

```java
// Spring Retry
@Retryable(
    value = {ServiceUnavailableException.class, TimeoutException.class},
    maxAttempts = 3,
    backoff = @Backoff(delay = 1000, multiplier = 2)
)
public <T> T executeWithRetry(Supplier<T> operation) {
    return operation.get();
}
```

### 3.3 服务间重试 (gRPC)

```yaml
# application.yml
grpc:
  client:
    default:
      retry:
        max-attempts: 3
        backoff:
          initial: 100ms
          max: 2s
          multiplier: 2
        retryable-status-codes:
          - UNAVAILABLE
          - DEADLINE_EXCEEDED
          - RESOURCE_EXHAUSTED
```

### 3.4 消息消费重试 (Kafka)

```yaml
# Kafka Consumer配置
spring:
  kafka:
    consumer:
      enable-auto-commit: false
      max-poll-records: 10
    listener:
      ack-mode: MANUAL
    properties:
      retry:
        max-attempts: 3
        backoff:
          initial-interval: 1000
          multiplier: 2
          max-interval: 10000
```

消费失败处理：
```
第1次失败 → 1秒后重试
第2次失败 → 2秒后重试
第3次失败 → 4秒后重试
全部失败 → 进入死信队列(DLQ) → 告警 → 人工处理
```

## 4. 熔断机制

### 4.1 Resilience4j配置

```yaml
resilience4j:
  circuitbreaker:
    instances:
      default:
        sliding-window-type: COUNT_BASED
        sliding-window-size: 10
        failure-rate-threshold: 50       # 50%失败率触发熔断
        wait-duration-in-open-state: 30s  # 30秒后半开
        permitted-number-of-calls-in-half-open-state: 3
        automatic-transition-from-open-to-half-open-enabled: true
      ai-service:
        sliding-window-size: 5
        failure-rate-threshold: 60
        wait-duration-in-open-state: 60s
      payment-service:
        sliding-window-size: 5
        failure-rate-threshold: 30        # 支付更敏感
        wait-duration-in-open-state: 120s
```

### 4.2 熔断状态机

```
        ┌──────────┐
        │  CLOSED  │ ← 正常状态
        └────┬─────┘
             │ 失败率 > 阈值
             ▼
        ┌──────────┐
        │   OPEN   │ ← 熔断（快速失败）
        └────┬─────┘
             │ 等待时间到期
             ▼
        ┌──────────┐
        │HALF_OPEN │ ← 半开（尝试放行少量请求）
        └────┬─────┘
             │
      ┌──────┴──────┐
      ▼              ▼
   成功→CLOSED    失败→OPEN
```

## 5. 降级策略

### 5.1 降级配置

```java
@CircuitBreaker(name = "recommendation", fallbackMethod = "recommendationFallback")
public FeedResponse getPersonalizedFeed(String userId) {
    return recommendationEngine.generate(userId);
}

public FeedResponse recommendationFallback(String userId, Exception e) {
    log.warn("Recommendation service degraded", e);
    // 降级：返回热门空间
    return spaceService.getHotSpaces(20);
}
```

### 5.2 各服务降级方案

| 服务 | 降级条件 | 降级行为 |
|------|----------|----------|
| 推荐服务 | 熔断/超时 | 返回热门空间 |
| AI推理 | 熔断/超时 | 返回预设回复 |
| 搜索服务 | 熔断 | 返回数据库直查结果 |
| 支付服务 | 熔断 | 展示维护公告 |
| 推送服务 | 熔断 | 记录消息待重推 |

## 6. 幂等性设计

### 6.1 幂等键

```java
// 客户端生成幂等键
@PostMapping("/orders")
public ApiResponse createOrder(
    @RequestHeader("X-Idempotency-Key") String idempotencyKey,
    @RequestBody CreateOrderRequest request
) {
    // 1. 检查Redis中是否已有此幂等键的结果
    String cached = redisTemplate.opsForValue().get("idem:" + idempotencyKey);
    if (cached != null) {
        return JSON.parseObject(cached, ApiResponse.class);
    }
    // 2. 执行操作
    ApiResponse response = orderService.create(request);
    // 3. 缓存结果（24小时）
    redisTemplate.opsForValue().set("idem:" + idempotencyKey,
        JSON.toJSONString(response), 24, TimeUnit.HOURS);
    return response;
}
```

### 6.2 事件消费幂等

```java
@KafkaListener(topics = "solra.user.events")
public void handleUserEvent(ConsumerRecord<String, String> record) {
    String eventId = extractEventId(record.value());
    // Redis SETNX 防止重复消费
    Boolean isNew = redisTemplate.opsForValue()
        .setIfAbsent("event:" + eventId, "processed", 7, TimeUnit.DAYS);
    if (Boolean.TRUE.equals(isNew)) {
        processEvent(record.value());
    }
}
```

## 7. 超时与限流

### 7.1 超时链

```
客户端 → 30s (整体)
  → Gateway → 25s
    → Service A → 10s
      → Service B → 5s
        → Database → 3s
```

### 7.2 限流策略

```yaml
# Sentinel配置
sentinel:
  rules:
    - resource: "createOrder"
      grade: QPS
      count: 100          # 每秒100次
    - resource: "aiChat"
      grade: QPS
      count: 50           # 每秒50次
    - resource: "register"
      grade: QPS
      count: 10           # 每秒10次（防刷）
```
