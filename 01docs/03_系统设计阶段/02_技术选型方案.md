## 六、选型原则

| 原则 | 说明 |
|------|------|
| 成熟优于新奇 | 优先有大规模生产案例的技术 |
| 统一优于多样 | 跨子项目统一语言/框架/工具链 |
| 渐进复杂度 | 技术验证期用最简单方案 |
| 端边云一致 | 端侧和云侧共享代码/协议 |
| 可替换性 | 每项技术保留替换路径 |
