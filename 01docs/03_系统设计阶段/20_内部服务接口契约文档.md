## 1. 内部接口概述

内部服务间通信采用 gRPC（同步调用）+ Kafka（异步事件）混合模式。

## 2. AUTH认证服务接口

### 2.1 auth_service.proto

```protobuf
syntax = "proto3";
package solra.auth;

service AuthService {
  // 验证Token
  rpc ValidateToken(ValidateTokenRequest) returns (ValidateTokenResponse);

  // 获取用户信息
  rpc GetUserInfo(GetUserInfoRequest) returns (UserInfo);

  // 批量获取用户信息
  rpc BatchGetUserInfo(BatchGetUserInfoRequest) returns (BatchGetUserInfoResponse);

  // 检查权限
  rpc CheckPermission(CheckPermissionRequest) returns (CheckPermissionResponse);

  // 获取用户角色
  rpc GetUserRoles(GetUserRolesRequest) returns (GetUserRolesResponse);
}

message ValidateTokenRequest {
  string token = 1;
}

message ValidateTokenResponse {
  bool valid = 1;
  string user_id = 2;
  repeated string roles = 3;
  int64 expires_at = 4;
  string error = 5;
}

message GetUserInfoRequest {
  string user_id = 1;
}

message UserInfo {
  string user_id = 1;
  string nickname = 2;
  string avatar_url = 3;
  string status = 4;
  bool youth_mode = 5;
}

message BatchGetUserInfoRequest {
  repeated string user_ids = 1;
}

message BatchGetUserInfoResponse {
  map<string, UserInfo> users = 1;
}
```

## 3. AVT虚拟人服务接口

### 3.1 avatar_service.proto

```protobuf
syntax = "proto3";
package solra.avatar;

service AvatarService {
  // 获取虚拟人信息
  rpc GetAvatar(GetAvatarRequest) returns (Avatar);

  // 批量获取虚拟人信息
  rpc BatchGetAvatars(BatchGetAvatarsRequest) returns (BatchGetAvatarsResponse);

  // 获取用户默认虚拟人
  rpc GetDefaultAvatar(GetDefaultAvatarRequest) returns (Avatar);

  // AI推理（内部调用）
  rpc Infer(InferRequest) returns (stream InferResponse);
}

message Avatar {
  string avatar_id = 1;
  string user_id = 2;
  string name = 3;
  string style = 4;
  string thumbnail_url = 5;
  string status = 6;
}

message InferRequest {
  string conversation_id = 1;
  string avatar_id = 2;
  string user_id = 3;
  string message = 4;
  InferConfig config = 5;
}

message InferConfig {
  string model = 1;        // llm-13b / llm-7b / slm-3b
  int32 max_tokens = 2;
  float temperature = 3;
  bool stream = 4;
}

message InferResponse {
  string token = 1;
  string message_id = 2;
  string emotion = 3;
  bool is_final = 4;
}
```

## 6. SAF安全服务接口

### 6.1 safety_service.proto

```protobuf
syntax = "proto3";
package solra.safety;

service SafetyService {
  // 内容安全检查
  rpc CheckContent(CheckContentRequest) returns (CheckContentResponse);

  // 敏感词过滤
  rpc FilterText(FilterTextRequest) returns (FilterTextResponse);
}

message CheckContentRequest {
  string content_type = 1;  // text / image / model / audio
  bytes content = 2;
  string user_id = 3;
  string context = 4;       // 额外上下文（空间ID等）
}

message CheckContentResponse {
  string result = 1;        // PASS / REJECT / MANUAL_REVIEW
  float confidence = 2;
  string reason = 3;
  repeated string tags = 4;
}

message FilterTextRequest {
  string text = 1;
  string level = 2;         // strict / normal / lenient
}

message FilterTextResponse {
  string filtered_text = 1;
  bool has_sensitive = 2;
  repeated string matched_words = 3;
}
```

## 8. 接口调用约定

### 8.1 超时配置

| 接口类型 | 超时 | 重试 |
|----------|------|------|
| 查询接口 | 3s | 2次 |
| 命令接口 | 5s | 1次 |
| AI推理 | 30s | 0次 |
| 批量接口 | 10s | 1次 |

### 8.2 错误处理

```protobuf
// common/error.proto
message Error {
  int32 code = 1;
  string message = 2;
  repeated ErrorDetail details = 3;
}

message ErrorDetail {
  string field = 1;
  string reason = 2;
  string message = 3;
}
```

### 8.3 gRPC元数据

| Key | 说明 |
|-----|------|
| `x-request-id` | 请求追踪ID |
| `x-user-id` | 调用方用户ID |
| `x-service-name` | 调用方服务名 |
| `x-trace-id` | 分布式追踪ID |

| 策略 | 说明 |
|------|------|

| 向后兼容 | 只新增字段，不修改已有字段编号 |
| 废弃字段 | 标记 `[deprecated = true]` |
| 新方法 | 在同一个Service中新增方法 |
