## 2. API网关 - Kong

### 2.1 选型理由

| 对比维度 | Kong | APISIX | Nginx |
|----------|------|--------|-------|
| 插件生态 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 性能 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| K8s原生 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 企业支持 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| 社区活跃 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

**结论**：Kong插件生态丰富，企业级支持好，选为默认API网关。

### 2.2 核心插件配置

| 插件 | 用途 | 配置要点 |
|------|------|----------|
| JWT | 认证 | Token验证、过期检查、角色提取 |
| Rate Limiting | 限流 | 用户级100req/s、IP级500req/s |
| CORS | 跨域 | 允许Web/移动端域名 |
| Request Transformer | 请求转换 | 统一Header注入（TraceID/UserID） |
| Prometheus | 监控 | 网关指标暴露 |
| Zipkin | 链路追踪 | TraceID透传 |
| IP Restriction | IP白名单 | 管理后台IP限制 |

## 3. 服务网格 - Istio

### 3.1 核心能力

| 能力 | 实现方式 |
|------|----------|
| 流量管理 | VirtualService + DestinationRule（金丝雀/蓝绿/AB测试） |
| 安全 | mTLS自动加密 + AuthorizationPolicy（服务间零信任） |
| 可观测性 | 自动注入Envoy Sidecar（指标/日志/追踪） |
| 故障注入 | FaultInjection（延迟/中断测试） |
| 熔断 | CircuitBreaker（异常比例熔断） |
| 重试 | Retry策略（可配置次数/超时） |

### 3.2 流量管理策略

```yaml
# 金丝雀发布示例
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
spec:
  http:
  - match:
    - headers:
        x-canary:
          exact: "true"
    route:
    - destination:
        host: avt-service
        subset: v2
      weight: 5
  - route:
    - destination:
        host: avt-service
        subset: v1
      weight: 95
```

### 3.3 安全策略

```yaml
# 零信任网络策略
apiVersion: security.istio.io/v1beta1
kind: AuthorizationPolicy
spec:
  action: DENY
  rules:
  - from:
    - source:
        notNamespaces: ["solra-auth"]
    to:
    - operation:
        paths: ["/api/auth/internal/*"]
```

## 4. 消息队列 - Kafka

### 4.1 集群配置

| 配置项 | 值 |
|--------|-----|
| 版本 | Kafka 3.7 (KRaft) |
| 节点数 | 3 |
| 分区数 | 12/Topic（默认） |
| 副本因子 | 3 |
| 保留策略 | 7天或1TB |
| 压缩 | snappy |

### 4.2 Topic设计

| Topic | 分区数 | 消费者组 | 用途 |
|-------|--------|----------|------|
| solra.user.events | 12 | avt/soc/grw/mon | 用户事件 |
| solra.space.events | 12 | soc/grw/saf/not | 空间事件 |
| solra.avatar.events | 6 | saf/not | 虚拟人事件 |
| solra.social.events | 12 | grw/not | 社交事件 |
| solra.transaction.events | 6 | grw/not | 交易事件 |
| solra.content.events | 6 | saf/not | 内容事件 |
| solra.notification.events | 6 | not | 通知事件 |

### 4.3 消费保证

| 保证 | 实现 |
|------|------|
| 至少一次 | enable.auto.commit=false，手动提交 |
| 幂等消费 | 基于事件ID去重（Redis SETNX） |
| 顺序消费 | 同一聚合根ID路由到同一分区 |
| 死信队列 | 消费失败3次后进入DLQ |
| 监控 | Kafka Lag监控 + 告警 |

## 5. 分布式缓存 - Redis

### 5.1 集群配置

| 配置项 | 值 |
|--------|-----|
| 模式 | Redis Cluster (6节点) |
| 版本 | Redis 7.2 |
| 持久化 | AOF everysec |
| 淘汰策略 | allkeys-lru |
| 最大内存 | 16GB/节点 |

### 5.2 缓存策略

| 数据类型 | TTL | 用途 |
|----------|-----|------|
| Session | 7d | 用户登录会话 |
| Token | 15min | Access Token |
| RefreshToken | 7d | Refresh Token |
| Presence | 30s (心跳) | 用户在线状态 |
| HotSpaces | 5min | 热门空间列表 |
| Recommendations | 30min | 个性化推荐 |
| RateLimiter | 窗口时间 | 限流计数 |
| UserProfile | 1h | 用户信息缓存 |

### 5.3 缓存模式

| 模式 | 场景 | 说明 |
|------|------|------|
| Cache-Aside | 通用读缓存 | 应用管理缓存填充 |
| Write-Through | 会话写入 | 写DB同步写Redis |
| Write-Behind | 非关键写入 | 异步批量写回 |
