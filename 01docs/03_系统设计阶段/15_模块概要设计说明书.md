## 1. 模块划分总览

基于DDD限界上下文，系统划分为10个一级模块（对应10个子项目），每个一级模块下包含若干二级模块。

| 一级模块 | 子项目 | 二级模块数 | 功能点数 |
|----------|--------|-----------|---------|
| M1 认证授权 | qooauth | 4 | 5 |
| M2 虚拟人交互 | qoobrain/qoobody | 6 | 12 |
| M3 空间消费 | qooweb/qoocloud | 5 | 10 |
| M4 空间创建 | qoocore/qoodev | 4 | 8 |
| M5 社交存在 | qoosvc | 4 | 6 |
| M6 用户增长 | qoochain | 4 | 6 |
| M7 变现 | qoostore | 4 | 6 |
| M8 内容安全 | qoocompliance | 3 | 5 |
| M9 通知 | qoogear | 2 | 3 |
| M10 基础设施 | qoocommunity | 3 | 4 |

**合计**：39个二级模块，65个核心功能点（不含通用功能）

## 2. M1 认证授权模块 (qooauth)

### 2.1 模块结构

```
qooauth/
├── auth-api/           # REST API层
├── auth-core/          # 核心业务逻辑
├── auth-oauth/         # 第三方OAuth集成
└── auth-security/      # 安全组件（Token/加密）
```

### 2.2 二级模块

| 模块ID | 模块名 | 职责 | 关键类 |
|--------|--------|------|--------|
| M1.1 | 用户注册登录 | 账号注册/登录/找回密码 | UserService, AuthController |
| M1.2 | 第三方OAuth | 微信/Apple/Google登录 | OAuthService, OAuthProvider |
| M1.3 | Token管理 | JWT签发/验证/刷新 | TokenProvider, TokenValidator |
| M1.4 | 权限管理 | RBAC角色/权限/鉴权 | RoleService, PermissionEvaluator |

### 2.3 模块依赖

```
M1.1 → M1.2 (OAuth注册时创建用户)
M1.1 → M1.3 (登录时签发Token)
M1.1 → M1.4 (用户绑定角色)
M1.3 → M1.4 (Token中携带权限)
```

## 3. M2 虚拟人交互模块 (qoobrain + qoobody)

### 3.1 模块结构

```
qoobrain/
├── brain-api/          # AI对话API
├── brain-core/         # 推理引擎/意图识别/路由
├── brain-memory/       # 记忆管理/RAG
└── brain-model/        # 模型管理/SLM/LLM适配

qoobody/
├── body-api/           # 虚拟人管理API
├── body-avatar/        # 虚拟人创建/编辑
├── body-animation/     # 动作驱动/表情生成
└── body-voice/         # ASR/TTS
```

### 3.2 二级模块

| 模块ID | 模块名 | 职责 | 关键类 |
|--------|--------|------|--------|
| M2.1 | 虚拟人创建管理 | 创建/编辑/装扮/风格切换 | AvatarService |
| M2.2 | AI对话引擎 | 意图识别/模型路由/对话管理 | ConversationEngine |
| M2.3 | 记忆系统 | RAG检索/长期记忆/上下文管理 | MemoryService |
| M2.4 | 动作驱动 | Motion Matching/IK/动画混合 | AnimationDriver |
| M2.5 | 表情情感 | 情感识别/BlendShape驱动 | EmotionEngine |
| M2.6 | 语音交互 | ASR语音识别/TTS语音合成 | VoiceService |

### 3.3 模块依赖

```
M2.2 → M2.3 (对话引用记忆)
M2.2 → M2.5 (回复带情感)
M2.2 → M2.6 (语音输入输出)
M2.1 → M2.4 (虚拟人动作)
M2.1 → M2.5 (虚拟人表情)
```

## 4. M3 空间消费模块 (qooweb + qoocloud)

### 4.1 模块结构

```
qooweb/
├── web-portal/         # Web门户（Vue3）
└── web-api/            # Web API网关

qoocloud/
├── cloud-discovery/    # 空间发现/搜索/推荐
├── cloud-session/      # 空间会话/负载均衡
├── cloud-realtime/     # 实时通信(WebSocket/WebRTC)
└── cloud-live/         # 直播/活动
```

### 4.2 二级模块

| 模块ID | 模块名 | 职责 | 关键类 |
|--------|--------|------|--------|
| M3.1 | 空间发现 | 推荐流/热门/分类浏览 | DiscoveryService |
| M3.2 | 空间搜索 | 全文搜索/语义搜索/过滤 | SearchService |
| M3.3 | 空间会话 | 进入/退出/实例管理/状态同步 | SessionService |
| M3.4 | 实时通信 | WebSocket/WebRTC信令/消息广播 | RealtimeService |
| M3.5 | 直播活动 | 推流/拉流/弹幕/礼物 | LiveService |

### 4.3 模块依赖

```
M3.1 → M3.2 (搜索结果融入推荐)
M3.1 → M3.3 (推荐进入空间)
M3.3 → M3.4 (空间内实时交互)
M3.5 → M3.4 (直播信令)
```

## 5. M4 空间创建模块 (qoocore + qoodev)

### 5.1 模块结构

```
qoocore/
├── core-engine/        # C++17渲染引擎（Vulkan/Metal/GLES）
├── core-physics/       # 物理引擎（Bullet）
├── core-network/       # 网络层（WebRTC/QUIC）
└── core-asset/         # 资源管理

qoodev/
├── dev-ide/            # IDE界面（PySide6）
├── dev-editor/         # 空间编辑器
├── dev-debugger/       # 脚本调试器
├── dev-profiler/       # 性能剖析
└── dev-asset-mgr/      # 资源浏览器
```

### 5.2 二级模块

| 模块ID | 模块名 | 职责 | 关键类 |
|--------|--------|------|--------|
| M4.1 | 渲染引擎 | Vulkan/Metal/GLES多后端渲染 | Renderer, Pipeline |
| M4.2 | 物理引擎 | 碰撞检测/刚体模拟/布料 | PhysicsWorld |
| M4.3 | 空间编辑 | 场景编辑/物体操作/材质编辑 | SpaceEditor |
| M4.4 | 资源管理 | 上传/优化/版本管理/CDN | AssetManager |

### 5.3 模块依赖

```
M4.3 → M4.1 (编辑预览渲染)
M4.3 → M4.2 (物理预览)
M4.3 → M4.4 (资源引用)
M4.4 → M3.2 (资源搜索)
```

## 12. 跨模块交互矩阵

| 调用方 | 被调用方 | 调用方式 | 说明 |
|--------|----------|----------|------|
| M3.1 空间发现 | M3.2 空间搜索 | 同步(gRPC) | 搜索结果融入推荐 |
| M3.3 空间会话 | M3.4 实时通信 | 同步(本地) | 空间内消息转发 |
| M3.3 空间会话 | M5.4 在线状态 | 异步(Kafka) | 进入/离开广播 |
| M3.1 空间发现 | M7.4 创作者结算 | 异步(Kafka) | 空间热度→收入 |
| M4.3 空间编辑 | M4.4 资源管理 | 同步(gRPC) | 资源引用 |
| M4.4 资源管理 | M8.1 自动审核 | 异步(Kafka) | 资源上传→审核 |
| M5.3 消息服务 | M8.1 自动审核 | 同步(gRPC) | 消息发送前审核 |
| M6.3 任务系统 | M7.2 订单支付 | 异步(Kafka) | 购买行为→任务进度 |
| M9.1 消息推送 | M5.4 在线状态 | 同步(gRPC) | 判断推送渠道 |
