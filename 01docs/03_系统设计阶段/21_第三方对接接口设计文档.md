## 2. 微信开放平台对接

### 2.1 微信登录

```
流程:
1. 客户端调起微信授权 → 用户确认
2. 微信返回 authorization_code
3. 服务端用 code 换取 access_token + openid
4. 服务端用 access_token 获取用户信息
5. 匹配/创建 Solra 用户 → 签发 JWT
```

**接口调用**：
```
GET https://api.weixin.qq.com/sns/oauth2/access_token
  ?appid={APPID}
  &secret={SECRET}
  &code={CODE}
  &grant_type=authorization_code

Response:
{
  "access_token": "...",
  "expires_in": 7200,
  "refresh_token": "...",
  "openid": "oXXXX",
  "unionid": "oYYYY"
}

GET https://api.weixin.qq.com/sns/userinfo
  ?access_token={ACCESS_TOKEN}
  &openid={OPENID}
```

### 2.2 微信支付

**统一下单**：
```
POST https://api.mch.weixin.qq.com/v3/pay/transactions/jsapi

Request:
{
  "appid": "wx...",
  "mchid": "123...",
  "description": "Solra虚拟商品购买",
  "out_trade_no": "SO20260630120000001",
  "notify_url": "https://api.solra.io/api/v1/store/payments/wechat/callback",
  "amount": { "total": 9900, "currency": "CNY" },
  "payer": { "openid": "oXXXX" }
}
```

**回调处理**：
```
POST /api/v1/store/payments/wechat/callback (微信→Solra)

验证步骤:
1. 验证签名（微信平台证书）
2. 解密资源数据（AES-256-GCM）
3. 查询订单状态
4. 更新订单状态 → 发放商品
5. 返回应答 { "code": "SUCCESS" }
```

## 3. Apple Sign In 对接

```
流程:
1. 客户端获取 Apple authorization_code + identity_token
2. 服务端验证 identity_token (JWT, RS256, Apple公钥)
3. 解析 sub (用户唯一标识) + email
4. 匹配/创建 Solra 用户 → 签发 JWT

验证端点:
POST https://appleid.apple.com/auth/token
  client_id={BUNDLE_ID}
  client_secret={CLIENT_SECRET}
  code={CODE}
  grant_type=authorization_code
```

## 4. 云服务对接

### 4.1 CDN (腾讯云CDN)

```
上传流程:
1. 客户端/服务端上传文件到源站(MinIO)
2. 触发CDN预热
3. 返回CDN URL

CDN URL格式:
https://cdn.solra.io/{bucket}/{path}

鉴权配置:
- TypeA: 时间戳+MD5签名 URL鉴权
- Referer白名单: *.solra.io
```

### 4.2 短信服务 (腾讯云SMS)

```
发送验证码:
POST https://sms.tencentcloudapi.com/

参数:
- PhoneNumberSet: ["+8613800138000"]
- TemplateId: "1234567"
- TemplateParamSet: ["123456"]  // 验证码
- SmsSdkAppId: "1400000000"
```

### 4.3 对象存储 (MinIO S3兼容)

```
客户端直传:
1. 服务端生成 Presigned URL
2. 客户端用URL直接上传到MinIO
3. 上传完成后回调服务端

Presigned URL生成:
POST /api/v1/crt/assets/upload-url
→ { uploadUrl: "https://...", assetId: "uuid" }
```

## 6. 第三方接口容错

### 6.1 超时与重试

| 接口 | 超时 | 重试 | 退避 |
|------|------|------|------|
| 微信OAuth | 5s | 2次 | 1s, 2s |
| 微信支付 | 10s | 3次 | 1s, 2s, 4s |
| 短信发送 | 5s | 1次 | - |
| 推送 | 3s | 2次 | 1s, 2s |
| CDN刷新 | 30s | 2次 | 5s, 10s |

### 6.2 降级策略

| 第三方故障 | 降级行为 |
|-----------|----------|
| 微信登录不可用 | 引导用户使用手机号注册 |
| 微信支付不可用 | 展示支付宝/Apple Pay备选 |
| 短信不可用 | 提示用户稍后重试，支持邮箱验证码 |
| 推送不可用 | 用户打开App时拉取离线消息 |

### 6.3 监控告警

- 第三方接口成功率 < 95% → P2告警
- 第三方接口P99延迟 > 5s → P3告警
- 支付回调延迟 > 1min → P1告警
