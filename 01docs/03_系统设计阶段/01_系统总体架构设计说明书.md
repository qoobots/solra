## 一、文档概述

### 1.1 编写目的

本文档描述索拉（Solra）系统的总体架构设计，定义系统的分层结构、核心组件、技术选型和关键架构决策。

### 1.2 适用范围

适用于索拉项目全部 10 个子项目、4 个客户端平台、6 个技术层次。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 子项目 | DDD 限界上下文，独立业务域 + 部署单元，共 10 个 |
| 模块 | 子项目内按技术/业务内聚的功能组，共 32 个 |
| Core SDK | C++17 跨平台核心引擎 |
| SLM/LLM | 端侧/云端大语言模型 |

## 二、架构设计目标与原则

### 2.1 设计目标

| 目标 | 技术挑战 | 架构策略 |
|------|---------|---------|
| 低延迟进入（< 2s） | 移动端网络 + 3D 资产大 | 流式加载 + 预测预加载 + CDN |
| 高保真 AI 交互 | GPU 成本 + 端侧算力 | 端侧 SLM + 云端 LLM 混合推理 |
| 规模化服务（千万 DAU） | 状态同步 + 推理并发 | 事件驱动 + 无状态 + 边缘计算 |

### 2.2 设计原则

| 原则 | DDD 映射 |
|------|---------|
| 子项目独立自治 | 每个子项目 = 独立微服务群 |
| 事件驱动松耦合 | 领域事件发布/订阅 |
| 端边云协同 | CQRS 读写分离 |
| 流式优先 | 空间/对话/同步均流式设计 |
| 安全内建 | 防腐层（ACL）模式 |

### 2.3 设计约束

- 中国用户数据 100% 境内存储
- 等保 2.0 三级合规
- PCI-DSS 支付安全合规
- 开源模型 Apache 2.0 协议优先

## 三、总体架构视图

### 3.1 逻辑架构（四层）

```
L0 客户端层：iOS(SwiftUI+Metal) | Android(Compose+Vulkan)
             Web(Vue3) | Desktop(Tauri2+Rust+Vue3)
             └─ Solra Core SDK (C++17)

L1 接入层：APISIX 网关（鉴权/限流/灰度/协议转换）

L2 业务服务层：10 个微服务（Java 17 + Spring Boot 3.3 + gRPC）
             核心域 AVT/SPC | 支撑域 CRT/SOC/GRW/MON
             通用域 AUTH/SAF/NOT

L3 AI 推理层：LLM Router | 安全模型 | 嵌入 | TTS | 推荐（Python 3.11 + FastAPI）

L4 基础设施层：PostgreSQL | Redis | ES | Kafka | MinIO | K8s + ArgoCD
```

### 3.2 物理架构

```
┌─────────────────────────────────────────────────┐
│  K8s Cluster - 业务 (10 个 Namespace)            │
│  ┌─────────────────────────────────────────────┐│
│  │ Namespace: avatar │ space │ creator │ ...   ││
│  │ 每个 Namespace 包含对应子项目的微服务群      ││
│  └─────────────────────────────────────────────┘│
├─────────────────────────────────────────────────┤
│  K8s Cluster - AI 推理 (GPU 节点)               │
│  ┌─────────────────────────────────────────────┐│
│  │ vLLM Server × N | Safety Model | TTS | Emb  ││
│  │ GPU: NVIDIA A100/H100 | Karpenter 自动扩缩  ││
│  └─────────────────────────────────────────────┘│
├─────────────────────────────────────────────────┤
│  K8s Cluster - 数据 (StatefulSet)               │
│  ┌─────────────────────────────────────────────┐│
│  │ PostgreSQL (1主2从) | Redis (6节点)         ││
│  │ Kafka (3节点) | Milvus | Elasticsearch      ││
│  └─────────────────────────────────────────────┘│
├─────────────────────────────────────────────────┤
│  边缘 CDN 节点：空间资产缓存 + 预览渲染          │
└─────────────────────────────────────────────────┘
```

## 四、分层架构设计

### 4.1 表现层（客户端）

| 平台 | 技术 | 3D 渲染 | AI 推理 |
|------|------|---------|---------|
| iOS | SwiftUI | Metal 3 | CoreML + llama.cpp |
| Android | Compose | Vulkan 1.3 | NNAPI + llama.cpp |
| Windows Desktop | Tauri 2 + Vue3 | Vulkan/OpenGL | ONNX Runtime + llama.cpp |
| Web H5 | Vue3 | - | - |

### 4.2 应用层（业务服务）

每个微服务遵循 DDD 四层架构：

```
interfaces/  → gRPC/REST/事件监听器
application/ → 命令/查询处理器（用例编排）
domain/      → 聚合根/实体/值对象/领域服务/仓储接口
infrastructure/ → 持久化/消息/外部服务客户端
```

### 4.3 领域层（DDD 聚合设计）

| 子项目 | 聚合根 | 核心实体 | 值对象 |
|--------|--------|---------|--------|
| AVT | Conversation | Avatar, Memory, DialogueTurn | EmotionState, PersonalityProfile |
| SPC | SpaceVisit | Space, RecommendationFeed | SpaceLoadMetrics, PreviewCard |
| CRT | SpaceProject | Asset, Decoration | LightingPreset, AudioConfig |
| SOC | SpaceSession | Presenter, ChatMessage | Position3D, AudioRange |
| GRW | UserProfile | Achievement, EvangelistApplication | PresenceScore, FaithLevel |
| MON | Order | Subscription, VirtualItem | Price, Discount |
| AUTH | UserAccount | LoginSession, DeviceBinding | PhoneNumber, AuthToken |
| SAF | ReviewCase | ContentSnapshot, ReviewResult | SafetyScore |

### 4.4 基础设施层

- **API 网关**：APISIX（路由/鉴权/限流/灰度）
- **数据库**：PostgreSQL 16（OLTP）
- **缓存**：Redis 7 Cluster
- **消息队列**：Kafka（领域事件 + 行为日志）
- **对象存储**：MinIO / S3（空间资产）
- **向量数据库**：Milvus（虚拟人记忆/推荐）
- **搜索引擎**：Elasticsearch（空间搜索/日志）
- **容器编排**：Kubernetes + ArgoCD
- **可观测性**：Prometheus + Grafana + Jaeger + ELK

## 六、关键架构决策（ADR）

| ADR | 决策 | 理由 |
|-----|------|------|
| ADR-001 | 自研流式加载框架 | 200MB 空间 < 2s 加载，现有方案不支持 |
| ADR-002 | 端侧+云端混合推理 | 降低延迟 + 控制成本 |
| ADR-003 | 微服务+事件驱动 | 10 子项目独立演进 |
| ADR-007 | 新增 Windows Desktop | WebGL 限制 GPU 利用率，Tauri 原生集成 Core SDK |
| ADR-008 | 自研 3D 引擎 | AI 辅助开发压缩工期至 5-7 月，包体 < 15MB |
