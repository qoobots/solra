# 03_测试环境搭建方案

> **文档编号**：05-03 | **版本号**：v1.0 | **编制人**：QA组 | **状态**：已发布 | **更新日期**：2026-06-30

## 1. 测试环境架构

```
dev (本地)          → Docker Compose单机
staging (预发布)    → K8s集群 (4节点, 等同生产缩微版)
perf (性能)         → K8s集群 (12节点, 等同生产)
```

## 2. 本地测试环境 (dev)

```bash
# 启动测试依赖
docker compose -f docker-compose.test.yml up -d

# 包含:
# - PostgreSQL 16 (端口 5433)
# - Redis 7 (端口 6380)
# - Kafka 3.7 (端口 9093)
# - MinIO (端口 9002)
# - Elasticsearch (端口 9201)
# - Milvus Standalone (端口 19531)
```

## 3. Staging环境

- 通过GitHub Actions自动部署
- 代码合并到main后自动触发
- 数据库使用匿名化生产数据副本
- 第三方服务使用Sandbox环境

## 4. Mock策略

| 依赖 | Mock方式 | 说明 |
|------|----------|------|
| 微信/支付宝 | WireMock | 模拟支付回调 |
| 短信服务 | WireMock | 模拟发送 |
| AI推理 | Mock Server | 预设返回 |
| 推送服务 | WireMock | 模拟推送 |

## 5. 环境清理

- 集成测试后自动清理Testcontainers
- Staging每日凌晨自动重置数据库
- 性能测试后恢复初始状态
