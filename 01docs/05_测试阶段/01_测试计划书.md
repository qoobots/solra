# 01_测试计划书

> **文档编号**：05-01 | **版本号**：v1.0 | **编制人**：QA组 | **状态**：已发布 | **更新日期**：2026-06-30

---

## 1. 测试目标

确保Solra平台10个子项目的功能正确性、性能达标、安全合规。覆盖78个功能点，5个客户端平台，3个交付阶段。

## 2. 测试范围

| 测试类型 | 范围 | 优先级 |
|----------|------|--------|
| 单元测试 | 所有Java/Python/C++/TS模块 | P0 |
| 集成测试 | 服务间API/gRPC/Kafka | P0 |
| 系统测试 | 端到端业务流程 | P0 |
| 性能测试 | 核心API + AI推理 + 实时通信 | P1 |
| 安全测试 | OWASP Top 10 + 渗透 | P1 |
| 兼容性测试 | 5平台 + 主流浏览器 | P1 |
| UAT | 真实用户场景验收 | P2 |

## 3. 测试策略

- **测试金字塔**：单元70% + 集成20% + E2E 10%
- **自动化率目标**：单元100% / 集成80% / E2E 50%
- **环境**：dev(开发自测) → staging(集成测试) → prod-like(性能测试)

## 4. 测试环境

| 环境 | 用途 | 配置 |
|------|------|------|
| dev | 开发自测 | 单节点，Mock第三方 |
| staging | 集成/回归 | K8s 4节点，真实依赖 |
| perf | 性能测试 | K8s 12节点，等同生产 |

## 5. 进度安排

| 阶段 | 时间 | 测试类型 |
|------|------|----------|
| P0 技术验证 | 1-6月 | 单元 + 集成 + 冒烟 |
| P1 产品验证 | 7-12月 | 系统 + 性能 + 安全 |
| P2 规模验证 | 13-18月 | 压测 + 容灾 + UAT |

## 6. 风险

| 风险 | 缓解措施 |
|------|----------|
| AI输出不确定性 | 模糊匹配 + 人工抽检 |
| 3D渲染差异 | 多设备截图对比 |
| 第三方依赖不稳定 | Mock + 契约测试 |
