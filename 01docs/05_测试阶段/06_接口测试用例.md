## 1. 接口测试策略

- 使用 REST Assured (Java) / pytest + httpx (Python) / Supertest (TS)
- 覆盖所有公开API的：正常路径 / 异常路径 / 边界条件 / 权限校验
- 通过Postman Collection / OpenAPI自动生成基础用例

## 2. 接口测试用例示例

### API-AUTH-01: 登录成功

```
POST /api/v1/auth/users/login
Request: {"account":"13800138000","password":"Test@123"}
Expect: 200, body.code=0, data.accessToken exists
```

### API-AUTH-02: 密码错误

```
POST /api/v1/auth/users/login
Request: {"account":"13800138000","password":"wrong"}
Expect: 401, body.code=10102
```

### API-AUTH-03: Token过期访问

```
GET /api/v1/space/spaces/feed
Header: Authorization: Bearer {expired_token}
Expect: 401, body.code=10101
```

### API-SPC-01: 分页查询空间

```
GET /api/v1/space/spaces?page=1&size=20
Expect: 200, data.items.length ≤ 20, data.pagination.total ≥ 0
```

### API-SPC-02: 搜索空间

```
GET /api/v1/space/spaces/search?q=图书馆
Expect: 200, data.items 包含相关空间
```

### API-AVT-01: 创建虚拟人

```
POST /api/v1/avatar/avatars
Request: {"name":"测试虚拟人","style":"realistic","faceParams":{...}}
Expect: 201, data.id exists
```

### API-AVT-02: 发送对话消息(SSE)

```
POST /api/v1/avatar/conversations/{id}/messages
Request: {"content":"你好","contentType":"text"}
Expect: 200, Content-Type: text/event-stream, 流式返回
```
