## 1. 测试框架

| 语言 | 单元测试 | 集成测试 | E2E |
|------|----------|----------|-----|
| Java | JUnit 5 + Mockito | Spring Boot Test + Testcontainers | - |
| Python | pytest + pytest-asyncio | httpx + Testcontainers | - |
| TypeScript | Vitest + Vue Test Utils | - | Playwright |
| C++ | Google Test + GMock | 自定义集成框架 | - |

## 2. 测试分层

```
┌────────────────────────┐
│     E2E Tests          │ ← Playwright (关键用户旅程)
├────────────────────────┤
│   Integration Tests    │ ← API/gRPC/Kafka/DB
├────────────────────────┤
│     Unit Tests         │ ← 类/函数级别
└────────────────────────┘
```

## 3. 测试数据管理

- 使用Testcontainers启动真实依赖（PG/Redis/Kafka）
- 测试数据通过Builder模式构建
- 每个测试类独立数据，`@Transactional`自动回滚
- 性能测试使用生产级数据量

## 4. CI集成

```yaml
# GitHub Actions
test:
  unit:
    runs-on: ubuntu-latest
    steps:
      - run: ./gradlew test
      - run: pytest
      - run: npm test
      - run: cd core/build && ctest
  integration:
    needs: unit
    runs-on: ubuntu-latest
    services: [postgres, redis, kafka]
    steps:
      - run: ./gradlew integrationTest
  e2e:
    needs: integration
    steps:
      - run: npx playwright test
```
