# 索拉（Solra）产品设计文档

> 从"刷屏幕"到"进入空间"的交互范式跃迁——索拉的交互设计、视觉语言与信息架构

---

## 一、设计哲学

### 1.1 核心设计原则

索拉的设计不是对抖音的"3D化套皮"，而是对移动端人机交互的一次**范式重构**。从"刷屏幕"（Screen-Swiping）到"进入空间"（Space-Entering），交互的基本假设从"用户在看2D内容流"转变为"用户是一个存在于3D场域中的在场者"。

| 设计原则 | 说明 | 反例（应避免的） |
|---------|------|-----------------|
| **空间优先（Space-First）** | 3D空间是交互的第一容器，UI浮于空间之上而非替代空间 | 用全屏菜单遮住空间，打断"在场感" |
| **虚拟人是界面** | 虚拟人不只是内容，而是交互入口——用户通过与虚拟人对话完成操作 | 传统按钮+表单模式（"设置虚拟人属性"页面） |
| **物理隐喻** | 空间中的操作遵循物理直觉：靠近=感兴趣、转身=离开、点头=确认 | 抽象UI逻辑与空间行为割裂 |
| **渐进沉浸** | 从2D卡片→3D预览→全沉浸空间，三步渐进而非突然跳入 | 冷启动直接进入满屏3D空间导致认知过载 |
| **退出即推进** | 退出当前空间的瞬间，是推送下一个空间的黄金时刻 | 退出到冷冰冰的列表页 |
| **静默在场** | 用户什么都不做也应该被感知到——虚拟人注视你、空间因你的存在而微变 | "只有你点击互动按钮时才有反馈" |

### 1.2 设计对标与差异化

| 维度 | 抖音（屏幕范式） | 索拉（空间范式） | 设计策略 |
|------|---------------|---------------|---------|
| 最小交互单元 | 上划 | 进入/退出 | 用"进入"替代"上划" |
| 内容切换方式 | 全屏自动切换 | 空间过渡动画+预览卡片 | 退出动画+下一个空间预览二合一 |
| 反馈机制 | 点赞震动/数字变化 | 虚拟人肢体反应/空间环境变化 | 反馈从"数字"变为"行为" |
| 导航结构 | Tab Bar + 无限Feed流 | 空间为家 + 预览流 + 侧边栏 | 以"我的空间"为锚点向外探索 |
| 社交信号 | 粉丝数/点赞数/评论数 | 访客数/停留时长/虚拟人关系深度 | 度量从"数量"变为"深度" |
| 创作表达 | 拍摄视频/编辑发布 | 搭建空间/训练虚拟人/发布 | 表达从"片段"变为"场域" |

### 1.3 用户认知模型的渐进构建

索拉面临的核心设计挑战：**用户没有"进入3D空间社交"的认知模型**。设计必须从用户熟悉的范式出发，逐步引导进入新范式。

```
认知迁移路径：
抖音用户习惯 "上划刷视频" 
    → 索拉先用 "2D卡片→轻3D预览" 建立认知桥梁
    → 再用 "全3D空间" 完成范式转移
    → 最终让用户形成 "进入→在场→退出→再进入" 的新习惯循环
```

### 2.2 主导航模式：空间即一切

索拉不采用传统的底部Tab Bar作为主导航。核心导航围绕"空间"展开：

| 导航入口 | 触发方式 | 目的 | 对标参考 |
|---------|---------|------|---------|
| **推荐空间流** | 退出当前空间自动展示 | 发现新空间（主力消费路径） | 抖音上划→下一个视频 |
| **我的空间** | 侧边栏入口 / 长按底部头像 | 回到自己的空间（归属锚点） | 微信"我" / 游戏"回城" |
| **探索发现** | 推荐流中上划到顶→进入"探索"视图 | 主动搜索/分类浏览 | 抖音"搜索"页 |
| **消息** | 右上角消息铃铛（空间内常驻） | 社交互动通知 | 抖音消息Tab |
| **空间预览卡片流** | 退出当前空间→在过渡层中横滑 | 快速浏览不进入，类似"扫一眼" | 微信朋友圈"扫一眼" |

## 三、核心交互设计

### 3.1 "刷空间"——索拉的核心交互循环

这是索拉最重要的交互模式，对标抖音的"上划"，但维度更高：

```
          ┌──────────────────────────────────────┐
          │                                      │
          │   ① 进入空间                          │
          │   流式加载动画（<2秒）                  │
          │   从卡片展开为全屏3D空间                │
          │         │                            │
          │         ▼                            │
          │   ② 空间内自由体验                     │
          │   第一人称视角漫游 + 与虚拟人互动        │
          │   3-5分钟（典型停留时长）               │
          │         │                            │
          │         ▼                            │
          │   ③ 触发退出（主动或自然衰减）           │
          │   上划退出 / 转身离开 / 停留超时         │
          │         │                            │
          │         ▼                            │
          │   ④ 退出过渡动画                       │
          │   空间缩小为卡片 + 预览下一个空间         │
          │   ← 这是"刷"的爽点时刻                 │
          │         │                            │
          │    ┌────┴────┐                        │
          │    ▼         ▼                        │
          │  进入下一个  横滑浏览                    │
          │  （默认行为）（探索行为）                 │
          │                                      │
          └──────────────────────────────────────┘
```

#### 3.1.1 进入空间的设计

| 进入场景 | 交互方式 | 动效设计 | 加载期间 |
|---------|---------|---------|---------|
| 从预览卡片进入 | 点击卡片→卡片展开铺满屏幕→空间首帧出现 | 卡片Scale+Position动画，0.5秒过渡 | 空间加载进度环（环形，置于卡片中央） |
| 从分享链接进入 | 点击链接→App启动→直接进入 | 品牌Logo+Slogan过渡→空间加载 | 品牌动画+加载进度 |
| 从"我的空间"进入 | 点击→直接进入（已在后台预加载） | 无过渡，瞬时进入 | 无需加载动画 |
| 退出后自动进入下一个 | 退出动效完成后自动触发 | 上一个空间缩小+下一个空间卡片放大 | 退出动画完成后0.3秒自动开始加载 |

#### 3.1.2 空间内交互——空间HUD（Head-Up Display）

空间HUD是浮于3D空间之上的极简UI层，设计原则是"最小遮挡、最轻存在感"：

```
┌─────────────────────────────────────────┐
│  ← 返回/退出     空间名称     [🔔] [···] │  ← 顶部栏（半透明，5秒后自动隐藏）
│                                          │
│                                          │
│              3D 空间区域                  │
│          （占据全屏，可360°旋转视角）       │
│                                          │
│          ┌─────────┐                     │
│          │ 虚拟人A  │   ← 可点击互动       │
│          │ 💬"嗨!"  │                     │
│          └─────────┘                     │
│                              ▲           │
│                      ┌───────┘           │
│                      │ 虚拟人B           │
│                      │ 正在走近你→       │
│                      └───────            │
│                                          │
│  ┌────────────────────────────────┐      │
│  │  "你刚才说的那个地方..."         │      │  ← 语音/文字输入区（底部浮层）
│  │  [🎤按住说话] [⌨键盘]           │      │
│  └────────────────────────────────┘      │
│                                          │
│  [👤好友] [🎬动作] [👁视角] [🚪退出]      │  ← 底部快捷栏
└─────────────────────────────────────────┘
```

**空间HUD的自动隐藏规则**：
- 进入空间3秒后，顶部栏淡出（opacity→0）
- 用户无操作10秒后，底部快捷栏半透明（opacity→30%）
- 任何点击/滑动操作立即恢复全部UI
- 虚拟人主动对话时顶部栏短暂显示（提示"XX正在和你说话"）

#### 3.1.3 退出空间的设计

退出是"刷空间"循环中最关键的设计环节——它不仅是离开，更是进入下一个空间的"邀请"：

| 退出方式 | 触发动作 | 视觉反馈 | 下一个空间推送 |
|---------|---------|---------|--------------|
| **上划退出** | 从底部向上滑动（对标抖音上划） | 空间整体向上缩小为卡片，露出下方下一个空间预览 | 自动展示推荐的下一个空间 |
| **转身离开** | 连续向一个方向"走"到空间边缘（适合沉浸用户） | 空间边界泛起涟漪→空间缩小→预览卡片 | 自动展示推荐的下一个空间 |
| **主动关闭** | 点击顶部X/退出按钮 | 空间缩小为卡片 | 展示推荐的下一个空间 |
| **停留超时** | 在同一个位置静止>5分钟（系统判定"可能离开了"） | 虚拟人提示"你去忙了吗？"→缩略模式 | 不自动推送 |

> **关键设计**：无论哪种退出方式，退出动画的最后0.3秒都是**下一个空间预览卡片的渐入**——用户退出动作的终点恰好是下一个空间进入动作的起点。这模仿了抖音"刷完一条自动播放下一"的无缝衔接。

#### 3.1.4 空间预览卡片设计

空间预览卡片是索拉介于"刷视频"和"进入空间"之间的关键过渡形态：

```
┌──────────────────────────────────────┐
│                                      │
│    ┌────────────────────────┐        │
│    │                        │        │
│    │   3D动态预览            │        │
│    │   （空间实时渲染快照，   │        │
│    │    非静态图片）          │        │
│    │                        │        │
│    │   ┌─┐                  │        │
│    │   │👤│ × 3 位虚拟人在线  │        │
│    │   └─┘                  │        │
│    │                        │        │
│    └────────────────────────┘        │
│                                      │
│    夜空咖啡馆 ☕                       │
│    一个可以在星空下聊天的日式咖啡馆      │
│                                      │
│    👤 2人在场  🕐 平均停留4分钟         │
│    ❤️ 1.2k  💬 386                  │
│                                      │
│    [进入空间]                          │
│                                      │
│    ── 上划查看下一个 ──                │  ← 替代"上划"
└──────────────────────────────────────┘
```

| 卡片元素 | 数据来源 | 更新频率 | 设计目的 |
|---------|---------|---------|---------|
| 3D动态预览 | 实时渲染快照（每5秒刷新） | 进入卡片视图时抓取一帧+定时刷新 | 让用户"看到空间里正在发生什么" |
| 虚拟人在线数 | 实时数据 | 实时 | 传达"这里有人，不冷清" |
| 空间标题+一句话描述 | 空间主设置 | 空间主更新时 | 让用户快速判断"这个空间是什么" |
| 在场人数 | 实时数据 | 实时 | 社交证明——"有人气的地方更好" |
| 平均停留时长 | 聚合数据 | T+1更新 | 质量信号——"别人愿意待的地方值得去" |
| 点赞/评论数 | 实时数据 | 实时 | 社会化认可 |

### 3.2 虚拟人交互设计

#### 3.2.1 对话交互模式

虚拟人交互有三种模式，适配不同用户状态：

| 模式 | 触发条件 | 交互方式 | 适用场景 |
|------|---------|---------|---------|
| **主动招呼** | 用户进入空间 / 长时间沉默 | 虚拟人主动说话+走近用户 | "被看见"的爽点时刻 |
| **被动回应** | 用户主动说话/点击虚拟人 | 虚拟人做出口头+肢体回应 | 日常聊天 |
| **环境陪伴** | 用户没有互动意图，只是在空间中待着 | 虚拟人做背景行为（看书/弹琴/浇花），偶尔看用户一眼 | 安静陪伴——最高级的在场感 |

#### 3.2.2 虚拟人视觉表达系统

| 表达维度 | 轻量版（技术验证期） | 完整版（产品验证期） |
|---------|------------------|-------------------|
| **面部表情** | 5种基础表情（微笑/惊讶/困惑/难过/平静） | 15+种精细表情+微表情过渡 |
| **口型同步** | 基于音量的简单开合 | 基于音素级识别的精细口型 |
| **身体语言** | 3种预设肢体动作（点头/挥手/歪头） | 20+种动作+动作平滑过渡 |
| **空间移动** | 固定位置，转头追踪用户 | 在空间中自由走动、靠近/远离用户 |
| **眼神追踪** | 注视用户（无追踪） | 视线跟随用户位置，偶尔扫视环境 |
| **情感可视化** | 名字旁的表情图标 | 粒子特效（开心=周围飘花瓣/生气=冒火苗） |

#### 3.2.3 虚拟人对话HUD

```
                     ┌─────────────────────┐
                     │  虚拟人：小雨        │
                     │  😊 好感度 Lv.4      │
                     │  "你昨天说喜欢咖啡"   │  ← 记忆调用提示
                     └──────┬──────────────┘
                            │
                     ┌──────▼──────────────┐
                     │  "我记得！所以你      │
                     │   今天想喝什么？"     │
                     └─────────────────────┘
                            │
              ┌─────────────┼─────────────┐
              ▼             ▼             ▼
         [文字回复]    [🎤语音]     [😊表情回应]
```

**对话气泡的设计规则**：
- 虚拟人的对话气泡位于其头顶上方，带轻微弹性动画（pop-in）
- 气泡背景色=虚拟人主题色（随情感微调：开心偏暖/失落偏冷）
- 用户自己的对话气泡从底部浮起
- 历史对话气泡随时间淡出（5秒后半透明，15秒后消失）——保持空间视觉干净

#### 3.2.4 虚拟人"惊喜时刻"的交互触发

| 惊喜类型 | 触发条件（系统侧） | 用户感知 | 频率控制 |
|---------|-----------------|---------|---------|
| 记住你 | 跨会话检测到用户提及过相同话题 | "咦，你上次也说过你喜欢这个..." | 每3-5次对话触发1次 |
| 毒舌反差 | 对话氛围轻松+好感度>3级+随机概率 | 突然一句调侃/吐槽 | 每10-15分钟最多1次 |
| 主动提问 | 检测到用户犹豫/沉默+未开启话题 | "我问你一个问题..." | 每次会话1-2次 |
| 小情绪 | 用户提到其他虚拟人/用户时 | 语速变化+表情变化 | 偶尔（不超过每日2次） |

> **关键设计**：惊喜的触发必须"不可预测"——如果用户能猜到虚拟人接下来会说什么，可变奖励机制就失效了。惊喜频率控制在"低于用户预期但高于用户感知阈值"的区间。

### 3.3 空间编辑交互设计

#### 3.3.1 空间编辑模式切换

```
[正常浏览模式]  ──点击"编辑"按钮（扳手图标）──▶  [编辑模式]

[编辑模式]特点：
• 摄像机切换为俯视角（上帝视角）
• 显示网格参考面
• 底部出现编辑工具栏
• 顶部显示"完成"/"撤销"/"重做"
• 空间中的虚拟人进入"待机状态"（坐下/站立不动）
```

#### 3.3.2 AI辅助搭建向导

这是降低UGC门槛的核心交互——用户用自然语言描述想要的空间，AI生成：

```
┌─────────────────────────────────────────┐
│  🏗 创建你的第一个空间                    │
│                                          │
```

## 四、视觉设计语言

### 4.1 设计定位

索拉的设计语言定义为 **"有机数字主义"（Organic Digitalism）**——在冰冷的数字渲染世界中创造有温度的存在体验。

> 直接参考：Apple visionOS 的空间设计哲学（玻璃材质+系统字体+深度层次），但融入更多"人情味"元素——虚拟人的手绘感头像、温暖色调、不规则有机形状。本质差异化：visionOS为"AR混合现实"设计，索拉为"纯虚拟的社交空间"设计——前者追求"融入物理世界"，后者追求"创造一个想去的新世界"。

### 4.5 动效设计原则

| 原则 | 说明 | 时长参考 |
|------|------|---------|
| **空间过渡** | 进入/退出空间的动效应有"穿越感"——不是简单的缩放，而是带有深度变化的透视动画 | 400-600ms |
| **虚拟人反馈** | 虚拟人的肢体反馈（点头/歪头）应有自然的缓入缓出，不能"机器人式" | 300-800ms（取决于动作幅度） |
| **UI浮现** | 空间内UI应以"从无到有"的淡入方式出现，而非突然出现 | 200-300ms |
| **列表滚动** | 预览卡片应有惯性+弹性边界，增强物理感 | 阻尼系数0.95 |
| **通知** | 重要通知使用呼吸灯效，非重要通知使用静默出现 | 1500ms周期 |

#### 关键动效规格

**① 进入空间动效**：
```
阶段1（0-200ms）：预览卡片放大至全屏，同时卡片四角的圆角渐变为直角
阶段2（200-400ms）：2D卡片"裂开"露出3D空间，空间从中心向外扩散式渲染
阶段3（400-600ms）：3D空间完成加载，第一帧出现，UI元素淡入（opacity 0→1）
```

**② 退出→推送下一个动效**：
```
阶段1（0-200ms）：空间缩小为卡片，向屏幕上方移动，同时下一个空间卡片从下方滑入
阶段2（200-400ms）：当前空间卡片缩小到30%大小，停在上方；下一个空间卡片停在中部
阶段3（400-500ms）：下一个空间卡片高亮闪烁（光晕扫过），提示"点击进入"
```

**③ 虚拟人主动走近**：
```
0-600ms：虚拟人转身面向用户
600-1200ms：开始走近（步行动画+距离缩短）
1200-1500ms：停在"对话舒适距离"（1.5米），头微偏→做出问候动作
```

## 六、微交互设计

### 6.1 "被看见"的微交互集合

这些都是索拉区别于所有竞品的体验细节：

| 时刻 | 微交互 | 设计意图 |
|------|--------|---------|
| **进入空间** | 最近的虚拟人转头看向你，头顶冒出一个"👋"气泡 | "你被注意到了"——这是"在场感"的第一个信号 |
| **被认出** | 虚拟人说"嗨[你的名字]"时，名字用专属颜色+轻微放大动画 | 强化"被记住"的爽感 |
| **被长时间注视** | 如果你持续停留在一个虚拟人面前>3秒，它会做出"被盯着有点害羞"的微动作 | 让用户感觉到虚拟人"也在感知你" |
| **离开时** | 虚拟人抬头看向你离开的方向，停留1秒后低头继续自己的事 | "你的离开被注意到了"——反面增强在场感 |
| **再次回来** | 虚拟人会说"你回来啦"（距离上一次<30分钟）/ "好久不见"（>1天） | 时间感知让"关系"显得真实 |
| **送出的礼物被"使用"** | 如果你送虚拟人的装饰品被它"戴上"，你会收到通知："小雨戴上了你送的贝壳项链" | 让付费不只是"购买"而是"关系投资" |

### 6.2 加载与过渡微交互

| 场景 | 微交互 | 技术实现 |
|------|--------|---------|
| 空间加载中 | 加载环由暗到亮旋转，且旋转速度随加载进度加快 | CSS动画+进度绑定 |
| 预加载命中 | 加载环瞬间跳满+轻微弹跳→直接进入 | 速度曲线: ease-out-bounce |
| 网络较慢（>5秒） | 加载环旁边出现一句随机文案："正在为你准备空间..." / "虚拟人正在整理房间..." | 减轻等待焦虑 |
| 加载失败 | 加载环变为红色+裂开动画→"加载失败，轻触重试" | 将错误状态情感化 |

### 6.3 情感反馈设计

| 用户行为 | 反馈 | 强度 | 频率限制 |
|---------|------|------|---------|
| 虚拟人说出让你惊喜的话 | 屏幕边缘泛起微弱的温暖橙光晕 | 中等 | 每次会话1-2次 |
| 完成一个成就 | 成就徽章从屏幕中央"炸开"+粒子飘落 | 强 | 每次成就触发 |
| 收到第一个访客 | 屏幕边缘金色粒子+特殊提示音+推送通知 | 强 | 仅首次 |
| 与虚拟人好感度升级 | 虚拟人周围短时爱心粒子+升级音效 | 中等 | 每级1次 |
| 空间被推荐到首页 | 屏幕顶部横幅"🎉 你的空间入选今日推荐" | 强 | 每日最多1次 |
| 被布道者身份认证 | 专属动效（索拉紫光柱从天而降）+ 勋章展示 | 很强 | 仅1次 |

## 七、音效设计

### 7.1 音效设计原则

> 索拉的音效不应是传统App的"叮咚"声，而应该让用户感觉到"这是一个有生命的世界在回应我"。音效设计参考原神的UI音效哲学——每个声音都在讲述这个世界的材质和情绪。

| 音效类别 | 设计方向 | 参考 |
|---------|---------|------|
| **空间过渡** | 空间感音效——像穿过一层薄膜/水面的声音 | 游戏传送到新区域的音效 |
| **虚拟人互动** | 温暖、有机——类似风铃/水晶碰撞/木琴 | 《塞尔达传说》NPC互动音效 |
| **成就/奖励** | 上升感的音阶——由低到高的音程 | 《原神》获得物品的音效 |
| **通知** | 柔和不刺耳——类似远处传来的钟声/共鸣 | iOS 静音模式的触感反馈思路 |
| **UI操作** | 微弱但可感知——类似纸张翻动/水滴 | Material Design音效哲学 |

### 7.2 关键音效清单

| 音效ID | 场景 | 描述 | 时长 |
|--------|------|------|------|
| sfx_enter_space | 进入空间 | 空间展开声+细微的环境音引入 | 1.5秒 |
| sfx_exit_space | 退出空间 | 空间收缩声+轻微"嗖"过渡 | 0.8秒 |
| sfx_avatar_greet | 虚拟人打招呼 | 温暖的风铃声 | 0.5秒 |
| sfx_avatar_surprise | 虚拟人惊喜 | 上升的木琴音阶 | 0.8秒 |
| sfx_message_receive | 收到消息 | 远处传来的钟声 | 0.3秒 |
| sfx_like | 被点赞 | 轻铃+微震动反馈 | 0.2秒 |
| sfx_achievement | 成就解锁 | 上升音阶+大共鸣 | 1.5秒 |
| sfx_level_up | 升级 | 三连音上升 | 1.0秒 |
| sfx_gift_send | 送出礼物 | 包装拆开声+喜悦感音效 | 1.0秒 |
| sfx_swipe_card | 横滑卡片 | 纸张滑动声 | 0.15秒 |

## 九、空状态与错误状态设计

### 9.1 空状态

| 场景 | 空状态设计 | 引导行动 |
|------|-----------|---------|
| 新用户首次进入App | 光之门动画 → "欢迎来到索拉，一个你能被看见的地方" → 引导进入第一个推荐空间 | "开始探索"按钮 |
| 我的空间为空 | 未搭建的空间显示为"虚空"（星空底+一个光点代表你） → "在这里创造你的第一个空间" | "AI帮我搭建" / "从模板开始" |
| 还没有虚拟人 | 一个空椅子+光晕 → "邀请你的第一个虚拟人朋友" | "添加虚拟人" |
| 探索页无结果 | "这个领域还没有人探索过，成为第一个吧" | "创建空间"按钮 |
| 消息为空 | 一只虚拟人猫在睡觉 → "你的虚拟人朋友们在等你开口" | "去看看推荐空间" |
| 收藏为空 | 一个空的展示柜 → "遇到喜欢的空间，点❤️收藏" | "去探索" |

### 9.2 错误状态

| 场景 | 错误提示 | 恢复操作 |
|------|---------|---------|
| 网络断开 | "存在网络暂时中断..." + 离线模式激活（可访问已缓存空间） | 自动重连，手动"重试" |
| 空间加载失败 | "这个空间暂时无法进入，先看看别的？" + 自动推送下一个空间 | 自动切换 |
| 虚拟人推理服务不可用 | 虚拟人显示"💤"状态 → "小雨正在休息，你可以逛逛空间等她" | 自动恢复（30秒轮询） |
| 支付失败 | "交易未完成，已为你保留物品" + 失败原因 | "重试" / "更换支付方式" |
| 发布空间失败 | "发布遇到了一点问题，你的空间已自动保存" | "重试发布" / "继续编辑" |

## 十、设计待验证假设

以下设计决策需要在原型中通过用户测试验证：

| 编号 | 假设 | 验证方法 | 成功标准 |
|------|------|---------|---------|
| D-H1 | 用户理解的"上划退出→下一个空间"流畅度不低于抖音的上划刷视频 | A/B测试：退出动效速度200ms/400ms/600ms | 退出后继续进入下一个空间的比例>70% |
| D-H2 | 空间预览卡片的3D动态预览比静态封面图带来更高的点击进入率 | A/B测试：动态vs静态预览 | 进入率提升>30% |
| D-H3 | 自动隐藏HUD比常驻HUD带来更长的停留时间 | A/B测试：自动隐藏vs常驻 | 停留时长提升>20% |
| D-H4 | 虚拟人的主动招呼（vs被动回应）是"再次进入"的最强驱动因子 | 对照实验：主动型虚拟人vs被动型虚拟人 | 次日留存差异>10% |
| D-H5 | "对话式空间搭建"的完成率高于传统编辑器 | A/B测试：AI向导vs手动编辑器 | 首次搭建完成率>80% vs >40% |
| D-H6 | 退出时推送下一个空间的动画（缩小+预览卡片）是"继续刷"的充分驱动 | 对照实验：有推送动画vs退出到列表页 | 继续消费比例>60% vs <30% |
