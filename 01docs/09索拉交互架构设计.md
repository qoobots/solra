# 索拉（Solra）交互架构设计文档

> 从"刷屏幕"到"进入空间"的交互范式跃迁——索拉的交互流程、状态机、手势规范、组件交互行为与可用性工程

---

## 一、交互设计总纲

### 1.1 文档定位

本文档是 `05索拉产品设计.md` 的交互层深化，专注于**可落地的交互规格**。`05` 回答了"空间HUD长什么样"（What），本文档回答"用户怎么操作、系统怎么响应、状态怎么转换"（How）。

```
文档层级关系：
┌─────────────────────────────────────────────────────────┐
│  05 索拉产品设计    → 信息架构 + 视觉语言 + 核心交互概念   │
│  09 索拉交互设计    → 交互流程 + 状态机 + 手势规范 + 组件  │  ← 本文档
│  07 索拉应用架构    → DDD + 微服务 + 事件驱动             │
│  08 索拉技术架构    → 技术栈选型 + 工程规范               │
└─────────────────────────────────────────────────────────┘
```

### 1.2 交互设计原则

| 原则 | 说明 | 验收标准 |
|------|------|---------|
| **即时响应** | 任何用户操作的视觉反馈不超过100ms | 点击→反馈<100ms，加载→骨架屏<200ms |
| **容错优先** | 误操作可撤销，危险操作二次确认 | 删除/退出均有确认步骤和撤销窗口 |
| **渐进披露** | 复杂功能分层展示，不一次性暴露所有选项 | 空间内默认仅显示4个核心按钮 |
| **物理直觉** | 交互遵循物理世界的隐喻（惯性、弹性、重力感） | 卡片滚动有惯性+边界弹性 |
| **一致性** | 相同操作在任何场景下产生相同反馈 | 全App上划=退出/返回上一级 |
| **可预测** | 用户能预判操作的后果 | 按钮文案=操作结果（不说"确定"，说"进入空间"） |

### 1.3 交互度量指标

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 首次进入空间耗时（含加载） | <2秒 | 端侧埋点 |
| 退出→看到下一个空间预览 | <500ms | 端侧埋点 |
| 虚拟人首Token响应 | <1.5秒（端侧<500ms） | 端侧埋点 |
| 空间内手势误触率 | <3% | 埋点统计 |
| 空间编辑首次完成率 | >80%（AI向导模式） | 埋点统计 |
| 用户无操作→HUD自动隐藏 | 10秒 | 计时器触发 |
| 任何操作→HUD完全恢复 | <200ms | 计时器触发 |

## 二、全局交互系统

### 2.1 全局手势规范

索拉采用统一的手势语义层，确保同一手势在App内任何场景下的行为一致：

#### 2.1.1 手势优先级与冲突解决

```
手势冲突解决优先级（高→低）：
┌─────────────────────────────────────────────┐
│  L1: 系统手势（控制中心/通知栏/返回Home）      │
│  L2: 索拉全局手势（上划退出/右划返回）         │
│  L3: 空间内手势（视角旋转/物体点击）           │
│  L4: 浮层内手势（面板滑动/列表滚动）           │
│  L5: 编辑手势（拖拽/旋转/缩放）               │
└─────────────────────────────────────────────┘
```

#### 2.1.2 全局手势总表

| 手势 | 空间浏览模式 | 空间编辑模式 | 预览卡片流 | 浮层面板 | 全局含义 |
|------|------------|------------|-----------|---------|---------|
| **单指点击** | 选择/确认/点击物体 | 选中物品 | 进入空间 | 点击选项/按钮 | 确认/选择 |
| **单指横滑** | 旋转视角（左右） | 平移视角 | 横滑切换卡片 | — | 水平浏览 |
| **单指竖滑** | 调整仰角（上下） | 平移视角 | 上划=下一个卡/下划=回当前空间 | 面板内滚动 | 垂直滚动/浏览 |
| **从底部上划** | 退出空间 | 退出编辑（提示保存） | 退出卡片流 | 关闭面板 | 退出/返回 |
| **从左侧右划** | — | — | — | — | 返回上一级 |
| **从右侧左划** | 打开侧边栏 | — | 打开侧边栏 | — | 打开侧边栏 |
| **双指缩放** | —（空间内禁止缩放） | 缩放选中物品 | — | — | 放缩操作 |
| **双指旋转** | — | 旋转选中物品 | — | — | 旋转操作 |
| **长按（>500ms）** | 查看物体/虚拟人详情 | 触发删除确认 | 收藏/分享空间 | 触发更多菜单 | 查看详情/更多 |
| **双击** | 重置视角 | 聚焦选中物品 | — | — | 重置/聚焦 |
| **三指点击** | — | — | — | — | 截图分享（保留） |
| **摇动设备** | — | 撤销最近操作 | — | — | 撤销 |

#### 2.1.3 手势触发阈值

| 手势 | 最小位移 | 最小速度 | 最大时间 | 说明 |
|------|---------|---------|---------|------|
| 上划退出 | 120pt（屏幕底部起） | 300pt/s | — | 超过阈值即进入退出动效 |
| 横滑切卡片 | 60pt | 200pt/s | — | 超过阈值自动吸附下一张 |
| 长按 | — | — | 500ms | 首次500ms后触发，持续按每300ms递增强度 |
| 双击 | — | — | 300ms内两次点击 | 两次点击间隔<300ms |
| 双指缩放 | 10pt（初始距离） | — | — | 距离变化>10pt触发 |
| 边缘右划返回 | 80pt（从左边缘） | 250pt/s | — | 从左边缘0-20pt区域开始 |

| **我的空间为空** | 星空底+一个光点(代表你) | "AI帮我搭建" / "从模板开始" | 空白中的可能性 |
| **还没有虚拟人** | 空椅子+光晕漂浮 | "添加虚拟人" | 等待的感觉 |
| **探索无结果** | 探索者剪影+文字 | "创建空间"按钮 | 开拓者的召唤 |
| **消息为空** | 虚拟人猫在睡觉 | "去看看推荐空间" | 慵懒的期待 |
| **收藏为空** | 空的展示柜 | "去探索" | 收藏家的起点 |
| **好友列表为空** | 两个光球在远处 | "在空间中认识新朋友" | 社交的起点 |

### 6.2 错误状态处理

| 场景 | 错误视觉 | 用户行动 | 自动恢复 | 降级方案 |
|------|---------|---------|---------|---------|
| **网络断开** | 顶部红色横幅"网络中断" + 离线图标 | — | 每5秒自动重连 | 可浏览已缓存空间 |
| **空间加载失败** | 卡片变灰+"加载失败" + 重试按钮 | 点重试/划走 | 自动推送下一个 | 跳过失败空间 |
| **虚拟人服务不可用** | 虚拟人显示"💤休息中" | — | 30秒轮询恢复 | 空间仍可浏览 |
| **推荐服务不可用** | 展示热门空间兜底列表 | 下拉刷新 | 自动重试 | 热门兜底 |
| **支付失败** | Toast:"交易未完成，已保留" | 重试/换支付方式 | — | 保留订单 |
| **发布失败** | "发布遇到问题，已自动保存" | 重试/继续编辑 | — | 草稿保存 |
| **登录过期** | 全屏"需要重新登录" | 点击重新登录 | — | 缓存内容只读 |
| **存储空间不足** | Toast:"存储空间不足" | 清理缓存按钮 | — | 降低资产质量 |

### 6.3 边界状态

| 场景 | 交互行为 |
|------|---------|
| **列表滚动到顶** | 继续下划→下拉刷新(橡皮筋效果→加载指示器→松开触发刷新) |
| **列表滚动到底** | 继续上划→加载更多(底部转圈动画)或无更多提示 |
| **输入框为空** | 发送按钮灰色不可点击 |
| **输入框超限** | 字符计数器红色+超过限制无法输入更多 |
| **网络切换(WiFi→蜂窝)** | Toast提示"已切换到移动网络"（仅首次），继续正常使用 |
| **应用切后台** | 保留当前空间状态30秒→超时后下次打开重新进入 |
| **应用从后台恢复** | <30分钟：恢复原空间；>30分钟：虚拟人说"你回来啦"；>1小时：刷新推荐 |
| **低电量模式** | 降低3D渲染帧率(60→30fps)、关闭粒子特效、降低纹理分辨率 |
| **横竖屏切换** | 仅竖屏支持；横屏时提示"请竖屏使用索拉" |

## 八、交互设计待验证假设

| 编号 | 假设 | 验证方法 | 成功标准 |
|------|------|---------|---------|
| IX-H1 | 上划退出比点击退出按钮更自然，退出后继续进入下一个空间的比例更高 | A/B测试：仅按钮vs仅手势vs两者都支持 | 手势退出→继续刷的比例 > 按钮退出的2倍 |
| IX-H2 | 3D动态预览卡片比静态封面图的点击进入率高 | A/B测试：50%用户看到动态，50%看到静态 | 动态组进入率提升>30% |
| IX-H3 | HUD自动隐藏提升停留时长和沉浸感 | A/B测试：自动隐藏vs常驻 | 停留时长提升>20%，沉浸感评分提升>15% |
| IX-H4 | 虚拟人主动对话是"再次进入"的最强驱动因子 | 对照实验：主动型vs被动型虚拟人 | 次日留存差异>10% |
| IX-H5 | AI向导空间搭建完成率高于传统编辑器 | A/B测试：AI向导vs传统编辑器 | AI组首次完成率>80%，传统组>40% |
| IX-H6 | 退出时推送下一个空间预览动效是持续消费的充分驱动 | 对照实验：推送动画vs退出到列表 | 推送组继续消费比例>60% |
| IX-H7 | 语音输入使用率>50%（在对话场景中） | 埋点统计 | 对话场景语音使用率>50% |
| IX-H8 | 触觉反馈显著提升"爽点"感知（通过满意度评分） | A/B测试：有触觉vs无触觉 | 满意度评分提升>15% |
| IX-H9 | 空间编辑撤销功能显著降低编辑放弃率 | 埋点统计编辑会话 | 有撤销功能的编辑完成率>70% |
| IX-H10 | 社交信号系统降低多人空间中的社交焦虑 | 用户访谈+埋点（信号使用率） | 使用信号后对话发起率>30% |

## 十、附录

### 10.1 交互动效命名规范

```
交互事件命名：domain-action-state
  例：space-enter-transition  /  avatar-greet-popup  /  card-swipe-snap

动效参数命名：property-element-duration
  例：opacity-hud-fadeout-300ms  /  scale-bubble-popin-300ms

触觉反馈命名：haptic-scene-intensity
  例：haptic-btn-tap-light  /  haptic-achievement-heavy
```

### 10.2 交互设计检查清单（开发交付前）

- [ ] 所有可点击元素触控区域 ≥ 44×44pt
- [ ] 点击反馈 < 100ms（视觉）+ 触觉
- [ ] 所有手势的触发阈值已定义且一致
- [ ] HUD自动隐藏/恢复逻辑完整
- [ ] 加载状态有骨架屏或进度指示
- [ ] 空状态有引导性视觉+行动按钮
- [ ] 错误状态有清晰说明+恢复操作
- [ ] 重要操作（退出/删除/发布）有确认步骤
- [ ] 可逆操作（删除/编辑）支持撤销（3-5秒窗口）
- [ ] 所有动效支持"减弱动态效果"
- [ ] VoiceOver/TalkBack核心路径可达
- [ ] 横竖屏/分屏模式/后台切换行为定义完整
