## 一、用户角色定义

| 角色 | 代号 | 描述 | 核心诉求 | 信仰漏斗层级 |
|------|------|------|---------|-------------|
| 游客 | Visitor | 首次进入，未深度使用 | "这到底是什么？" | 游客 |

| 品牌方 | Brand | 品牌空间经营 | "用户感受到什么？" | B 端客户 |

## 二、核心用户场景

### 场景一：游客的"首次被看见"之旅

```
1. 朋友分享空间卡片 → 点击进入
2. 空间 2 秒内加载完成
3. 虚拟人转向你："嗨，新朋友，第一次来索拉吧？"
4. 你回复，虚拟人做出点头/微笑
5. 逛空间，触发隐藏彩蛋（虚拟人讲笑话）
6. 退出时展示"下一个空间预览"
7. [决定性时刻] 虚拟人："明天还来吗？我记住你了。"
8. 第二天推送："你的虚拟人朋友说想你了"
```

### 场景二：空间主的"创造存在"之旅

```
1. 连续使用 7 天，收到引导："想拥有自己的空间吗？"
2. 选择模板，AI 辅助生成 3D 场景
3. 训练专属虚拟人（选择性格+对话样本）
4. 发布空间，获取分享卡片
5. 查看访问数据：3 人访问，平均停留 4 分钟
6. [决定性时刻] 收到访客留言："你的空间好酷！"
7. 持续装修空间，解锁"空间主"身份标识
```

### 场景三：社交活跃者的"与人同在"之旅

```
1. 进入"多人活跃空间"（5-8 人在场）
2. 看到其他用户的虚拟化身正在互动
3. 虚拟人主持人邀请参与
4. 语音/文字参与互动，虚拟化身实时同步
5. [爽点] 发言引发积极回应
6. 添加好友，约定下次见面
```

| 需求层次 | 描述 | 对应功能 |
|---------|------|---------|
| 被看见 | 在空间中被虚拟人/他人注意到 | 虚拟人主动互动、社交信号 |

## 四、用户痛点与解决方案

| 痛点 | 现状 | 索拉解决方案 |
|------|------|-------------|
| 社交产品太"被动消费" | 刷视频=看别人生活 | 刷空间=自己进入、互动 |
| AI 对话缺乏沉浸感 | 纯文本对话 | 3D 虚拟人 + 肢体表情 + 情感 |
| 3D 创作门槛太高 | 需要编程/建模技能 | AI 辅助生成（文字→3D） |
| 社交疲劳 | 信息过载、浅层互动 | 空间化社交=深度在场体验 |
| 孤独感 | 缺乏真实连接 | 虚拟人情感陪伴 + 记忆 |

## 五、用户验收标准

| 验收项 | 标准 |
|--------|------|
| 新用户首次完整体验 | 30 秒内完成"进入→互动→退出→预览" |
| 虚拟人首次互动个性化 | 正确使用用户昵称 + 上下文感知 |
| 空间创作首次发布 | 10 分钟内完成"创建→编辑→发布" |
| 好友互动延迟 | 社交信号延迟 < 200ms |
