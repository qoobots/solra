## 2. AUTH 认证域流程

### 2.1 用户注册流程

```
用户端 → Gateway → AUTH Service → User Aggregate
  1. 用户提交手机号/邮箱
  2. 发送验证码（SMS/Email）
  3. 用户输入验证码 + 设置密码
  4. 创建User聚合根（用户ID生成）
  5. 初始化默认Profile
  6. 分配默认角色（普通用户）
  7. 签发JWT Token
  8. 返回Token + 用户基本信息
```

### 2.2 登录认证流程

```
用户端 → Gateway → AUTH Service
  1. 提交账号密码/第三方Token
  2. 验证凭据有效性
  3. 查询用户权限（RBAC）
  4. 生成Access Token (15min) + Refresh Token (7d)
  5. 写入Session（Redis）
  6. 返回Token对
```

### 2.3 Token刷新流程

```
客户端检测Token过期 → AUTH Service
  1. 提交Refresh Token
  2. 验证Refresh Token有效性
  3. 重新签发Access Token
  4. 可选：轮换Refresh Token（防重放）
```

### 2.4 第三方登录流程

```
用户端 → 第三方OAuth → AUTH Service
  1. 用户选择第三方（微信/Apple/Google）
  2. 跳转第三方授权页
  3. 第三方回调携带Authorization Code
  4. AUTH用Code换取Access Token
  5. 查询/创建绑定用户
  6. 执行标准登录流程
```

### 2.5 权限验证流程（中间件）

```
请求 → AuthMiddleware
  1. 提取Header中的Bearer Token
  2. 验证Token签名与有效期
  3. 从Redis获取Session
  4. 提取用户ID + 角色列表
  5. 匹配API所需权限
  6. 放行/拒绝
```

## 3. AVT 虚拟人交互域流程

### 3.1 虚拟人创建流程

```
用户端 → qoobody Service → AI模型推理
  1. 用户选择基础模板/上传照片
  2. 照片质量检测（分辨率/光照/人脸检测）
  3. AI人脸特征提取（MediaPipe/InsightFace）
  4. 生成3D头部模型（FLAME参数）
  5. 体型参数配置
  6. 皮肤材质生成（GAN纹理合成）
  7. 保存Avatar聚合根
  8. 异步生成LOD多级模型
```

### 3.2 虚拟人换装流程

```
用户端 → qoobody Service
  1. 选择服装/配饰
  2. 服装物理适配（骨骼绑定）
  3. 碰撞检测（布料模拟预览）
  4. 材质替换/混搭
  5. 保存装扮快照
```

### 3.3 AI对话交互流程

```
用户输入 → Gateway → qoobrain Service
  1. 文本/语音输入
  2. ASR语音识别（设备端SLM）
  3. 上下文组装（短期记忆+长期记忆+角色设定）
  4. 意图识别（闲聊/任务/情感）
  5. 路由决策：
     - 设备端SLM（简单对话，<1B模型）
     - 云端LLM（复杂推理，7-13B模型）
  6. 生成回复文本
  7. TTS语音合成（VITS）
  8. 嘴型同步（BlendShape驱动）
  9. 情感表情生成
  10. 记忆更新（RAG向量存储）
```

### 3.4 虚拟人动作驱动流程

```
输入（文本/音频/传感器） → qoobody Service
  1. 动作意图解析
  2. 动作库检索匹配
  3. 动作混合（Motion Matching）
  4. IK逆运动学求解
  5. 物理约束检测
  6. 骨骼动画数据输出
  7. 渲染管线消费
```

### 3.5 情感识别与响应流程

```
多模态输入 → EmotionEngine
  1. 文本情感分析（NLP）
  2. 语音语调分析（声学特征）
  3. 面部表情识别（如果开启摄像头）
  4. 多模态融合打分
  5. 情感状态更新（PAD模型）
  6. 驱动表情+语气+动作
```

### 3.6 虚拟人记忆管理流程

```
对话结束 → Memory Service
  1. 对话摘要提取（LLM）
  2. 关键实体识别（NER）
  3. 向量化嵌入（text-embedding-3-small）
  4. 存入向量数据库（Milvus）
  5. 长期记忆重要性评分
  6. 低分记忆衰减/归档
```

### 3.7 多语言翻译流程

```
非中文输入 → Translation Pipeline
  1. 语言检测
  2. 翻译为中文（内部表示）
  3. AI推理（中文）
  4. 翻译回目标语言
  5. TTS目标语言合成
```

### 3.8 虚拟人行为决策流程

```
环境感知 → Behavior Tree → Action
  1. 感知环境事件（用户进入/消息/时间）
  2. 行为树评估
  3. 选择最优行为
  4. 执行行为（打招呼/推荐/引导）
  5. 行为结果反馈
```

## 4. SPC 空间消费域流程

### 4.1 空间进入流程

```
用户端 → Gateway → qooweb Service
  1. 选择空间/通过邀请链接
  2. 空间权限验证（公开/私密/付费）
  3. 加载空间元数据
  4. 分配空间实例（负载均衡）
  5. 建立WebRTC/WebSocket连接
  6. 同步空间状态（场景/物体/其他用户）
  7. 渲染空间
  8. 用户Avatar加入广播
```

### 4.2 空间内交互流程

```
用户在空间中 → 实时通信
  1. 移动（位置同步，20Hz）
  2. 语音通话（WebRTC Audio，Opus编码）
  3. 文字聊天（WebSocket消息）
  4. 3D物体交互（拾取/放置/触发）
  5. 表情动作（动画广播）
  6. 空间内导航/传送
```

### 4.3 内容推荐流流程

```
用户打开首页 → Recommendation Engine
  1. 获取用户画像（兴趣标签/行为历史）
  2. 协同过滤（相似用户偏好）
  3. 热度加权（时间衰减）
  4. 多样性保障（探索-利用平衡）
  5. 生成推荐列表
  6. 实时更新（滑动时增量加载）
```

### 4.4 空间搜索流程

```
用户搜索 → Search Service
  1. 关键词分词（中文/英文）
  2. 全文检索（Elasticsearch）
  3. 语义搜索（向量相似度）
  4. 多维度过滤（分类/标签/人数/评分）
  5. 排序（相关性+热度+个性化）
  6. 返回搜索结果
```

### 4.5 实时直播/活动流程

```
主播端 → Live Service → 观众端
  1. 主播创建直播间
  2. 视频推流（RTMP/SRT）
  3. 实时转码（H.265/H.264多码率）
  4. CDN分发
  5. 观众拉流（HLS/LL-HLS）
  6. 实时弹幕/礼物
  7. 录制存储（点播回放）
```

### 4.6 空间离开与清理流程

```
用户离开 → Session清理
  1. 用户主动离开/超时断开
  2. 广播离开事件
  3. 保存用户空间状态
  4. 释放连接资源
  5. 空间实例空闲回收
```

## 5. CRT 空间创建域流程

### 5.1 空间创建流程

```
创建者 → qoocore Service
  1. 选择空间模板（空地/房间/户外/科幻...）
  2. 配置基础属性（名称/描述/分类/标签）
  3. 设置访问权限（公开/好友/私密/付费）
  4. 设置容量上限
  5. 创建Space聚合根
  6. 初始化场景图（Scene Graph）
  7. 返回空间ID
```

### 5.2 空间编辑流程（qoodev）

```
开发者 → qoodev Desktop → qoocore API
  1. 进入编辑模式
  2. 放置3D物体（从资源库拖拽）
  3. 物体变换（位移/旋转/缩放）
  4. 材质编辑（PBR参数调整）
  5. 光照配置（方向光/点光/环境光）
  6. 碰撞体设置
  7. 脚本编写（Lua/Python行为脚本）
  8. 预览测试
  9. 发布空间
```

### 5.3 资源上传流程

```
开发者 → Asset Service
  1. 上传3D模型（GLTF/GLB/FBX）
  2. 上传纹理（PNG/JPG/KTX2）
  3. 上传音频（MP3/WAV/OGG）
  4. 资源验证（格式/大小/内容安全）
  5. 自动优化（纹理压缩/LOD生成）
  6. 存储到OSS（MinIO/S3）
  7. CDN预热
  8. 返回资源URL
```

### 5.4 空间模板发布流程

```
开发者 → Template Market
  1. 将空间打包为模板
  2. 设置模板元数据
  3. 设置定价（免费/付费）
  4. 提交审核
  5. 内容安全审核
  6. 上架/驳回
```

### 5.5 脚本调试流程

```
qoodev IDE → Script Runtime
  1. 编写行为脚本
  2. 语法检查/静态分析
  3. 挂载到空间物体
  4. 进入调试模式
  5. 设置断点
  6. 单步执行
  7. 变量监视
  8. 热重载
```

## 6. SOC 社交存在域流程

### 6.1 好友添加流程

```
用户A → SOC Service → 用户B
  1. 用户A搜索/推荐用户B
  2. 发送好友申请
  3. 推送通知给用户B
  4. 用户B接受/拒绝
  5. 建立双向好友关系
  6. 更新双方好友列表
```

### 6.2 实时状态同步流程

```
用户状态变更 → Presence Service
  1. 用户上线/离线/忙碌/勿扰
  2. 更新Redis Presence Key（TTL心跳）
  3. 广播状态变更给好友/空间内用户
  4. WebSocket推送
```

### 6.3 群组管理流程

```
创建者 → Group Service
  1. 创建群组（名称/头像/简介）
  2. 邀请成员
  3. 成员角色管理（群主/管理员/成员）
  4. 群公告/群设置
  5. 群空间关联
```

### 6.4 消息路由流程

```
发送者 → Message Service → 接收者
  1. 消息序列化
  2. 消息持久化（消息队列+数据库双写）
  3. 在线：WebSocket实时推送
  4. 离线：存储到离线消息队列
  5. 接收者上线后拉取离线消息
  6. 已读回执
```

## 7. GRW 用户增长域流程

### 7.1 新用户引导流程

```
新注册用户 → Growth Engine
  1. 兴趣标签采集（引导问答）
  2. 推荐初始空间/虚拟人模板
  3. 新手任务分发
  4. 进度追踪
  5. 任务完成奖励
```

### 7.2 邀请裂变流程

```
邀请者 → GRW Service
  1. 生成个人邀请码/链接
  2. 分享到社交平台
  3. 被邀请者注册（携带邀请码）
  4. 建立邀请关系
  5. 双方获得奖励
  6. 防刷校验
```

### 7.3 任务系统流程

```
用户 → Task Engine
  1. 任务配置（条件/奖励/周期）
  2. 用户行为事件监听
  3. 任务进度更新
  4. 条件满足判定
  5. 发放奖励

```

### 7.4 等级与成就流程

```
经验获取 → Level System
  1. 行为积分累计

  3. 成就解锁判定

  5. 升级推送
```

## 8. MON 变现域流程

### 8.1 虚拟商品购买流程

```
用户 → Store Service → Payment Gateway
  1. 浏览商品（服装/配饰/特效/空间道具）
  2. 加入购物车
  3. 选择支付方式（微信/支付宝/Apple Pay）
  4. 创建订单
  5. 调用支付网关
  6. 支付回调验证
  7. 发放商品
  8. 更新用户资产
```

### 8.2 订阅服务流程

```
用户 → Subscription Service
  1. 选择订阅方案（月度/季度/年度）
  2. 首次支付
  3. 激活会员权益
  4. 自动续费（到期前7天提醒）
  5. 续费扣款
  6. 取消订阅
```

### 8.3 创作者分成流程

```
周期结算 → Revenue Service
  1. 统计周期内创作者收入
  2. 计算平台抽成比例
  3. 生成结算单
  4. 创作者确认
  5. 打款（企业对公/个人提现）
  6. 开具发票
```

### 8.4 广告投放流程

```
广告主 → Ad Service
  1. 创建广告计划
  2. 设置定向（人群/空间类型）
  3. 设置预算与出价
  4. 提交素材审核
  5. 广告投放（RTB竞价）
  6. 曝光/点击数据回传
  7. 效果报表
```

## 9. SAF 内容安全域流程

### 9.1 内容审核流程

```
内容提交 → SAF Service
  1. 自动审核（文本/图片/3D模型/音频）
     - 敏感词过滤（AC自动机）
     - 图片鉴黄/暴恐（CNN模型）
     - 3D模型合规检测
  2. 自动判定：通过/驳回/人工复审
  3. 人工复审队列
  4. 审核结果通知
  5. 违规内容处置（删除/封禁）
```

### 9.2 用户举报流程

```
举报者 → SAF Service → 审核员
  1. 提交举报（目标/类型/描述/截图）
  2. 举报去重（相同目标合并）
  3. 优先级排序（严重程度+举报人数）
  4. 审核员处理
  5. 处置决定（警告/限制/封禁/驳回）
  6. 通知举报者处理结果
```

### 9.3 实时内容过滤流程

```
实时消息/语音 → SAF Filter
  1. 消息拦截（敏感词实时替换）
  2. 语音流实时检测（声学模型）
  3. 命中规则处理（拦截/替换/警告）
  4. 日志记录
```

```
用户 → Message Center
```

## 11. INF 基础设施域流程

### 11.1 服务注册与发现流程

```
服务启动 → Service Registry (Consul/Nacos)
  1. 服务启动时注册自身信息
  2. 定期发送心跳（TTL 30s）
  3. Gateway查询健康服务列表
  4. 故障实例自动摘除
  5. 服务下线注销
```

### 11.2 配置热更新流程

```
运维 → Config Center → 服务实例
  1. 修改配置（Web UI/API）
  2. 发布配置变更
  3. Config Center推送变更通知
  4. 服务实例拉取新配置
  5. 配置生效（无需重启）
  6. 变更审计日志
```

### 11.3 分布式链路追踪流程

```
请求进入 → Trace Pipeline
  1. Gateway生成TraceID
  2. 透传TraceID到下游
  3. 各服务记录Span
  4. Span上报到Collector
  5. 存储到Jaeger/Tempo
  6. 可视化查询
```
