## 一、文档概述

本文档定义索拉（Solra）项目的非功能性需求，涵盖性能、安全、可用性、兼容性、可维护性、可扩展性六大维度。

---

## 二、性能需求

### 2.1 响应时间

| 指标 | 技术验证期 | 产品验证期 | 规模验证期 |
|------|-----------|-----------|-----------|
| 空间加载时间 (P95) | < 5s | < 3s | < 2s |
| 虚拟人回复首 Token (P95) | < 3s | < 2s | < 1.5s |
| 端侧推理首 Token | < 500ms | < 400ms | < 300ms |
| API 响应时间 (P95) | < 200ms | < 150ms | < 100ms |
| 推荐流刷新 (P95) | < 1s | < 500ms | < 300ms |
| CDN 首包时间 (P95) | < 500ms | < 300ms | < 200ms |
| 消息投递延迟 (P99) | < 1s | < 500ms | < 200ms |

### 2.2 吞吐量

| 指标 | 目标 |
|------|------|
| API 网关 QPS | ≥ 100,000（单实例） |
| 消息队列吞吐 | ≥ 10,000,000 msg/s（成熟期） |
| GPU 推理并发 | ≥ 1,000 并发请求（单集群） |
| CDN 带宽 | ≥ 1 Tbps（规模验证期） |

### 2.3 资源消耗

| 指标 | 目标 |
|------|------|
| 客户端包体大小 | iOS/Android < 150MB, Desktop < 200MB |
| 端侧推理内存 | < 2GB（1-3B 量化模型） |
| 单个空间 GPU 内存 | < 8GB（端侧）, 按需分配（云端） |
| 微服务实例内存 | < 2GB（JVM 堆） |

## 三、可用性需求

### 3.1 可用性目标

| 服务级别 | 可用性 | 最大停机时间/年 | 说明 |
|---------|--------|----------------|------|
| 核心服务（AVT/SPC/AUTH/INF） | 99.99% | < 52 分钟 | 多活部署 |
| 支撑服务（CRT/SOC/GRW） | 99.95% | < 4.4 小时 | 主备切换 |
| 非关键服务（MON/NOT） | 99.9% | < 8.8 小时 | 单实例可接受 |

### 3.2 容灾要求

| 要求 | 目标 |
|------|------|
| RPO（恢复点目标） | < 5 分钟（数据库） |
| RTO（恢复时间目标） | < 15 分钟（核心服务） |
| 数据备份频率 | 每日全量 + 每小时增量 |
| 多活部署 | 至少 2 个可用区 |

### 3.3 降级策略

| 场景 | 降级措施 |
|------|---------|
| 云端 LLM 不可用 | 端侧 SLM 全量接管 |
| 网络断开 | 端侧推理 + 本地缓存空间 |
| GPU 集群满载 | 排队 + 免费用户限流 |
| 数据库主库故障 | 自动切换从库 |

## 四、安全需求

### 4.1 认证与授权

| 需求 | 标准 |
|------|------|
| 认证方式 | JWT 双 Token（Access 15min + Refresh 7天） |
| 多因素认证 | 异地/新设备二次验证 |
| 密码策略 | bcrypt (cost=12)，最少 8 位含大小写数字 |
| 角色权限 | RBAC 五角色（游客/使用者/创作者/管理员/超管） |
| 第三方登录 | OAuth2.0/OIDC（微信/Apple/Google/Facebook） |

### 4.2 数据安全

| 需求 | 标准 |
|------|------|
| 传输加密 | TLS 1.3（全链路） |
| 存储加密 | AES-256（KMS 托管密钥） |
| 敏感字段加密 | 手机号/身份证号应用层独立加密 |
| 日志脱敏 | 手机号→138****1234，身份证号→310***********1234 |
| 数据分类 | 公开/内部/敏感/机密 四级 |
| 数据存储 | 中国用户数据 100% 境内存储 |

### 4.3 内容安全

| 需求 | 标准 |
|------|------|
| 文本审核 | 敏感词 + 违规意图检测，F1 > 0.99 |
| 图片审核 | 多模态安全模型，延迟增加 < 200ms |
| 3D 场景审核 | 纹理 + 模型 + 文字综合检测 |
| 审核时效 | 自动审核 < 200ms，人工审核 < 4 小时 |

### 4.4 合规要求

| 法规 | 要求 |
|------|------|
| 网络安全法 | 等保 2.0 三级 |
| 个人信息保护法 | 用户可控数据范围，知情同意 |
| 数据安全法 | 数据分类分级，出境管控 |
| 未成年人保护 | 实名认证 + 防沉迷 + 消费限制 |
| PCI-DSS | 支付令牌化，不存储完整卡号 |

## 五、兼容性需求

### 5.1 平台兼容

| 平台 | 最低版本 | 推荐版本 |
|------|---------|---------|
| iOS | 17.0 | 最新 |
| Android | 12 (API 31) | 14+ |
| Windows | 10 (21H2) | 11 |
| Web | Chrome 100+, Safari 16+, Edge 100+ | 最新 |

### 5.2 设备分级

| 级别 | GPU 要求 | 体验 |
|------|---------|------|
| 旗舰 | Adreno 740+ / A17 Pro+ | 完整 PBR 渲染 + 端侧推理 |
| 中端 | Adreno 640-730 / A14-A16 | LOD 中等细节 + 端侧推理 |
| 低端 | 其他支持 Vulkan/Metal | 简化渲染 + 云端推理 |

## 七、可扩展性需求
