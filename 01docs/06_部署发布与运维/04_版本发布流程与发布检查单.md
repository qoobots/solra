# 04_版本发布流程与发布检查单

> **文档编号**：06-04 | **版本号**：v1.0 | **编制人**：DevOps组 | **状态**：已发布 | **更新日期**：2026-06-30

---

## 1. 发布流程

```
开发完成 → 代码冻结 → 创建Release分支 → 最终测试 → 审批 → 金丝雀发布 → 全量 → 打Tag
```

## 2. 发布检查单

### 开发阶段
- [ ] 所有PR已合并
- [ ] CI全部通过
- [ ] 代码冻结公告已发

### 测试阶段
- [ ] 单元测试100%通过
- [ ] 集成测试100%通过
- [ ] 回归测试100%通过
- [ ] 性能测试达标
- [ ] 安全扫描通过

### 发布阶段
- [ ] Release Notes已准备
- [ ] 数据库迁移脚本已验证
- [ ] 回滚方案已确认
- [ ] 监控告警已调整
- [ ] 值班人员已安排

### 发布后
- [ ] 监控指标正常
- [ ] 错误率无异常
- [ ] 用户反馈无重大问题
- [ ] 发布总结已完成

## 3. 发布窗口

| 环境 | 时间窗口 |
|------|----------|
| Staging | 任意时间 |
| 生产（常规） | 周四 14:00-18:00 |
| 生产（紧急） | 审批后随时 |

## 4. 版本号更新

```bash
# 更新所有模块版本号
./tools/scripts/bump-version.sh 1.1.0

# 创建Git Tag
git tag -a v1.1.0 -m "Release v1.1.0"
git push origin v1.1.0
```
