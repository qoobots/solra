## 1. 应急响应流程

```
发现 → 确认 → 定级 → 响应 → 恢复 → 复盘
```

## 2. 常见故障预案

### 数据库主库宕机
1. Patroni自动Failover → 验证新主库 → 更新连接串 → 通知团队

### K8s节点宕机
1. Pod自动调度到其他节点 → 验证服务恢复 → 检查数据完整性

### DDoS攻击
1. WAF自动拦截 → 通知安全组 → 评估影响 → 清洗流量

### 数据损坏
1. 停止写入 → 从备份恢复 → 验证完整性 → 恢复服务
