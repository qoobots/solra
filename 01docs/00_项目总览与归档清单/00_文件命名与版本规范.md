## 一、文件命名规范

### 1.1 通用格式

```
[阶段编号]_[文档名称].[扩展名]
```

### 1.2 文档命名示例

| 文档类型 | 命名示例 |
|----------|---------|
| 根级核心文档 | `01互联网创业与社会趋势分析.md` |
| 阶段目录文档 | `01_BRD业务需求说明书.md` |
| 模块级文档 | `02架构设计.md` / `03交互设计.md` |
| Proto 定义 | `{domain}/v1/{service}.proto` |
| Java 源码 | `{ClassName}.java`（大驼峰） |
| Python 源码 | `{module_name}.py`（小蛇形） |
| C++ 源码 | `{class_name}.cpp` / `{class_name}.h`（小蛇形） |

### 1.3 命名规则

1. 使用下划线 `_` 分隔各字段
2. 文档名称使用中文描述
3. 文件名中不使用空格和特殊字符（`-` 除外）
4. 代码文件遵循各自语言的社区命名规范
5. 目录名使用中文或英文，英文优先

### 1.4 代码仓库文件命名

| 语言/平台 | 文件命名 | 目录命名 |
|-----------|---------|---------|
| Java | PascalCase.java | kebab-case |
| Python | snake_case.py | snake_case |
| C++ | snake_case.cpp/.h | snake_case |
| TypeScript/Vue | PascalCase.vue / camelCase.ts | kebab-case |
| Kotlin | PascalCase.kt | camelCase |
| Swift | PascalCase.swift | PascalCase |
| Rust | snake_case.rs | snake_case |
| Proto | snake_case.proto | snake_case |

## 二、版本号规范

### 2.1 文档版本管理

| 版本状态 | 版本号示例 | 说明 |
|----------|-----------|------|
| 草稿 | v0.1 ~ v0.9 | 编写中，未评审 |
| 评审中 | v1.0-rc1 | 提交评审 |
| 正式发布 | v1.0 | 评审通过，正式归档 |
| 小幅修订 | v1.1 | 内容修订，不改变结构 |
| 重大变更 | v2.0 | 结构或核心内容重大调整 |

### 2.2 代码版本管理

遵循语义化版本（SemVer 2.0）：`v{MAJOR}.{MINOR}.{PATCH}`

- **MAJOR**：不兼容的 API/Proto 修改
- **MINOR**：向下兼容的新增功能
- **PATCH**：向下兼容的 Bug 修复

### 2.3 Proto 版本管理

```
contracts/proto/solra/{domain}/v1/{service}.proto
```

- 通过目录 `v1`/`v2` 管理不兼容变更
- 同版本内通过新增字段实现向后兼容
- 字段编号永不重用

### 2.4 变更记录规范

每个文档应在末尾或专门的变更记录章节记录：

```markdown
| 版本 | 日期 | 变更人 | 变更内容 |
|------|------|--------|---------|
| v1.0 | 2026-06-30 | xxx | 初始版本 |
| v1.1 | 2026-07-15 | xxx | 新增第X章 |
```

## 三、Git 提交规范

### 3.1 提交信息格式

采用 Conventional Commits 规范：

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

### 3.2 Type 类型

| Type | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat(avt): 实现虚拟人对话流式响应` |
| `fix` | Bug 修复 | `fix(spc): 修复空间加载超时问题` |
| `docs` | 文档变更 | `docs: 完善归档目录文档` |
| `style` | 格式调整 | `style: 统一缩进为 4 空格` |
| `refactor` | 代码重构 | `refactor(auth): 重构 Token 刷新逻辑` |
| `test` | 测试相关 | `test(avt): 新增情感模型边界测试` |
| `chore` | 构建/工具 | `chore: 升级 Gradle 到 8.7` |
| `perf` | 性能优化 | `perf(core): 优化 Vulkan 渲染管线` |

### 3.3 Scope 范围

按子项目前缀：`avt`/`spc`/`crt`/`soc`/`grw`/`mon`/`auth`/`saf`/`not`/`inf`

## 四、分支命名规范

| 分支类型 | 命名格式 | 示例 |
|----------|---------|------|
| 功能分支 | `feature/{subproject}-{desc}` | `feature/avt-emotion-engine` |
| 修复分支 | `fix/{subproject}-{desc}` | `fix/spc-loading-timeout` |
| 热修复 | `hotfix/{desc}` | `hotfix/auth-token-leak` |
| 发布分支 | `release/v{version}` | `release/v1.0.0` |
