# 00_文件命名与版本规范

## 一、文件命名规范

### 通用格式
```
[阶段编号]_[文档名称]_[版本号].[扩展名]
```

### 示例
- `01_BRD业务需求说明书_v1.0.docx`
- `02_系统架构设计说明书_v2.1.md`
- `05_性能测试报告_v1.0.pdf`

### 命名规则
1. 使用下划线 `_` 分隔各字段
2. 文档名称使用中文描述
3. 版本号采用 `v主.次` 格式
4. 文件名中不使用特殊字符

## 二、版本号规范

### 文档版本管理
| 版本状态 | 版本号示例 | 说明 |
|----------|-----------|------|
| 草稿     | v0.1~v0.9 | 编写中，未评审 |
| 评审中   | v1.0-rc1  | 提交评审 |
| 正式发布 | v1.0      | 评审通过，正式归档 |
| 修订     | v1.1      | 小幅修改 |
| 升级     | v2.0      | 重大变更 |

### 版本变更记录模板

| 版本号 | 日期 | 修改人 | 修改内容 | 审核人 |
|--------|------|--------|----------|--------|
|        |      |        |          |        |
