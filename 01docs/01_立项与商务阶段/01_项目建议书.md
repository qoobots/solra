## 一、项目背景

### 1.1 行业背景

陌生人社交产品遵循"网络带宽驱动内容形态升级"的演进规律：
- 3G 时代（2008）：图文社交（微博）
- 4G 时代（2013-2015）：短视频社交（快手→抖音）
- 5G 时代（2022-至今）：3D 空间社交呼之欲出

当前 5G 渗透率正处于爬坡期，类似 2014 年 4G 阶段——基础设施已就绪，但尚无匹配的杀手级应用。5G 网络在 2 秒时间窗口内可下载约 200MB 数据，这使 200-900MB 的 3D 空间实时加载成为可能。

### 1.2 市场机会

- 全球社交网络市场规模超过 2000 亿美元
- 3D 虚拟空间/元宇宙赛道 2026 年预计规模达 800 亿美元
- Roblox DAU 超 7000 万，验证了 UGC 3D 空间的用户需求
- Character.AI 验证了 AI 虚拟人情感陪伴的付费意愿
- 当前市场上缺乏将"AI 虚拟人 + 3D 空间 + 算法推荐"三者融合的产品

### 1.3 技术可行性

- **大语言模型（LLM）成熟**：Qwen2.5 等开源模型使端侧+云端混合推理成为可能
- **端侧 AI 推理普及**：llama.cpp 量化推理 + NPU 加速使手机端运行 1-3B 模型
- **3D 渲染技术进步**：Vulkan 1.3 / Metal 3 使移动端高质量 3D 渲染成为现实
- **AI 编程工具**：使自研 3D 引擎的工作量从传统 37-47 人月降至 12.5-17.6 人月

## 二、项目目标

### 2.1 战略目标

构建"下一代去中心化自主虚拟世界"——集 AI 虚拟人、3D 实时交互、算法分发能力于一体的社交平台。

### 2.2 产品目标

| 目标 | 指标 | 时间 |
|------|------|------|
| 空间进入时间 | < 2 秒（P95） | 技术验证期 |
| 虚拟人回复延迟 | < 1.5 秒（P95） | 产品验证期 |
| 用户次日留存率 | > 40% | 产品验证期 |
| 支持 DAU | 1000 万+ | 规模验证期 |
| 付费转化率 | > 10% | 规模验证期 |

### 2.3 商业目标

三层收入引擎：用户订阅（50-60%）+ 创作者经济（25-35%）+ 品牌服务（10-20%）

## 四、项目初步方案

### 4.1 技术方案概要

| 层次 | 技术选型 |
|------|---------|
| 协议定义 | Protobuf + Avro + OpenAPI 3.0 |
| 核心引擎 | C++17 + Vulkan 1.3/Metal 3 + CMake |
| 业务服务 | Java 17 + Spring Boot 3.3 + gRPC |
| AI 服务 | Python 3.11 + FastAPI + vLLM + llama.cpp |
| 客户端 | SwiftUI(iOS) / Compose(Android) / Vue3(Web) / Tauri2+Rust(Desktop) |
| 基础设施 | Kubernetes + Terraform + ArgoCD + APISIX |

### 4.2 架构方案

- DDD 领域驱动设计，10 个子项目 = 10 个限界上下文
- 微服务 + 事件驱动 + CQRS 混合架构
- 端+云混合 AI 推理（端侧 SLM 1-3B + 云端 LLM 7-13B）
- 自研轻量 3D 引擎（包体 < 15MB，GPU 利用率 100%）

### 4.3 三阶段交付计划

| 阶段 | 时间 | 目标 | 功能数 |
|------|------|------|--------|
| 技术验证期 | 第 1-2 年 | 跑通 3D 渲染+虚拟人对话+流式加载 | 46 |
| 产品验证期 | 第 3-4 年 | 验证留存驱动因子，启动 UGC | 40 |
| 规模验证期 | 第 5-7 年 | 千万 DAU，信仰体系自传播 | 11 |

## 七、风险与应对措施

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 端侧推理延迟不达标 | 核心体验崩盘 | 中 | 模型路由支持纯云端降级 |
| 3D 流式加载不稳定 | 用户跳出率高 | 中高 | 渐进加载策略，低端机轻量模式 |
| GPU 推理成本失控 | 毛利率被侵蚀 | 中高 | 端侧优先，成本感知调度，FinOps |
| 内容安全漏审 | 监管风险 | 低 | 人机双重审核，安全模型持续迭代 |
| 用户认知门槛高 | 增长缓慢 | 中 | 渐进式引导，"刷空间"类比"刷视频" |

## 八、结论与建议

索拉项目符合"5G 时代杀手应用"的历史规律窗口，技术可行性已通过 DDD 四层架构代码验证（78/78 功能就绪）。建议：

1. **立即启动** Core SDK（C++17 3D 引擎）的 CI 编译验证，这是整个技术栈的关键路径
2. **优先完成**端侧推理引擎与 3D 渲染的联调，验证"进入空间→虚拟人互动"核心体验闭环
3. **同步推进** 4 个客户端（iOS/Android/Web/Desktop）的 UI 与 Core SDK 集成
4. **分阶段投入**：技术验证期保持精干团队（20-40 人），验证核心假设后再扩编
