## 二、高风险

### R-001：端侧 3D 渲染性能不足

| 属性 | 内容 |
|------|------|
| 风险等级 | 🔴 高 |
| 影响范围 | 全平台用户体验，核心体验闭环 |
| 发生概率 | 中（40%） |
| 影响程度 | 极高——空间无法流畅渲染则产品不成立 |
| 触发条件 | 自研引擎 Vulkan/Metal 后端实现延迟或不达标 |
| 缓解措施 | 1. AI 辅助生成 GPU 后端代码（4-10x 加速比） |
|  | 2. 四级 LOD 降级（旗舰/中端/低端/极简） |
|  | 3. 低端机降级为"轻 3D"模式 |
| 应急方案 | 评估 Unity 6 作为备选引擎（包体 30-50MB，3 个月可集成） |
| 负责人 | C++ 引擎组长 |
| 跟踪频率 | 每周 |

### R-002：GPU 推理成本失控

| 属性 | 内容 |
|------|------|
| 风险等级 | 🔴 高 |
| 影响范围 | 商业模式可持续性 |
| 发生概率 | 中高（50%） |
| 影响程度 | 极高——大规模并发 GPU 成本可能吞噬毛利 |
| 触发条件 | DAU 快速增长 + 云端推理占比过高 |
| 缓解措施 | 1. 端侧 SLM 分担 50%+ 日常对话 |
|  | 2. 推理路由成本感知调度 |
|  | 3. FinOps 实时成本监控与告警 |
|  | 4. 按订阅级别分配推理资源（免费→端侧，Pro→云端） |
| 应急方案 | 限制免费用户云端推理额度 |
| 负责人 | AI 组长 |
| 跟踪频率 | 每周 |

### R-003：内容安全漏审导致监管风险

| 属性 | 内容 |
|------|------|
| 风险等级 | 🔴 高 |
| 影响范围 | 产品合规，应用商店下架风险 |
| 发生概率 | 低（15%） |
| 影响程度 | 极高——监管处罚或应用下架 |
| 触发条件 | AI 审核漏审 + 人工审核未覆盖 + 用户大规模举报 |
| 缓解措施 | 1. Qwen2.5-Guard-7B 多模态安全模型 |
|  | 2. AI 自动初筛 + 人工审核工作台双重保障 |
|  | 3. 用户举报→审核→处理→申诉全闭环 |
|  | 4. 安全模型持续迭代训练 |
| 应急方案 | 紧急下线问题内容，全量回溯审核 |
| 负责人 | SAF 负责人 |
| 跟踪频率 | 每日 |

## 三、中风险

### R-004：3D 空间流式加载不稳定

| 风险等级 | 🟡 中 | 概率 | 中（40%） |
|---------|------|------|----------|
| 影响 | 用户跳出率高，留存差 |
| 缓解 | 渐进加载（白模→纹理→模型），CDN 边缘节点，预测性预加载 |
| 应急 | 降低空间复杂度，预缓存热门空间 |

### R-005：WebRTC 连通率低

| 风险等级 | 🟡 中 | 概率 | 中（35%） |
|---------|------|------|----------|
| 影响 | 多人社交体验受损 |
| 缓解 | TURN 中继服务器兜底，UDP→TCP 降级，国内 BGP 多线 |
| 应急 | 降级为异步社交模式（文字+表情，无实时音视频） |

### R-006：用户认知门槛高

| 风险等级 | 🟡 中 | 概率 | 中高（50%） |
|---------|------|------|----------|
| 影响 | 增长缓慢，获客成本高 |
| 缓解 | "刷空间"类比"刷视频"降低认知，渐进式引导 |
| 应急 | 先推 2D 空间预览模式，降低首次使用门槛 |

### R-007：分布式系统运维复杂

| 风险等级 | 🟡 中 | 概率 | 中（30%） |
|---------|------|------|----------|
| 影响 | 故障恢复慢，影响可用性 |
| 缓解 | K8s + ArgoCD GitOps，完善的 Runbook，自动扩缩容 |
| 应急 | 核心服务多活部署，自动故障转移 |

### R-008：核心人才流失

| 风险等级 | 🟡 中 | 概率 | 低（20%） |
|---------|------|------|----------|
| 影响 | 研发进度受阻 |
| 缓解 | 文档完备，Code Review 知识共享，关键模块至少两人掌握 |

### R-009：大模型/开源依赖变更

| 风险等级 | 🟡 中 | 概率 | 低（20%） |
|---------|------|------|----------|
| 影响 | 技术方案被迫调整 |
| 缓解 | 选用 Apache 2.0 协议的模型，抽象推理接口便于替换 |
| 应急 | 模型路由支持多模型热切换 |

### R-010：5G 渗透率不及预期

| 风险等级 | 🟡 中 | 概率 | 中低（25%） |
|---------|------|------|----------|
| 影响 | 用户基数增长受限 |
| 缓解 | WiFi 环境优先策略，空间体积分级（小型 < 50MB） |
| 应急 | 推出"轻空间"模式，4G 可流畅运行 |
