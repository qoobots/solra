## 2. 快速开始

### 2.1 克隆项目

```bash
git clone https://github.com/solra-project/solra.git
cd solra
```

### 2.2 一键启动（Docker Compose）

```bash
# 启动所有基础设施（数据库/缓存/消息队列等）
docker compose -f docker-compose.dev.yml up -d

# 验证服务状态
docker compose ps
```

### 2.3 基础设施服务

`docker-compose.dev.yml` 启动的服务：

| 服务 | 端口 | 用途 |
|------|------|------|
| PostgreSQL | 5432 | 主数据库 |
| Redis | 6379 | 缓存 |
| Kafka | 9092 | 消息队列 |
| MinIO | 9000/9001 | 对象存储/控制台 |
| Elasticsearch | 9200 | 搜索引擎 |
| Milvus | 19530 | 向量数据库 |
| Consul | 8500 | 服务发现 |
| Jaeger | 16686 | 链路追踪UI |
| Grafana | 3000 | 监控面板 |

## 3. 各模块本地运行

### 3.1 Java微服务

```bash
# 以AVT服务为例
cd ai-services/avt-service

# 编译
./gradlew build

# 运行（开发模式）
./gradlew bootRun --args='--spring.profiles.active=dev'

# 验证
curl http://localhost:8080/api/v1/avatar/health
```

### 3.2 Python AI服务

```bash
# 以qoobrain为例
cd ai-services/brain

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -e ".[dev]"

# 运行
uvicorn solra_brain.main:app --reload --port 8000

# 验证
curl http://localhost:8000/health
```

### 3.3 Vue 3前端

```bash
cd clients/web

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 访问 http://localhost:5173
```

### 3.4 C++核心引擎

```bash
cd core

# 配置
cmake -B build -DCMAKE_BUILD_TYPE=Debug

# 编译
cmake --build build -j$(nproc)

# 运行测试
cd build && ctest --output-on-failure
```

## 6. 常见问题

### Q1: Docker服务启动失败

```bash
# 检查端口占用
lsof -i :5432
lsof -i :6379

# 清理并重启
docker compose down -v
docker compose up -d
```

### Q2: Gradle依赖下载慢

```bash
# 使用阿里云镜像
# gradle.properties
systemProp.org.gradle.internal.repository.max.retries=3
```

### Q3: Python依赖冲突

```bash
# 重建虚拟环境
rm -rf venv
python -m venv venv
pip install -e ".[dev]"
```

## 7. 本地开发最佳实践

1. **每天从main拉最新代码**，避免大合并冲突
2. **功能分支存活不超过1天**
3. **提交前运行本地检查**：`make check`
4. **使用Docker Compose启动依赖服务**，避免手动安装数据库
5. **环境变量通过.env文件管理**，不提交敏感信息
