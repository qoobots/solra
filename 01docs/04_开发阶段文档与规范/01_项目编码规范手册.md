## 2. Java 编码规范

### 2.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 全小写，点分隔 | `com.solra.avatar.service` |
| 类名 | UpperCamelCase | `AvatarService`, `UserController` |
| 方法名 | lowerCamelCase | `getUserById`, `createSpace` |
| 常量 | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT`, `DEFAULT_PAGE_SIZE` |
| 变量 | lowerCamelCase | `userId`, `spaceName` |
| 测试类 | 被测类名 + Test | `AvatarServiceTest` |
| 枚举 | UpperCamelCase | `UserStatus`, `AccessType` |

### 2.2 包结构规范

```
com.solra.{domain}/
├── controller/       # REST控制器
├── service/          # 业务服务接口
│   └── impl/         # 业务服务实现
├── repository/       # 数据访问层
├── model/
│   ├── entity/       # JPA实体
│   ├── dto/          # 数据传输对象
│   ├── vo/           # 视图对象
│   └── event/        # 领域事件
├── config/           # 配置类
├── exception/        # 异常定义
└── util/             # 工具类
```

### 2.3 方法规范

```java
// ✅ 正确
public User getUserById(UUID userId) {
    return userRepository.findById(userId)
        .orElseThrow(() -> new NotFoundException("User not found: " + userId));
}

// ❌ 错误 - 方法过长
public void doEverythingAndMore(...) { /* 200行代码 */ }

// 方法长度: ≤50行
// 参数数量: ≤5个（超过用DTO封装）
// 圈复杂度: ≤15
```

### 2.4 异常处理规范

```java
// ✅ 正确 - 使用自定义业务异常
throw new NotFoundException("Space not found: " + spaceId);

// ❌ 错误 - 直接抛出通用异常
throw new RuntimeException("Space not found");

// ❌ 错误 - 吞掉异常
try { ... } catch (Exception e) {}

// ✅ 正确 - 记录并重新抛出
try { ... } catch (DataAccessException e) {
    log.error("Database error", e);
    throw new SystemException("Data access error", e);
}
```

### 2.5 Lombok使用规范

```java
// ✅ 允许的注解
@Data           // 仅用于DTO/VO
@Builder        // 建造者模式
@Slf4j          // 日志
@RequiredArgsConstructor  // 构造器注入

// ❌ 禁止的注解
@SneakyThrows   // 隐藏受检异常
@Cleanup        // 资源管理应显式try-with-resources
```

## 3. TypeScript/Vue 3 编码规范

### 3.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | kebab-case | `space-card.vue`, `use-auth.ts` |
| 组件名 | PascalCase | `SpaceCard`, `ChatPanel` |
| 函数/变量 | camelCase | `getSpaces`, `userName` |
| 类型/接口 | PascalCase | `SpaceInfo`, `UserProfile` |
| 常量 | UPPER_SNAKE_CASE | `MAX_UPLOAD_SIZE` |
| 组合式函数 | use开头 | `useAuth`, `useWebSocket` |

### 3.2 Vue 3组件规范

```vue
<!-- ✅ 正确 -->
<script setup lang="ts">
import { ref, computed } from 'vue'

interface Props {
  spaceId: string
  showActions?: boolean
}

const props = withDefaults(defineProps<Props>(), {
  showActions: true
})

const emit = defineEmits<{
  enter: [spaceId: string]
  like: [spaceId: string]
}>()

const isLiked = ref(false)

const coverStyle = computed(() => ({
  backgroundImage: `url(${props.coverUrl})`
}))
</script>

<template>
  <div class="space-card" :style="coverStyle">
    <!-- 单根节点 -->
  </div>
</template>

<style module>
.space-card { /* ... */ }
</style>
```

### 3.3 禁止事项

- ❌ Options API（统一使用 Composition API + `<script setup>`）
- ❌ `any` 类型（除极特殊情况）
- ❌ 直接操作DOM（使用ref/template ref）
- ❌ 深层嵌套 > 4层
- ❌ 组件文件 > 300行

## 4. Python 编码规范

### 4.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 模块/文件 | snake_case | `avatar_generator.py` |
| 类名 | PascalCase | `AvatarGenerator` |
| 函数/方法 | snake_case | `generate_avatar()` |
| 变量 | snake_case | `user_id` |
| 常量 | UPPER_SNAKE_CASE | `MAX_IMAGE_SIZE` |
| 私有成员 | _前缀 | `_internal_method()` |

### 4.2 类型注解

```python
# ✅ 正确 - 使用类型注解
from typing import Optional, List
from uuid import UUID

async def get_avatar(
    avatar_id: UUID,
    include_outfits: bool = False
) -> Optional[dict]:
    """获取虚拟人信息。"""
    ...

# ❌ 错误 - 无类型注解
def get_avatar(avatar_id):
    ...
```

### 4.3 FastAPI路由规范

    request: GenerateRequest,

## 5. C++17 编码规范

### 5.1 核心规范

```cpp
// ✅ 使用现代C++特性
auto result = std::make_unique<Renderer>(config);
auto callback = [this](const Event& e) { handle(e); };

// ✅ RAII资源管理
class Texture {
public:
    explicit Texture(const std::string& path);
    ~Texture();  // 自动释放GPU资源
    Texture(const Texture&) = delete;
    Texture& operator=(const Texture&) = delete;
    Texture(Texture&&) noexcept = default;
};

// ❌ 禁止
// - 裸指针所有权传递
// - C风格类型转换 (使用static_cast/dynamic_cast)
// - 手动new/delete (使用智能指针)
// - 全局变量
```

### 5.2 渲染引擎规范

```cpp
// 命名空间
namespace solra::render {

// 接口使用纯虚类
class IRenderBackend {
public:
    virtual ~IRenderBackend() = default;
    virtual void initialize(const RenderConfig& config) = 0;
    virtual void render(const Scene& scene) = 0;
};

}  // namespace solra::render
```

## 6. SQL 编码规范

### 6.1 命名规范

```sql
-- ✅ 正确
CREATE TABLE user_roles (
    user_id UUID NOT NULL REFERENCES users(id),
    role_id UUID NOT NULL REFERENCES roles(id),
    granted_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (user_id, role_id)
);

-- ❌ 错误
CREATE TABLE UserRoles (  -- 大写表名
    UserId int,           -- PascalCase列名 + 错误类型
    ...
);
```

### 6.2 编写规范

- 关键字大写：`SELECT`, `FROM`, `WHERE`, `JOIN`
- 标识符小写下划线：`user_id`, `created_at`
- 所有字段显式列出，禁用 `SELECT *`
- 复杂查询添加注释
- 使用参数化查询，禁止拼接SQL

## 7. Git 提交规范

### 7.1 Conventional Commits

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

### 7.2 Type类型

| Type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug修复 |
| `docs` | 文档变更 |
| `style` | 代码格式（不影响逻辑） |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具变更 |
| `ci` | CI/CD变更 |

### 7.3 示例

```
feat(avatar): add AI avatar generation from photo

- Add InsightFace face detection
- Add FLAME parameter extraction
- Add GAN texture synthesis

Closes #123
```

## 8. 代码审查检查清单

| 检查项 | 说明 |
|--------|------|
| 命名规范 | 是否符合命名规范 |
| 单一职责 | 方法/类是否职责清晰 |
| 错误处理 | 异常是否正确处理 |
| 日志记录 | 关键路径是否有日志 |
| 单元测试 | 是否有测试覆盖 |
| 性能考虑 | 是否有N+1查询/大循环 |
| 安全考虑 | 是否有注入风险/敏感信息泄露 |
| 文档注释 | 公共API是否有JavaDoc/TSDoc |
