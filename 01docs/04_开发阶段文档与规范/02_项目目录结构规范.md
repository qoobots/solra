## 1. Monorepo顶层结构

├── README.md                    # 项目总览
├── LICENSE                      # 开源协议
├── Makefile                     # 顶层构建入口
├── build.gradle.kts             # Gradle构建（Java/Kotlin）
├── .editorconfig                # 编辑器配置
├── .gitignore
├── .github/                     # GitHub Actions CI/CD
│   └── workflows/
├── 01docs/                      # 项目文档（113份阶段化文档）
├── core/                        # 设备端核心引擎 (C++17)
│   ├── CMakeLists.txt
│   ├── src/
│   │   ├── render/              # 渲染引擎
│   │   ├── physics/             # 物理引擎
│   │   ├── network/             # 网络层
│   │   └── audio/               # 音频引擎
│   ├── include/
│   ├── tests/
│   └── docs/
├── ai-services/                 # AI服务 (Python)
│   ├── brain/                   # qoobrain AI大脑
│   ├── body/                    # qoobody 虚拟身体
│   ├── safety/                  # 内容安全模型
│   ├── shared/                  # 共享工具
│   ├── tests/
│   └── docs/
├── clients/                     # 客户端应用
│   ├── web/                     # qooweb (Vue 3)
│   ├── mobile/                  # mobile_app (UniApp)
│   ├── desktop/                 # qoodev/qooremote (PySide6)
│   └── mini-program/            # 微信小程序
├── contracts/                   # 接口契约
│   ├── proto/                   # gRPC Proto定义
│   ├── api/                     # OpenAPI规范
│   ├── avro/                    # Avro Schema (Kafka)
│   ├── database/                # SQL迁移脚本
│   ├── deploy/                  # K8s部署配置
│   └── flowcharts/              # 业务流程图
└── tools/                       # 开发工具脚本
    ├── codegen/                 # 代码生成
    ├── scripts/                 # 运维脚本
    └── docker/                  # Docker配置
```

## 2. Java微服务目录结构

```

│   │   ├── java/com/solra/{domain}/

│   │   │   │   └── impl/                   # 业务实现

## 6. 测试目录规范

| 测试类型 | 目录 | 命名规范 |
|----------|------|----------|
| 单元测试 | `tests/unit/` | `{ClassName}Test` |
| 集成测试 | `tests/integration/` | `{Feature}IntegrationTest` |
| 端到端测试 | `tests/e2e/` | `{Scenario}E2ETest` |
| 性能测试 | `tests/benchmark/` | `{Component}Benchmark` |
| 测试夹具 | `tests/fixtures/` | JSON/CSV数据文件 |

---

## 7. 禁止事项

- ❌ 循环依赖（模块间）
- ❌ 代码目录深度 > 6层
- ❌ 单文件 > 500行（Java）/ 300行（Vue/TS）/ 200行（Python）
- ❌ 同一文件混合多种职责
- ❌ 测试代码放在 `src/` 目录下
