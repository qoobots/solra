# 基础设施层（Infra）交互设计

> infra/ 的"交互"——开发者通过Git提交变更，系统自动执行基础设施变更

---

## 一、核心交互模式

```
开发者 → Git Push (infra变更)
  → CI: Terraform Plan → 输出变更预览
  → PR Review → 审查变更
  → Merge → Terraform Apply → ArgoCD Sync
  → 通知 (Slack/邮件)
```

---

## 二、Terraform 工作流

```
terraform init    → 初始化provider
terraform plan    → 预览变更
terraform apply   → 应用变更
terraform destroy → 销毁资源
```

---

## 三、ArgoCD 同步

```
Git变更 → ArgoCD 检测差异
  → 自动同步 (dev环境)
  → 手动同步 (staging/prod环境)
  → 健康检查 (Readiness Probe)
  → 回滚 (失败时)
```

---

## 四、告警通知

| 级别 | 通道 | 触发条件 |
|------|------|---------|
| P0 | 电话+Slack | 生产服务不可用 |
| P1 | Slack+邮件 | 服务降级/延迟过高 |
| P2 | Slack | 非核心服务异常 |
