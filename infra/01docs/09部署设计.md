# 基础设施层（Infra）部署设计

> infra/ 自身部署——Terraform引导 → K8s部署 → GitOps自动化

---

## 一、部署顺序

```
1. terraform/      → 创建云资源 (VPC/节点/数据库)
2. kubernetes/base/ → K8s命名空间/RBAC/资源配额
3. monitoring/      → Prometheus/Grafana/Loki/Tempo
4. apisix/          → API网关部署
5. kubernetes/charts/ → 微服务/AI服务 Helm部署
6. kubernetes/argocd/ → GitOps 自动同步接管
```

---

## 二、环境部署状态

| 环境 | Terraform | K8s基础 | 监控 | 服务部署 | 状态 |
|------|----------|---------|------|---------|------|
| dev | ⬜ | ⬜ | ⬜ | ⬜ | ⬜ |
| staging | ⬜ | ⬜ | ⬜ | ⬜ | ⬜ |
| prod | ⬜ | ⬜ | ⬜ | ⬜ | ⬜ |

---

## 三、CI/CD流水线

```
Git Push (infra/**)
  → terraform fmt -check
  → terraform validate
  → terraform plan (输出到PR Comment)
  → Merge → terraform apply
  → K8s manifests apply → ArgoCD接管
```
