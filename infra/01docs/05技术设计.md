# 基础设施层（Infra）技术设计

> infra/ 技术选型——Terraform + K8s + Helm + ArgoCD + APISIX

---

## 一、技术选型

| 维度 | 选择 | 用途 |
|------|------|------|
| IaC | Terraform / OpenTofu | 多云资源编排 |
| 容器编排 | Kubernetes 1.30+ | 工作负载管理 |
| 包管理 | Helm 3 | K8s应用打包 |
| GitOps | ArgoCD | 自动同步部署 |
| 网关 | APISIX | API路由/认证/限流 |
| 监控 | Prometheus + Grafana | 指标采集展示 |
| 日志 | Loki + Fluent Bit | 日志聚合 |
| 追踪 | Tempo + OpenTelemetry | 链路追踪 |
| 密钥 | External Secrets / Vault | Secret管理 |
| 网络 | Cilium | CNI + 网络策略 |

## 三、标准资源配置

```yaml
# 通用 Deployment 模板
apiVersion: apps/v1
kind: Deployment
spec:
  replicas: 2
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 0
  template:
    spec:
      containers:
      - name: app
        resources:
          requests: {cpu: 250m, memory: 256Mi}
          limits: {cpu: 500m, memory: 512Mi}
        livenessProbe: ...
        readinessProbe: ...
```
