# 基础设施层（Infra）应用架构设计

> infra/ 在索拉架构中的位置——基础设施支撑层，承载所有服务的运行环境

---

## 一、架构定位

```
clients/ ← 用户终端
    ↓
APISIX ← infra/apisix (网关)
    ↓
K8s Cluster ← infra/kubernetes (编排)
    ├── services/ (微服务)
    ├── ai-services/ (AI服务)
    ├── PostgreSQL / Redis / Kafka (中间件)
    └── Monitoring ← infra/monitoring (观测)
    ↓
Cloud Resources ← infra/terraform (云资源)
```

---

## 二、基础设施层级

```
L0: 云资源 (Terraform)
    ├── Compute (K8s节点 / GPU节点)
    ├── Network (VPC / 子网 / 负载均衡)
    └── Storage (对象存储 / 块存储)

L1: 容器编排 (Kubernetes)
    ├── 命名空间隔离
    ├── 工作负载调度
    ├── 服务发现/负载均衡
    └── 自动伸缩(HPA)

L2: 服务网格 (APISIX)
    ├── 路由/Load Balance
    ├── 认证/鉴权
    └── 限流/熔断

L3: 可观测性 (Monitoring)
    ├── Metrics → Prometheus/Grafana
    ├── Logs → Loki
    └── Traces → Tempo
```

---

## 三、GitOps 架构

```
GitHub (infra/)
    ↓
CI: Terraform Plan → Apply
    ↓
ArgoCD
    ├── dev cluster (自动同步)
    ├── staging cluster (手动触发)
    └── prod cluster (审批后同步)
```
