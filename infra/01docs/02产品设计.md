# 基础设施层（Infra）产品设计

> infra/ 产品形态——可复制、可审计、可恢复的基础设施即代码

---

## 一、产品形态

infra/ 的"产品"是一套可执行的声明式配置文件集合，输出一个可运行的分布式系统环境。

```
infra/
├── docker/     → 容器镜像
├── terraform/  → 云资源
├── kubernetes/ → 工作负载
├── apisix/     → API网关
└── monitoring/ → 可观测性
```

---

## 二、部署架构

```
┌─────────────────────────────────┐
│          互联网用户              │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  APISIX (API Gateway)           │
│  ├── 路由 → 微服务              │
│  ├── 认证 → JWT校验             │
│  └── 限流 → Rate Limit          │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  Kubernetes Cluster             │
│  ├── services/ (10微服务)       │
│  ├── ai-services/ (5 AI服务)    │
│  ├── PostgreSQL                 │
│  ├── Redis                      │
│  └── Kafka                      │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  Monitoring Stack               │
│  Prometheus + Grafana + Loki    │
└─────────────────────────────────┘
```

---

## 三、环境策略

| 环境 | 用途 | 资源规模 | 数据 |
|------|------|---------|------|
| dev | 日常开发 | 最小 (1节点) | 测试数据 |
| staging | 预发布验证 | 中等 (3节点) | 脱敏生产数据 |
| prod | 生产 | 完整 (5+节点) | 生产数据 |

---

## 四、GitOps 流程

```
Git Push (infra变更)
  → CI: Terraform plan
  → PR Review
  → Merge
  → Terraform apply
  → ArgoCD sync (K8s)
  → 健康检查
```
