# 客户端层（Clients）产品设计

> 索拉客户端产品形态——空间优先的3D社交终端体验

---

## 一、产品形态

| 客户端 | 产品定位 | 核心差异 |
|--------|---------|---------|
| iOS | 第一优先级体验平台 | Metal原生渲染+CoreML推理 |
| Android | 最大用户量平台 | Vulkan渲染+NNAPI推理 |
| Web | 轻量访问端 | 空间发现/用户管理/消息中心（不承担3D渲染） |
| Windows Desktop | 主力3D体验平台 | Tauri 2+Rust FFI→Core C++17引擎，全GPU性能 |

> **架构决策（2026-06-30）**：Web 客户端不承担复杂 3D 场景渲染。大型 3D 社交游戏的性能需求（原生 GPU 利用率、64-bit 内存、端侧 AI 推理、WebRTC 原生编解码）需要桌面原生客户端来满足。

---

## 二、统一技术栈

| 维度 | iOS | Android | Web | Windows Desktop |
|------|-----|---------|-----|-----------------|
| 语言 | Swift 5.9 | Kotlin 2.0 | TypeScript 5 | Rust + TypeScript 5 |
| UI | SwiftUI | Compose | Vue 3 | Vue 3 (复用Web端) |
| 渲染 | Metal 3 | Vulkan | — (不承担3D) | OpenGL/Vulkan (Core SDK) |
| 网络 | URLSession | OkHttp | fetch/SSE | reqwest + WebSocket |
| 存储 | CoreData | Room | IndexedDB | SQLite + 文件系统 |
| Core SDK | SolraCore.framework | libsolracore.so | N/A | Rust FFI → libsolracore.dll |
| 构建 | Xcode | Gradle KTS | Vite | Cargo + Vite |
| 壳层 | — | — | — | Tauri 2 |

## 三、导航架构

```
移动端 (iOS/Android):
App 入口
├── Tab1: 发现 (推荐流)
│   ├── 空间预览卡片
│   └── → 全屏3D空间
├── Tab2: 我的空间
│   ├── 个人空间管理
│   └── 虚拟人配置
├── Tab3: 消息/通知
└── Tab4: 我的 (设置/隐私)

Web 端 (轻量访问):
├── 首页: 空间发现 Feed 流
├── 登录
├── 空间详情
├── 创建空间（轻量编辑器）
├── 个人主页
├── 商城
├── 排行榜
└── 消息中心

Desktop 端 (主力3D体验):
├── 窗口1: 空间发现 Feed（Vue 3 WebView）
├── 窗口2: 3D 空间渲染（OpenGL 原生视口 + Core SDK）
├── 虚拟人对话面板（Vue 3 浮层）
└── 系统托盘 + 通知
```

## 四、Core SDK 集成

```
iOS:
  SwiftUI → Swift Bridge → SolraCore.framework → C++17

Android:
  Compose → Kotlin Bridge (JNI) → libsolracore.so → C++17

Windows Desktop:
  Vue 3 UI → Tauri IPC → Rust FFI (unsafe C ABI) → libsolracore.dll → C++17

Web:
  Vue 3 → HTTP/gRPC-Web → 云端服务（不集成Core SDK）
```
