# 客户端层（Clients）安全设计

> 客户端安全——代码保护、数据加密、运行时安全

---

## 一、安全层级

```
传输安全 (TLS 1.3 + Certificate Pinning)
  → 存储安全 (KeyChain/KeyStore + 加密文件)
    → 运行时安全 (越狱/Root检测 + 反调试)
      → 代码安全 (混淆 + Integrity Check)
```

---

## 二、关键安全措施

| 措施 | iOS | Android | Web |
|------|-----|---------|-----|
| 代码混淆 | Swift代码混淆 | ProGuard/R8 | Source Map隐藏 |
| 越狱/Root检测 | 是 | 是 | N/A |
| SSL Pinning | 是 | 是 | — |
| KeyChain/KeyStore | Token/密钥 | Token/密钥 | Cookie |
| 文件加密 | Data Protection | EncryptedFile | — |
| 内存保护 | 敏感数据memset | 敏感数据memset | — |

---

## 三、数据最小化

| 原则 | 实践 |
|------|------|
| 不存储明文密码 | 仅存Token |
| 对话日志脱敏 | 本地日志不含用户输入 |
| 崩溃日志脱敏 | 去除个人信息 |
| 网络请求不传设备ID | 用匿名会话ID |
