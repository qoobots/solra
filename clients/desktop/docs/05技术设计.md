## 1. 技术选型

### 1.1 壳层：Tauri 2

| 项目 | 选型 | 说明 |
|------|------|------|
| 框架 | Tauri 2.x | Rust 后端 + WebView 前端 |
| 窗口管理 | `tauri::Window` | 原生窗口控制 |
| 系统插件 | shell / dialog / notification | 官方插件 |
| 包体积 | < 5MB（壳层） | 不含 Core SDK DLL |

**选型理由**：
- 比 Electron 体积小 20x+
- Rust 内存安全 + 高性能
- 原生 FFI 调用能力（加载 Core SDK）
- 跨平台（Windows/macOS/Linux）

### 1.2 前端：Vue 3 + TypeScript

| 项目 | 选型 | 版本 |
|------|------|------|
| 框架 | Vue 3 (Composition API) | 3.4+ |
| 语言 | TypeScript (strict) | 5.3+ |
| UI 库 | Element Plus | 2.5+ |
| 构建 | Vite | 5.x |
| 状态管理 | Pinia | 2.x |
| 路由 | vue-router | 4.x |
| 3D | Three.js | 0.185 |
| 测试 | Vitest | 2.x |

### 1.3 后端：Rust

| 项目 | 选型 | 说明 |
|------|------|------|
| 异步运行时 | Tokio (full) | 多线程异步 |
| HTTP 客户端 | reqwest | 支持 JSON + Stream |
| 序列化 | serde + serde_json | Rust 标准序列化 |
| 动态加载 | libloading | 加载 Core SDK DLL |
| 日志 | log + env_logger | 分级日志 |
| 错误处理 | anyhow + thiserror | 灵活错误处理 |

## 2. 编码规范

### 2.1 TypeScript / Vue

```typescript
// 命名规范
// 组件：PascalCase，如 RenderViewport.vue
// 组合式函数：camelCase + use 前缀，如 useRenderer.ts
// Store：camelCase + Store 后缀，如 renderStore.ts
// 类型/接口：PascalCase，如 SceneData, RenderConfig
// 常量：UPPER_SNAKE_CASE，如 MAX_FPS

// 文件组织
// 每个 .vue 文件 < 300 行
// 每个 .ts 文件 < 500 行
// 复杂逻辑抽取为 composables
```

### 2.2 Rust

```rust
// 命名规范
// 模块：snake_case，如 render_cmd
// 函数：snake_case，如 init_renderer
// 结构体：PascalCase，如 RenderConfig
// 常量：UPPER_SNAKE_CASE，如 DLL_SEARCH_PATHS

// 错误处理
// 使用 anyhow::Result 作为 IPC 命令返回值
// 使用 thiserror 定义自定义错误类型
// 避免 unwrap()，使用 ? 操作符
```

### 3.1 Core SDK DLL 加载（ffi.rs）

// 6 个搜索路径优先级
const SEARCH_PATHS: &[&str] = &[

    "../../../core/build/Release/libsolracore.dll",
    // 3. 开发构建输出 (Debug)

    "@executable_path/../Frameworks/libsolracore.dylib",

### 3.2 渲染初始化流程（render.rs）

```rust
pub fn init_render(width: u32, height: u32) -> Result<RenderInfo> {
    let sdk = get_sdk()?;
    
    let config = RenderConfig {
        width,
        height,
        backend: Backend::Auto,
        vsync: true,
        msaa: 4,
        native_window: std::ptr::null_mut(), // 离屏模式
    };
    
    unsafe { (sdk.render_init)(&config) };
    
    let gpu_info = unsafe { (sdk.get_gpu_info)() };
    
    Ok(RenderInfo {
        initialized: true,
        gpu_backend: gpu_info.to_string(),
        resolution: (width, height),
    })
}
```

### 3.3 渲染循环

```rust
// IPC 命令：前端每帧调用
#[tauri::command]
async fn render_frame(state: State<'_, AppState>) -> Result<FrameResult, String> {
    let sdk = state.get_sdk()?;
    
    unsafe { (sdk.begin_frame)() };
    // Core SDK 内部执行：
    //   1. 场景图脏标记更新
    //   2. 视锥体裁剪
    //   3. GPU 命令录制
    //   4. 提交到命令队列
    unsafe { (sdk.end_frame)() };
    
    let fps = unsafe { (sdk.get_fps)() };
    
    Ok(FrameResult { fps, frame_time_ms: 1000.0 / fps })
}
```

### 3.4 前端 Three.js 降级渲染

```typescript
// RenderViewport.vue
const initRenderer = async () => {
  // 尝试初始化 Core SDK
  const result = await invoke<RenderInfo>('init_renderer', {
    width: 1280, height: 800
  });
  
  if (result.initialized) {
    renderStore.useCoreSdk = true;
    renderStore.gpuBackend = result.gpu_backend;
    startCoreRenderLoop();
  } else {
    // 降级到 Three.js WebGL
    renderStore.useCoreSdk = false;
    renderStore.gpuBackend = 'WebGL (降级模式)';
    initThreeJS();
    startThreeJSRenderLoop();
  }
};
```

## 4. 构建流程

### 4.1 开发模式

```
npm run tauri dev
  ├── Vite dev server (localhost:5173) → HMR
  ├── cargo build (Rust 后端) → 自动重编译
  └── Tauri WebView → 加载 localhost:5173
```

### 4.2 生产构建

```
npm run tauri build
  ├── vue-tsc -b → TypeScript 类型检查
  ├── vite build → 前端打包到 dist/
  ├── cargo build --release → Rust 优化编译
  ├── 资源打包（图标、配置文件）
  └── 生成安装包：
      ├── Windows: .msi / .exe (NSIS)
      ├── macOS: .dmg / .app bundle
      └── Linux: .AppImage / .deb
```

### 4.3 构建产物

```
src-tauri/target/release/bundle/
├── msi/          # Windows 安装包
├── dmg/          # macOS 安装包
└── appimage/     # Linux AppImage
```

## 5. 性能优化

### 5.1 前端优化

- Vue 3 异步组件 + Suspense
- 虚拟列表（空间列表大量条目时）
- Three.js 纹理压缩（KTX2/Basis）
- Canvas 离屏渲染
- requestAnimationFrame 节流

### 5.2 后端优化

- Tokio 多线程运行时（线程数 = CPU 核心数）
- IPC 命令异步处理
- Core SDK 调用零拷贝（共享内存）
- DLL 预加载 + 函数指针缓存

### 5.3 渲染优化

- 视锥体裁剪（Core SDK 侧）
- LOD 多级细节
- GPU 实例化渲染
- 纹理 Mipmap
- 延迟着色（未来）
