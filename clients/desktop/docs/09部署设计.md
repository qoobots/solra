## 2. 构建流程

### 2.1 开发构建

```bash
# 安装依赖
cd clients/desktop
npm install

# 启动开发模式（Vite + Tauri）
npm run tauri dev

# 仅前端开发
npm run dev

# 仅 Rust 后端编译
cd src-tauri && cargo build
```

### 2.2 生产构建

```bash
# 完整构建（TypeScript 检查 + 前端打包 + Rust Release 编译 + 打包）
npm run tauri build

# 分步执行
npm run build                    # vue-tsc -b && vite build
cd src-tauri && cargo build --release
```

### 2.3 平台特定构建

```bash
# Windows
npm run tauri build -- --target x86_64-pc-windows-msvc

# macOS (Intel)
npm run tauri build -- --target x86_64-apple-darwin

# macOS (Apple Silicon)
npm run tauri build -- --target aarch64-apple-darwin

# Linux
npm run tauri build -- --target x86_64-unknown-linux-gnu
```

## 4. Core SDK 部署

### 4.1 DLL 搜索路径

运行时按以下优先级搜索 `libsolracore`：

```
Windows:
  1. ./libsolracore.dll                     # 可执行文件同级
  2. ../../../core/build/Release/            # 开发构建输出
  3. ../../../core/build/Debug/              # 开发构建输出
  4. libsolracore.dll                        # 系统 PATH

macOS:
  1. @executable_path/../Frameworks/        # Bundle 内部
  2. ./libsolracore.dylib                   # 可执行文件同级

Linux:
  1. ./libsolracore.so                      # 可执行文件同级
  2. libsolracore.so                        # 系统 LD_LIBRARY_PATH
```

### 4.2 打包策略

| 场景 | 策略 |
|------|------|
| 开发环境 | 从 `core/build/` 自动发现 |
| 生产打包 | 复制 DLL 到安装包内 |
| 增量更新 | DLL 可独立更新（热更新） |
| 降级 | DLL 不存在时使用 Three.js WebGL |

## 5. 发布流程

### 5.1 版本号规范

遵循语义化版本 (SemVer)：`MAJOR.MINOR.PATCH`

| 变更类型 | 版本号变化 |
|------|-----------|
| 重大不兼容变更 | MAJOR +1 |
| 新功能（兼容） | MINOR +1 |
| Bug 修复 | PATCH +1 |

### 5.2 发布检查清单

- [ ] 所有单元测试通过
- [ ] TypeScript 类型检查通过
- [ ] Rust clippy 无警告
- [ ] 各平台构建成功
- [ ] 安装包签名完成
- [ ] 更新日志已编写
- [ ] 版本号已更新（package.json / Cargo.toml / tauri.conf.json）

### 5.3 发布渠道

| 渠道 | 用途 | 更新策略 |
|------|------|---------|
| **Canary** | 每日构建，最新代码 | 自动更新 |
| **Beta** | 功能测试 | 手动更新 |
| **Stable** | 正式发布 | 手动更新 + 通知 |

## 6. 自动更新

### 6.1 Tauri Updater

```json
// tauri.conf.json
{
  "plugins": {
    "updater": {
      "endpoints": [
        "https://releases.solra.io/desktop/{{target}}/{{current_version}}"
      ],
      "pubkey": "..." // 更新包签名公钥
    }
  }
}
```

### 6.2 更新流程

```
1. 应用启动 → 检查更新端点
2. 比较版本号
3. 有新版本 → 下载更新包
4. 校验签名
5. 提示用户重启
6. 重启后应用更新
```

### 6.3 增量更新

| 策略 | 说明 |
|------|------|
| 前端资源 | 差分更新（仅下载变更文件） |
| Rust 后端 | 全量替换二进制 |
| Core SDK DLL | 独立更新通道 |

## 7. CI/CD 流程

### 7.1 GitHub Actions 工作流

```yaml
# .github/workflows/desktop-build.yml
name: Desktop Build
on:
  push:
    branches: [main]
    paths: ['clients/desktop/**', 'core/**']
  pull_request:
    paths: ['clients/desktop/**', 'core/**']

jobs:
  test:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cd clients/desktop && npm ci
      - run: cd clients/desktop && npm test
      - run: cd clients/desktop && cargo test --manifest-path src-tauri/Cargo.toml

  build:
    needs: test
    if: github.ref == 'refs/heads/main'
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
    steps:
      - run: cd clients/desktop && npm run tauri build
      - uses: actions/upload-artifact@v4
        with:
          name: desktop-${{ matrix.os }}
          path: clients/desktop/src-tauri/target/release/bundle/
```

### 7.2 环境变量

| 变量 | 用途 |
|------|------|
| `TAURI_SIGNING_PRIVATE_KEY` | 更新包签名私钥 |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | 私钥密码 |
| `VITE_API_BASE_URL` | API 服务器地址 |
| `VITE_WS_URL` | WebSocket 服务器地址 |
