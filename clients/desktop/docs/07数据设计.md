## 1. 数据分层

```
┌──────────────────────────────────┐
│          远程数据（云端）          │

│  空间元数据 / 用户信息 / 消息     │
├──────────────────────────────────┤
│         本地持久化（磁盘）         │
│  localStorage / IndexedDB        │
│  设置 / 缓存 / 会话              │
├──────────────────────────────────┤

└──────────────────────────────────┘
```

## 2. 本地存储设计

### 2.1 存储方案

| 方案 | 用途 | 容量 | 访问方式 |
|------|------|------|---------|
| `localStorage` | 设置、偏好、会话 Token | < 5MB | 同步 |
| `IndexedDB` | 空间缓存、资产缓存、消息历史 | 无限制 | 异步 |
| 文件系统 | 3D 资产本地副本、日志 | 无限制 | Rust 后端 |

## 3. 缓存策略

### 3.1 资产缓存

| 策略 | 值 | 说明 |
|------|-----|------|
| 最大缓存大小 | 2GB | 超过时 LRU 淘汰 |
| 默认 TTL | 7 天 | 纹理/网格/动画 |
| Shader 缓存 | 30 天 | 编译后的 SPIR-V |
| 检查间隔 | 每次启动 | 清理过期缓存 |
| 缓存优先级 | 当前空间 > 最近访问 > 其他 | LRU 排序 |

### 3.2 空间数据缓存

| 策略 | 值 |
|------|-----|
| 最大缓存条目 | 50 个空间 |
| TTL | 24 小时 |
| 离线可用 | ✅ 已缓存空间可离线浏览 |
| 更新策略 | 后台定时刷新（15 分钟） |

### 3.3 消息缓存

| 策略 | 值 |
|------|-----|
| 每个会话最大消息数 | 1000 条 |
| 自动清理 | 超过 30 天的消息 |
| 同步策略 | 增量同步（最后一条消息时间戳） |

## 4. 数据同步

### 4.1 设置同步

```
本地修改 → Pinia Store → localStorage 持久化
  → (可选) 云端同步 → 其他设备同步
```

### 4.2 场景数据同步

```
用户进入空间
  → 检查 IndexedDB 缓存
    → 缓存有效 → 直接使用
    → 缓存过期 → 后台更新，先用旧数据
    → 无缓存 → 从服务器加载
```

### 4.3 消息同步

```
WebSocket 连接建立
  → 拉取增量消息（lastMessageAt 之后）
  → 合并到 IndexedDB
  → 实时消息通过 WebSocket 推送
  → 存储到 IndexedDB + 更新 UI
```

## 6. 数据安全

| 措施 | 说明 |
|------|------|
| JWT Token | 存储在 localStorage，HTTPS 传输 |
| 敏感设置 | 不存储密码，仅存 Token |
| 缓存清理 | 退出登录时清除所有本地数据 |
| 加密存储 | 敏感资产（用户模型）可选项加密 |

## 7. 离线支持

| 功能 | 离线可用 | 说明 |
|------|---------|------|
| 已缓存空间浏览 | ✅ | 3D 场景数据已缓存 |
| 设置修改 | ✅ | 本地存储 |
| 消息查看 | ✅ | 已缓存的历史消息 |
| AI 对话 | 🔶 | 需 Core SDK 端侧推理 |
| 新空间加载 | ❌ | 需要网络 |
| 实时通话 | ❌ | 需要 WebRTC |
| 账号登录 | ❌ | 需要验证服务器 |
