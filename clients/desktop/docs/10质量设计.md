## 1. 测试策略

### 1.1 测试金字塔

```
           ┌──────┐
           │ E2E  │  少量端到端测试
           │ 10%  │  (Playwright / Tauri test)
          ┌┴──────┴┐
          │ 集成测试 │  模块间交互
          │   20%   │  (Vitest + Rust integration)
         ┌┴────────┴┐
         │  单元测试  │  函数/组件级别
         │    70%    │  (Vitest + cargo test)
         └──────────┘
```

### 1.2 测试工具

| 层级 | 工具 | 说明 |
|------|------|------|
| TypeScript 类型检查 | `vue-tsc` / `tsc --noEmit` | 构建前必须通过 |
| 前端单元测试 | Vitest + happy-dom | Vue 组件 + 工具函数 |
| 前端组件测试 | @vue/test-utils | 组件渲染 + 交互 |
| Rust 单元测试 | `cargo test` | Rust 模块测试 |
| Rust 集成测试 | `cargo test --test '*'` | IPC 命令测试 |
| E2E 测试 | Playwright (未来) | 完整用户流程 |
| Rust Lint | `cargo clippy` | 代码质量 |
| Rust 格式化 | `cargo fmt --check` | 代码风格 |

### 1.3 测试覆盖率要求

| 模块 | 行覆盖率 | 分支覆盖率 |
|------|---------|-----------|
| Vue 组件 | ≥ 70% | ≥ 60% |
| Pinia Stores | ≥ 80% | ≥ 70% |
| Composables | ≥ 80% | ≥ 70% |
| Rust IPC 命令 | ≥ 80% | ≥ 70% |
| Rust FFI 桥接 | ≥ 60% | ≥ 50% |
| Core SDK (C++) | ≥ 60% | ≥ 50% |

## 2. 性能基准

### 2.1 启动性能

| 指标 | 目标 | 测量方式 |
|------|------|---------|
| 冷启动时间 | < 3s | 从双击到首帧渲染 |
| 热启动时间 | < 1s | 从系统托盘恢复 |
| 首屏渲染 | < 500ms | Vite dev 模式 |
| Core SDK 加载 | < 200ms | DLL 加载 + 初始化 |

### 2.2 渲染性能

| 指标 | 最低 | 目标 |
|------|------|------|
| 3D 视口帧率 | 30 FPS | 60 FPS |
| UI 界面帧率 | 60 FPS | 120 FPS |
| 帧时间 (16.67ms @60fps) | < 33ms | < 16ms |
| GPU 占用率 | - | < 80% |

### 2.3 内存

| 指标 | 目标 |
|------|------|
| 空闲内存 | < 300MB |
| 加载 1 个空间 | < 500MB |
| 加载 5 个空间（缓存） | < 800MB |
| 显存占用 | < 2GB |
| 内存泄漏 | 0（连续运行 24h 无增长） |

### 2.4 网络

| 指标 | 目标 |
|------|------|
| API 响应时间 (P95) | < 500ms |
| 空间加载时间 | < 5s (含资产下载) |
| WebSocket 重连 | < 3s |

## 3. 测试用例清单

### 3.1 前端单元测试

| 测试对象 | 测试点 |
|---------|--------|
| SceneManager | 场景初始化、对象创建、光照配置 |
| SceneDataModel | JSON 解析、默认场景生成 |
| OrbitController | 旋转/缩放/平移计算、边界限制 |
| renderStore | 状态初始化、Core SDK 切换 |
| useRenderer | init/destroy/render 流程 |
| RenderViewport | 双通道切换、Canvas 创建 |

### 3.2 Rust 单元测试

| 测试对象 | 测试点 |
|---------|--------|
| ffi.rs | DLL 加载/卸载、函数指针获取 |
| render.rs | 初始化配置、GPU 信息获取 |
| render_cmd | IPC 参数验证、错误返回 |

### 3.3 集成测试

| 场景 | 测试点 |
|------|--------|
| Core SDK 可用 | 完整渲染管线初始化 → 帧渲染 → FPS 获取 |
| Core SDK 不可用 | 降级到 Three.js、无崩溃 |
| DLL 热替换 | 运行时 DLL 替换 → 重新加载 |
| 窗口缩放 | 渲染分辨率自适应 |

## 4. 监控与诊断

### 4.1 性能监控

| 指标 | 采集方式 | 显示位置 |
|------|---------|---------|
| FPS | requestAnimationFrame 计数 | 状态栏 + 调试面板 |
| 帧时间 | performance.now() | 调试面板 |
| GPU 后端 | Core SDK get_gpu_info() | 状态栏 |
| 内存使用 | performance.memory (Chrome) | 调试面板 |
| 网络延迟 | WebSocket ping/pong | 调试面板 |

### 4.2 日志系统

| 层级 | 日志工具 | 存储位置 |
|------|---------|---------|
| 前端 JS | console (分级) | 浏览器 DevTools |
| Rust 后端 | `env_logger` | `%APPDATA%/solra/logs/` |
| Core SDK | `spdlog` | `%APPDATA%/solra/logs/core/` |

**日志级别**：
- **开发**：DEBUG
- **生产**：INFO（WARN/ERROR 写入文件）

### 4.3 崩溃报告

| 组件 | 方案 |
|------|------|
| Rust 后端 | panic hook → 写入 crash dump |
| Core SDK | 信号处理 → 写入 minidump |
| 前端 | window.onerror → 收集堆栈 |
| 上报 | 用户确认后上报到 Sentry |
