## 2. 目录职责说明

### 2.1 `src/` — 前端

| 目录 | 职责 | 依赖方向 |
|------|------|---------|
| `views/` | 页面级组件，对应路由 | → components, stores |
| `components/` | 可复用组件（含 3D 渲染） | → stores, composables |
| `stores/` | 全局状态管理（Pinia） | → composables, services |
| `composables/` | 可复用逻辑（Composition API） | → services |
| `services/` | HTTP / IPC 通信层 | → types |
| `types/` | TypeScript 类型定义 | 无 |
| `styles/` | 全局样式 | 无 |
| `router/` | 路由配置 | → views |

### 2.2 `src-tauri/src/` — Rust 后端

| 目录 | 职责 | 依赖方向 |
|------|------|---------|
| `ipc/` | IPC 命令处理（Tauri invoke） | → core |
| `core/` | Core SDK 桥接 + 业务逻辑 | → libloading, serde |
| `lib.rs` | 模块注册 + Tauri 插件 | → ipc |
| `main.rs` | 程序入口 | → lib |

## 3. 命名规范

### 3.1 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| Vue 组件 | PascalCase | `RenderViewport.vue` |
| TypeScript 模块 | camelCase | `sceneManager.ts` |
| Store | camelCase + Store | `renderStore.ts` |
| Composable | camelCase + use 前缀 | `useRenderer.ts` |
| 类型文件 | camelCase | `render.ts` |
| Rust 模块 | snake_case | `render_cmd.rs` |
| 样式文件 | kebab-case | `main.scss` |

### 3.2 目录命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 功能目录 | kebab-case / 小写 | `renderer/`, `layout/` |
| Rust 模块目录 | 小写 | `ipc/`, `core/` |

## 4. 依赖关系图

```
views/
  └── components/
        └── stores/
              └── composables/
                    └── services/
                          ├── Tauri IPC → Rust ipc/ → Rust core/ → Core SDK
                          └── HTTP → Cloud API
```

**原则**：
- 上层依赖下层，下层不依赖上层
- `types/` 被所有层依赖
- `stores/` 不直接操作 DOM
- `services/` 是唯一与外部通信的层
