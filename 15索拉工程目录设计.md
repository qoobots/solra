# 索拉（Solra）工程目录设计文档

> 基于 Monorepo 的完整工程目录蓝图——从根目录到叶子文件，覆盖 10 个子项目 × 4 个平台的统一目录规范

---

## 一、目录设计总纲

### 1.1 设计原则

| 原则 | 说明 | 反例 |
|------|------|------|
| **约定优于配置** | 通过统一的目录命名约定，减少配置文件和团队认知成本 | 每个子项目自行定义目录结构 |
| **物理隔离 = 逻辑隔离** | 每个子项目拥有独立的顶级目录，清晰的物理边界即业务边界 | 跨子项目文件混放 |
| **自解释性** | 目录名直观反映用途，新人 5 分钟内能找到目标文件 | 使用缩写或项目内部代号 |
| **IDE 友好** | 每个可独立构建的单元有独立 `build.gradle.kts` / `CMakeLists.txt` / `pyproject.toml` | 根级大杂烩配置 |
| **生成/手写分离** | Proto 生成的代码统一在 `build/generated/` 下，与手写代码物理隔离 | 生成代码混入 `src/` |
| **测试镜像源码** | 测试目录结构与源码目录结构一一镜像 | 测试文件集中堆放 |

### 2.1 根级配置文件说明

| 文件 | 用途 | 关键内容 |
|------|------|---------|
| `README.md` | 项目入口文档 | 快速开始、架构概览、贡献指南、子项目导航 |

| `gradle.properties` | Gradle 全局属性 | JVM 参数、构建缓存、并行构建开关 |

### 3.1 Proto 代码生成约定

| 目标语言 | 输出目录 | 生成工具 |
|---------|---------|---------|
| Java | `services/*/build/generated/source/proto/` | `protoc` + `grpc-java` |
| Python | `ai-services/*/src/generated/` | `protoc` + `grpcio-tools` |
| Swift | `clients/ios/SolraCore/Generated/` | `protoc` + `grpc-swift` |
| Kotlin | `clients/android/app/build/generated/source/proto/` | `protoc` + `grpc-kotlin` |
| TypeScript | `clients/web/src/generated/` | `protoc` + `grpc-web` |

> **铁律**：生成代码不进入 Git 版本控制（`.gitignore` 排除），CI 管道中自动生成。

### 4.1 C++ 编码约定

| 约定项 | 规则 |
|--------|------|
| **命名空间** | `solra::core` / `solra::render` / `solra::inference` 等 |
| **头文件** | `.h` 扩展名，`#pragma once` 防重入 |
| **源文件** | `.cpp` (通用) / `.mm` (Objective-C++) |
| **测试文件** | `*_test.cpp`，与源文件同目录（小型）/ 独立 `tests/` 目录（集成/性能） |
| **构建** | CMake 3.20+，依赖 Conan 管理 |
| **格式化** | `clang-format`（Google Style） |
| **测试框架** | Google Test + Google Benchmark |

### 5.3 Java 服务命名约定

| 约定项 | 规则 | 示例 |
|--------|------|------|
| **基础包名** | `com.solra.{子项目缩写}.{服务名}` | `com.solra.avt.conversation` |
| **领域模型** | 无框架注解，纯净 Java 类 | `Conversation.java`（不含 `@Entity`） |
| **持久化实体** | `{Model}Entity`，仅出现在 `infrastructure/persistence/` | `ConversationEntity.java` |
| **领域服务** | `{动词}{名词}Service / {名词}Orchestrator` | `ConversationService`, `DialogueOrchestrator` |
| **仓储接口** | `I{聚合根}Repository`（domain/）/ `{技术}{聚合根}Repository`（infrastructure/） | `ConversationRepository` (接口) / `MySqlConversationRepository` (实现) |
| **命令对象** | `{动作}{名词}Command` | `StartConversationCommand` |
| **查询对象** | `{名词}Query` | `ConversationHistoryQuery` |
| **处理器** | `{命令}Handler` | `StartConversationHandler` |
| **gRPC 服务** | `{ProtoService}GrpcService` | `ConversationGrpcService` |
| **REST 控制器** | `{名词}RestController` | `ConversationRestController` |
| **事件监听器** | `{事件名}EventListener` | `SpaceEnteredEventListener` |
| **测试类** | `{被测类}Test` | `ConversationServiceTest` |
| **数据库迁移** | `V{序号}__{描述}.sql` | `V1__create_conversation_table.sql` |

## 十一、跨子项目文件命名规则

### 11.1 统一命名约定

| 元素 | 规则 | 示例 |
|------|------|------|
| **Java 包名** | `com.solra.{子项目缩写}.{服务名}` | `com.solra.avt.conversation` |
| **Python 包名** | `solra_ai_{服务名}` | `solra_ai_llm_router` |
| **Swift 模块** | `Solra{功能域}` | `SolraSpace`, `SolraAvatar` |
| **Kotlin 包名** | `com.solra.android.{功能域}` | `com.solra.android.space` |
| **TypeScript 目录** | 小写字母 + 连字符 | `design-system/`, `space-viewer/` |
| **Proto 文件** | 蛇形命名 `.proto` | `conversation.proto` |
| **Avro Schema** | 蛇形命名 `.avsc` | `user_events.avsc` |
| **Helm Chart** | `solra-{组件}` | `solra-services`, `solra-ai` |
| **Docker 镜像** | `solra/{服务名}:{version}` | `solra/conversation-svc:1.0.0` |
| **K8s 资源名** | `{子项目}-{服务}-{环境}` | `avt-conversation-svc-prod` |
| **数据库 Schema** | `solra_{子项目缩写}` | `solra_avt`, `solra_spc` |
| **数据库表名** | 蛇形命名 + 复数 | `conversations`, `dialogue_turns` |
| **Redis Key** | `solra:{子项目}:{实体}:{id}` | `solra:avt:conversation:12345` |
| **Pulsar Topic** | `persistent://solra/{子项目}/{事件名}` | `persistent://solra/avt/conversation.started` |

### 11.2 文件命名禁止事项

| ❌ 禁止 | ✅ 推荐 |
|--------|--------|
| 拼音命名 | 英文语义命名 |
| 中文文件名（代码文件） | 英文文件名 |
| `utils/` 放一切 | 按职责分目录（`api/`, `storage/`, `analytics/`） |
| `common/` / `shared/` 过大 | 按依赖方向拆分为细粒度共享库 |
| 测试文件与源码混放 | test/ 镜像 src/ 结构 |
| 变量/文件大小写混乱 | 严格遵守语言社区惯例 |
| 深层嵌套 > 5 层 | 扁平化，最多 4 层目录嵌套 |
